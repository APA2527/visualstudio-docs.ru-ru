# <a name="signing-in-to-visual-studio-subscriptions-may-fail-when-using-aliases"></a>Вход в подписки Visual Studio может быть неудачным при использовании псевдонимов
В зависимости от типа учетной записи при входе в систему [https://my.visualstudio.com](https://my.visualstudio.com?wt.mc_id=o~msft~docs) могут неправильно отображаться доступные подписки. Одна из причин такой проблемы — использование дополнительных имен вместо идентификатора входа, которому назначена подписка. Это называется "применение псевдонимов".

## <a name="what-is-aliasing"></a>Что такое применение псевдонимов?
Термин "применение псевдонимов" относится к ситуации, при которой пользователи могут использовать разные идентификаторы для входа в Windows (или в Active Directory) и доступа к электронной почте.

Например, работа компании может быть организована так, что для входа в основной каталог используется учетная запись Microsoft Online, например olivia@contoso.com, а для работы с электронной почтой пользователи вводят другие имена (псевдонимы), например OliviaG@contoso.com. Убедитесь, что пользователи выполняют вход, используя "Адрес электронной почты для входа", указанный на портале администрирования подписок на Visual Studio по адресу https://manage.visualstudio.com для доступа к подпискам

## <a name="as-an-administrator-what-options-do-i-have"></a>Какие в этой ситуации есть варианты у администратора?

В зависимости от типа учетной записи подписчика найдите применимое решение ниже:

### <a name="work-or-school-account-upn-mismatch-issue"></a>Проблема несоответствия имени участника-пользователя рабочей или учебной учетной записи

Несоответствие имени участника-пользователя может быть обнаружено, если в компании есть система Active Directory, где имя участника-пользователя не совпадает с основным адресом SMTP. 

#### <a name="how-to-detect-if-a-users-sign-in-address-has-a-upn-mismatch"></a>Как определить несоответствие имени участника-пользователя в адресе для входа

Попросите пользователя выполнить приведенные ниже действия:

1. Войдите в https://my.visualstudio.com с помощью адреса для входа, указанного в сообщении электронной почты о назначении подписки.  

    > [!NOTE]
    > Если у них нет сообщения электронной почты о назначении подписки, вы можете повторно отправить его с портала администрирования.  

2. Перейдите на вкладку **Подписки**.
3. Убедитесь, что адрес электронной почты, который отображается в правом верхнем углу рядом с надписью "Вы вошли как...", совпадает с адресом электронной почты для входа в сообщении о назначении подписки.  В противном случае они не смогут получить доступ к преимуществам подписки. 

   > [!div class="mx-imgBorder"]
   > ![Страница "Подписки"](_img/aliasing/aliasing-subscriptions-page.png)

#### <a name="how-to-correct-the-upn-mismatch"></a>Исправление несоответствия имени участника-пользователя

1. Войдите на портал администрирования Visual Studio по адресу https://manage.visualstudio.com 

2. Найдите пользователя с несовпадающим именем участника-пользователя.  Функция [Фильтр](search-license.md) может упростить эту задачу, если у вас много подписок. 

3. Измените адрес электронной почты для входа на имя участника-пользователя.

4. Сохраните изменения 

5. Попросите пользователя выйти из портала подписчика и войти снова с помощью имени участника-пользователя.   

### <a name="personal-account-aliasing-issue"></a>Проблема с псевдонимом личной учетной записи

Проблемы с псевдонимом также могут повлиять на личные учетные записи. 

#### <a name="how-to-detect-if-a-personal-account-has-an-aliasing-issue"></a>Как определить, имеет ли личная учетная запись проблемы с псевдонимом

1. Войдите в https://my.visualstudio.com.

2. Перейдите на вкладку **Подписки** и проверьте адрес, с которым вы вошли. 

3. Если этот адрес электронной почты отличается от адреса, используемого для доступа к веб-сайту, между вашей учетной записью и псевдонимом возникает конфликт. 

#### <a name="how-to-fix-a-personal-account-aliasing-issue"></a>Исправление проблемы с псевдонимом личной учетной записи

Платформа подписок Visual Studio назначает приоритет основному псевдониму для отображения сведений о подписке.  Чтобы устранить эту проблему, необходимо сделать основным другой псевдоним электронной почты для входа. 

1. Перейдите в раздел [Управление входом в Майкрософт](https://go.microsoft.com/fwlink/p/?linkid=842796).
2. Выполните вход в свою учетную запись Майкрософт по запросу. 
3. В разделе "Псевдонимы учетных записей" выберите **Сделать основным** рядом с адресом электронной почты, используемым для назначения подписки. 
4. В разделе "Псевдонимы учетных записей" выберите "Сделать основным" рядом с адресом электронной почты, используемым для назначения подписки. 
5. Выйдите из портала подписчиков Visual Studio (https://my.visualstudio.com) 
6. Войдите на портал снова с помощью нового основного псевдонима. 

### <a name="ensure-a-successful-experience-for-your-users"></a>Обеспечение успешной работы для пользователей

У администратора такой системы есть два способа обеспечить подписчикам успешный вход в систему https://my.visualstudio.com. 

- Первый вариант (рекомендуется) — использование учетной записи каталога в качестве адреса для входа в систему https://manage.visualstudio.com.
- Второй вариант, менее безопасный, — разрешить подписчикам выполнять вход с использованием адреса электронной почты, отличного от адреса каталога.

Оба варианта настраиваются на портале администрирования следующим образом:

1. Выполните вход в https://manage.visualstudio.com 

2. Если вы изменяете одного пользователя, выберите этого пользователя в таблице и щелкните правой кнопкой мыши, чтобы внести правки. Откроется панель, где можно изменить адрес электронной почты для входа.  

3. Внесите необходимые обновления в поле адреса электронной почты для входа. 

4. Нажмите "Сохранить", и изменения вступят в силу.  
Если необходимо внести эти изменения для большого количества пользователей, можно использовать функцию группового редактирования. Дополнительные сведения об этом процессе см. в разделе **Изменение нескольких подписчиков с помощью операции массового редактирования** в статье [Изменение подписок] (edit-license.md).
