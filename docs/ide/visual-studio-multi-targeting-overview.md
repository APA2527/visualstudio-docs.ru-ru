# <a name="framework-targeting-overview"></a>Framework, предназначенных для обзора

В Visual Studio можно указать версию .NET, что будет целевой для проекта. Для выполнения на другом компьютере, которой предназначено приложение должны быть совместимы с версией платформы, которая устанавливается на компьютере версии платформы приложений .NET Framework.

Дополнительные сведения о целевых платформ см. в разделе [целевые платформы](/dotnet/standard/frameworks).

Кроме того, можно создать решение, содержащее проекты, ориентированные на разные версии платформы .NET. Нацеливание на платформу помогает гарантировать, что приложение использует только функциональные возможности, доступные в версии указанной платформы.

> [!TIP]
> Вы также можете нацеливать приложения на различные платформы. Дополнительные сведения см. в разделе [Настройка для различных версий](../msbuild/msbuild-multitargeting-overview.md).

## <a name="framework-targeting-features"></a>Функции нацеливания на платформу

Среди прочего, доступны следующие возможности нацеливания на платформу:

- При открытии проекта, предназначенного для более ранней версии платформы, Visual Studio может автоматически обновить проект или оставить в качестве целевого объекта-является.

- При создании проекта .NET Framework, можно указать версию .NET Framework, вы будете работать.

- Вы можете [ориентированного на несколько платформ](/dotnet/standard/frameworks#how-to-specify-target-frameworks) в одном проекте.

- Можно ориентироваться на различные версию .NET, в каждом из нескольких проектов в одном решении.

- Можно изменить версию .NET, существующий проект целевых объектов.

   При изменении версии .NET, который ориентирован проект, Visual Studio делает все необходимые изменения в ссылки и файлы конфигурации.

При работе над проектом, нацеленным более ранней версии платформы, Visual Studio динамически изменяет среду разработки, следующим образом:

- Фильтрует элементы в диалоговых окнах **Добавить новый элемент**, **Добавить новую ссылку** и **Добавление ссылки на службу**, чтобы пропустить варианты, которые недоступны в целевой версии.

- Фильтрует пользовательские элементы управления на **панели элементов**, чтобы удалить те из них, которые недоступны в целевой версии, и отобразить только наиболее актуальные элементы, если доступно несколько элементов управления.

- Фильтрует **IntelliSense** чтобы пропустить языковые функции, недоступные в целевой версии.

- Фильтрует свойства в **свойства** окно, чтобы пропустить те, которые недоступны в целевой версии.

- Фильтрует пункты меню для пропуска возможности, недоступные в целевой версии.

- Для сборок система использует версию и параметры компилятора, которые подходят для целевой версии.

> [!NOTE]
> - Нацеливание на платформу не гарантирует правильную работу приложения. Нужно протестировать приложение, чтобы убедиться в том, что оно работает с целевой версией.
> - Вы не можете выбрать framework версии ниже .NET Framework 2.0.

## <a name="select-a-target-framework-version"></a>Выбор целевой версии платформы

При создании проекта .NET Framework, можно выбрать целевой версии .NET Framework, выбрав шаблон проекта. Список доступных платформ включает их установленные версии, соответствующие типу выбранного шаблона. Для шаблонов проектов .NET Framework, например шаблонов .NET Core, **Framework** не отображается в раскрывающемся списке.

## <a name="resolve-system-and-user-assembly-references"></a>Разрешение системных ссылок и пользовательских ссылок на сборки

Целевой версии .NET, необходимо сначала установить подходящие ссылки на сборки. Вы можете скачать пакеты разработчика для разных версий .NET на [загрузки .NET](https://www.microsoft.com/net/download/windows) страницы.

Для проектов .NET Framework **добавить ссылку** диалоговое окно позволяет отключить системные сборки, не относящиеся к целевой версии .NET Framework, чтобы они добавляться не могут случайно в проект. (Системные сборки — это файлы *DLL*, включенные в версию платформы .NET Framework.) Ссылки, которые относятся к версии платформы, которая выше, чем целевая версия не будут разрешены, и невозможно добавить элементы управления, зависящие от таких ссылок. Если вы хотите активировать такую ссылку, измените целевую версию платформы .NET Framework для проекта на ту, которая содержит эту ссылку. Дополнительные сведения см. в разделе [Практическое руководство. Целевая версия framework](../ide/how-to-target-a-version-of-the-dotnet-framework.md).

Дополнительные сведения о ссылках на сборки см. в разделе [Разрешение сборок во время разработки](../msbuild/resolving-assemblies-at-design-time.md).

## <a name="enable-linq"></a>Включение LINQ

При нацеливании на .NET Framework 3.5 или более поздней версии ссылка на **System.Core** и импорт уровня проекта для <xref:System.Linq> (только в Visual Basic) добавляются автоматически. Если вы хотите использовать функции LINQ, нужно также включить параметр `Option Infer` (только в Visual Basic). Ссылка и импорт удаляются автоматически при изменении целевой версии на более раннюю версию .NET Framework. Дополнительные сведения см. в разделе [Работа с LINQ](/dotnet/csharp/tutorials/working-with-linq).
