# <a name="c6284"></a>C6284
предупреждение C6284: объект передан как параметр "% d", если в вызове функции \<function > требуется строка.

 Это предупреждение означает, что строка формата указывает строку, например, спецификацию `%s` для `printf` или `scanf`, но вместо этого был C++ передан объект.

 Эта ошибка может привести к неправильному выходу или сбою.

 Это сообщение часто сообщается из-за передачи C++ объекта, реализующего некоторый строковый тип, например `std::string`, `CComBSTR` или `bstr_t` в вызове C @no__t в стиле по 4. В зависимости от реализации C++ класса, то есть если определены правильные операторы приведения, объекты String можно использовать прозрачно, C++ когда требуются строки C. Однако, поскольку параметры @no__t функции в стиле-2 по сути не типизированы, преобразование в строку не происходит.

 В зависимости от объекта, может быть целесообразно вставить оператор `static_cast` к соответствующему строковому типу, например `char *` или `TCHAR``*`, или же вызвать функцию-член, которая возвращает строку, например, @no__t – 3, в экземплярах `std::string`.

## <a name="example"></a>Пример
 В следующем коде создается это предупреждение, поскольку `CComBSTR` передается функции `sprintf`:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 В следующем коде для исправления этого предупреждения используется статическое приведение:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```
