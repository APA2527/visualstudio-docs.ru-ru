# <a name="c26402dont_heap_allocate_movable_result"></a>C26402 DONT_HEAP_ALLOCATE_MOVABLE_RESULT

Чтобы избежать путаницы в том, владеет ли указатель объектом, функция, возвращающая перемещаемый объект, должна выделить его в стеке и вернуть его по значению вместо того, чтобы возвращать объект, выделенный в куче. Если требуется семантика указателя, то вместо необработанного указателя возвращается смарт-указатель. См. раздел [ C++ основные рекомендации R. 3](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#Rr-ptr). *Предупреждать, если функция возвращает объект, который был выделен в функции, но имеет конструктор Move. Рекомендуется возвращать значение вместо этого.*
