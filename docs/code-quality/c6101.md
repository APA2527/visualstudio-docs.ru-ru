# <a name="c6101"></a>C6101
C6101 предупреждения: Возврат неинициализированной памяти

 Успешный путь через функцию не устанавливает именованный параметр `_Out_`. Это сообщение создано на основе аннотации SAL, которая указывает, что анализируемая функция всегда завершается успешно. Функция, которая не возвращает индикации успешного или неуспешного выполнения, должна установить все параметры `_Out_`, так как анализатор предполагает, что параметр `_Out_` не инициализированы данные перед вызовом функции, и что функция задаст параметр таким образом, чтобы он больше не инициализируется. Если функция указывает на успешное завершение или сбой, то в случае сбоя параметр `_Out_` не следует задавать, а также можно выявить неинициализированные расположения и избежать их. В любом случае следует избежать чтения из неинициализированных расположений. Если функция иногда не затрагивает параметр `_Out_`, который впоследствии используется, то параметр должен быть инициализирован перед вызовом функции и помечаться в аннотации `_Inout_` или более явно `_Pre_null_` или `_Pre_satisfies_()`, если это уместно. "Частичный успех" можно обработать аннотацией `_When_`. Дополнительные сведения см. [в разделе Использование аннотаций SAL для сокращенияC++ числа дефектов C/Code](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).
