# <a name="c6312"></a>C6312
C6312 предупреждения: Возможный бесконечный цикл: использование константы EXCEPTION_CONTINUE_EXECUTION в выражении фильтра исключений для try-except

 Это предупреждение указывает на использование константы `EXCEPTION_CONTINUE_EXECUTION` (или другой константы, результатом которой является-1) в выражении фильтра структурированного обработчика исключений. Использование значения константы `EXCEPTION_CONTINUE_EXECUTION` может привести к бесконечному циклу. Например, если аппаратное обеспечение вызвало исключение, инструкция, которая вызвала исключение, будет перезапущена. Если адрес, вызвавший исключение, по-прежнему поврежден, возникает еще одно исключение, которое будет обрабатываться таким же образом. Это приводит к бесконечному циклу.

 Явный вызов `RaiseException` не приведет непосредственно к бесконечному циклу, но будет продолжать выполнение кода в защищенном блоке. Это может быть неожиданным, и может привести к бесконечному циклу, если `RaiseException` был использован, чтобы избежать разыменования недопустимого указателя.

 Как правило, `EXCEPTION_CONTINUE_EXECUTION` должен возвращаться только функцией, вызываемой в критерии фильтра, что может исправить либо указатель, который вызвал исключение, либо базовую память.

## <a name="example"></a>Пример
 Это предупреждение создается в следующем коде:

```
#include <excpt.h>
#include <stdio.h>
#include <windows.h>

void f (char *ptr)
{
  __try
  {
    // exception occurs if the caller passes null ptr
    // code...
   *ptr = '\0';
  }
  __except (EXCEPTION_CONTINUE_EXECUTION)
    // When EXCEPTION_CONTINUE_EXECUTION is used, the handler
    //  block of the structured exception handler is not executed.
  {
    puts("This block is never executed");
  }
}
```

 Чтобы исправить это предупреждение, используйте следующий код:

```
#include <excpt.h>
#include <stdio.h>
#include <windows.h>

void f (char *ptr)
{
  __try
  {
    // exception occurs if the caller passes null ptr
    // code...
    *ptr = '\0';
  }
  __except (GetExceptionCode()== EXCEPTION_ACCESS_VIOLATION ?
               EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH)
  {
    puts("Error Occurred");
  }
}
```
