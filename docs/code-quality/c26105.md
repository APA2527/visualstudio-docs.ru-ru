# <a name="c26105"></a>C26105
C26105 предупреждения: Нарушение порядка блокировки. Получение блокировки \<lock > с уровнем @no__t 1level > вызывает инверсию порядка.

 SAL Concurrency поддерживает *уровни блокировки*. Чтобы объявить уровень блокировки, который обозначается строковым литералом без двойных кавычек, используйте `_Create_lock_level_`. Можно настроить порядок приобретения между двумя уровнями блокировки с помощью аннотации `_Set_lock_level_order_(A,B)`, в которой указывается, что блокировки с уровнем `A` должны быть получены перед блокировками, имеющими уровень `B`. Чтобы установить иерархию порядка блокировки (частичный порядок между уровнями блокировки), используйте несколько аннотаций `_Set_lock_level_order_`. Чтобы связать блокировку с уровнем блокировки, используйте аннотацию `_Set_lock_level_` при объявлении блокировки. Предупреждение C26105 выдается при обнаружении нарушения порядка блокировки.

## <a name="example"></a>Пример
 В следующем примере создается предупреждение C26105, так как в функции `OrderInversion` используется инверсия порядка блокировки.

```

_Create_lock_level_(MutexLockLevel);
_Create_lock_level_(TunnelLockLevel);
_Create_lock_level_(ChannelLockLevel);
_Lock_level_order_(MutexLockLevel, TunnelLockLevel);
_Lock_level_order_(TunnelLockLevel, ChannelLockLevel);
_Has_lock_level_(MutexLockLevel) HANDLE gMutex;

struct Tunnel
{
    _Has_lock_level_(TunnelLockLevel) CRITICAL_SECTION cs;
};

struct Channel
{
    _Has_lock_level_(ChannelLockLevel) CRITICAL_SECTION cs;
};

void OrderInversion(Channel* pChannel, Tunnel* pTunnel)
{
    EnterCriticalSection(&pChannel->cs);
    // Warning C26105
    WaitForSingleObject(gMutex, INFINITE);
    EnterCriticalSection(&pTunnel->cs);
    LeaveCriticalSection(&pTunnel->cs);
    LeaveCriticalSection(&pChannel->cs);
}
```
