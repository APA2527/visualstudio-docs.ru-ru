# <a name="c6225"></a>C6225

C6225 предупреждения: Неявное приведение между семантически различными целочисленными типами: присваивание значения 1 или TRUE значению HRESULT. Вместо этого рекомендуется использовать S_FALSE.

Это предупреждение означает, что `HRESULT` присваивается или инициализируется со значением явного 1. Логические типы указывают на успешность с ненулевым значением. успешно (`S_OK`) в `HRESULT` обозначается значением 0. Это предупреждение часто бывает вызвано случайной путаницой типов Boolean и `HRESULT`. Чтобы обозначить успешность, следует использовать символьную константу `S_OK`.

## <a name="example"></a>Пример

В следующем коде назначение `HRESULT` создает это предупреждение:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    // code ...
    hr = S_OK;
  }
  else
  {
    // code ...
    hr = 1;
  }
}
```

Чтобы исправить это предупреждение, используйте следующий код:

```cpp
VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
    // code ...
  }
  else
  {
    hr = S_FALSE;
    // code ...
  }
}
```

Для этого предупреждения тип `SCODE` эквивалентен `HRESULT`.

Чтобы указать на ошибку, следует использовать значение `E_FAIL` или другую константу.
