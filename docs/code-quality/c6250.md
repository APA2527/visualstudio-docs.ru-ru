# <a name="c6250"></a>C6250
C6250 предупреждения: Вызов \<function > VirtualFree без флага MEM_RELEASE может освободить память, но не имеет дескрипторов адресов (VAD); приводит к утечкам адресного пространства

 Это предупреждение означает, что вызов `VirtualFree` без флага `MEM_RELEASE` только отменяет фиксацию страниц и не освобождает их. Для отмены фиксации и освобождения страниц используйте флаг `MEM_RELEASE` в вызове `VirtualFree`. Если все страницы в регионе зафиксированы, функция сначала отменяет фиксацию, а затем освобождает их. После выполнения этой операции страницы находятся в свободном состоянии. При указании этого флага значение `dwSize` должно быть равно нулю, а `lpAddress` должен указывать на базовый адрес, возвращенный функцией `VirtualAlloc` при резервировании региона. Функция завершается ошибкой, если одно из этих условий не выполнено.

 Это предупреждение можно проигнорировать, если в коде впоследствии освобождается адресное пространство путем вызова `VirtualFree` с флагом `MEM_RELEASE`.

 Дополнительные сведения см. в разделе [VirtualAlloc](http://go.microsoft.com/fwlink/?LinkId=181201) и [VirtualFree](http://go.microsoft.com/fwlink/?LinkId=181202).

 Использование VirtualAlloc и VirtualFree имеет много ловушек с точки зрения утечек памяти и исключений. Чтобы избежать такого рода утечек и проблем с исключениями, используйте механизмы, предоставляемые C++ стандартной библиотекой шаблонов (STL). К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)и [vector](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [смарт-указатели](/cpp/cpp/smart-pointers-modern-cpp) и [ C++ Стандартная библиотека](/cpp/standard-library/cpp-standard-library-reference).

## <a name="example"></a>Пример
 Это предупреждение создается в следующем примере кода:

```
#include <windows.h>
#include <stdio.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( )
{
  LPVOID lpvBase;            // base address of the test memory
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo(&sSysInfo);
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS);
  //
  // code to access memory
  // ...

  if (lpvBase != NULL)
  {
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages
      {
        puts ("MEM_DECOMMIT Succeeded");
      }
      else
      {
       puts("MEM_DECOMMIT failed");
      }
  }
  else
  {
    puts("lpvBase == NULL");
  }
}
```

 Чтобы исправить это предупреждение, используйте следующий пример кода:

```
#include <windows.h>
#include <stdio.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( )
{
  LPVOID lpvBase;            // base address of the test memory
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo(&sSysInfo);
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS);
  //
  // code to access memory
  // ...

  if (lpvBase != NULL)
  {
   if (VirtualFree(lpvBase, 0,MEM_RELEASE )) // decommit & release
   {
     // code ...
   }
   else
   {
     puts("MEM_RELEASE failed");
   }
  }
  else
  {
    puts("lpvBase == Null ");
    // code...
  }
}
```
