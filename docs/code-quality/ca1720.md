# <a name="ca1720-identifiers-should-not-contain-type-names"></a>CA1720: идентификаторы не должны содержать имен типов

|||
|-|-|
|TypeName|IdentifiersShouldNotContainTypeNames|
|CheckId|CA1720|
|Категория|Microsoft.Naming|
|Критическое изменение|Критическое|

## <a name="cause"></a>Причина:

Имя параметра в члене содержит имя типа данных.

-или-

Имя участника содержит имя типа данных, специфичное для языка.

По умолчанию это правило смотрит только на внешне видимых членов, но это [настраивается.](#configurability)

## <a name="rule-description"></a>Описание правила

Названия параметров и членов лучше использовать для передачи их значения, чем для описания их типа, который, как ожидается, будет предоставляться с помощью инструментов разработки. Для имен членов, если необходимо использовать имя типа данных, используйте имя, не зависящем от языка, а не конкретное. Например, вместо имени `int`типа C', используйте имя типа `Int32`данных, независимого от языка, .

Каждый дискретный токен в имени параметра или участника проверяется в соответствии со следующими именами типов данных, относясь к конкретному языку, в бесчувственном виде:

- Bool
- WChar
- Int8
- UInt8
- Short
- UShort
- Int
- Uint
- Целое число
- UInteger
- Long
- ULong
- Без знака
- Со знаком
- Float
- Float32
- Float64

Кроме того, имена параметра также проверяются в отношении следующих независях от языка имен типов данных в случае бесчувственным образом:

- Объект
- Логическое
- CHAR
- Строка
- SByte
- Byte
- UByte
- Int16
- UInt16
- Int32
- UInt32
- Int64
- UInt64
- IntPtr
- Ptr
- Указатель
- UInptr
- UPtr
- UPointer
- Один
- Double
- Decimal
- Guid

## <a name="how-to-fix-violations"></a>Устранение нарушений

**При выстреле по параметру:**

Замените идентификатор типа данных в названии параметра либо термином, который лучше описывает его значение, либо более общим термином, таким как "значение".

**При увольнении по участнику:**

Замените идентификатор типа данных, относясь к языку, в имени участника термином, который лучше описывает его значение, эквивалентом, независимым от языка, или более общим термином, например «значение».

## <a name="when-to-suppress-warnings"></a>Когда подавлять предупреждения

Возможно, уместно использовать параметр на основе типов и имена членов. Однако для новой разработки не происходит никаких известных сценариев, в которых следует пресекать предупреждение из этого правила. Для библиотек, которые ранее отправлялись, возможно, придется пресекать предупреждение из этого правила.

## <a name="configurability"></a>Настраиваемость

Если вы работаете с помощью [анализаторов FxCop](install-fxcop-analyzers.md) (а не с помощью устаревшего анализа), можно настроить часть базы кода для выполнения этого правила, исходя из их доступности. Например, чтобы указать, что правило должно работать только на непубличной поверхности API, добавьте следующую пару значений ключа в файл .editorconfig в проекте:

```ini
dotnet_code_quality.ca1720.api_surface = private, internal
```

Эту опцию можно настроить только для этого правила, для всех правил или для всех правил этой категории (Именование). Для получения дополнительной информации [см.](configure-fxcop-analyzers.md)
