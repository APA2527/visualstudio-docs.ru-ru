# <a name="c6279"></a>C6279
Предупреждение C6279: \<переменная > выделена помощи скалярного оператора new, удаляется оператором для массивов delete]

Это предупреждение появляется только в C++ кода и указывает, что вызывающая функция выделила памяти скалярным **новый** оператор, но освободила ее с массивом **delete []** оператор. Если память выделяется скалярной **новый**, его следует освобождать при помощи скалярного оператора **удалить**.

Существует по крайней мере трех причин, это может привести к проблемам.

- Конструкторы для отдельных объектов в массиве не вызываются, несмотря на то, что деструкторы — это.

- Если глобальный (или определенного класса) **оператор new** и **оператор delete** не совместимы с **оператор new []** и **оператор delete []**, могут возникнуть непредвиденные результаты.

Точное последствия этой неполадки предсказать их трудно. Это может привести к непредвиденному поведению или происходит аварийный сбой из-за использования неинициализированной памяти, поскольку конструкторы не вызываются. Или может вызвать выделение памяти и сбои в ситуациях, где были переопределены операторы. В редких случаях несоответствие может оказаться незначительной. Средство анализа не различают в настоящее время эти ситуации.

## <a name="example"></a>Пример

В следующем коде создается данное предупреждение:

```cpp
class A
{
  // members
};

void f ( )
{
  A *pA = new A;
  //code ...
  delete[] pA;
}
```

Чтобы устранить это предупреждение, используйте следующий код:

```cpp
void f( )
{
  A *pA = new A;
  //code ...
  delete pA;
}
```

Чтобы избежать подобные проблемы выделения, используйте механизмы, предоставляемые библиотекой стандартных шаблонов (STL) C++. К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class), и [вектор](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [интеллектуальные указатели](/cpp/cpp/smart-pointers-modern-cpp) и [стандартной библиотеки C++](/cpp/standard-library/cpp-standard-library-reference).
