# <a name="how-to-enable-and-disable-automatic-code-analysis-for-managed-code"></a>Практическое руководство. Включение и отключение автоматического анализа кода для управляемого кода

Можно настроить (статический) анализ кода для выполнения после каждой сборки проекта управляемого кода. Можно задать различные свойства анализа кода для каждой конфигурации сборки, например Отладка и выпуск.

Эта статья относится только к прежним анализам, а не к интерактивному анализу кода с помощью [анализаторов кода](roslyn-analyzers-overview.md).

## <a name="to-enable-or-disable-automatic-code-analysis"></a>Включение или отключение автоматического анализа кода

1. В **Обозреватель решений**щелкните правой кнопкой мыши проект и выберите пункт **свойства**.

1. В диалоговом окне Свойства проекта перейдите на вкладку **анализ кода** .

   > [!TIP]
   > В новых типах проектов, таких как .NET Core и .NET Standard приложений, отсутствует вкладка **анализ кода** . Анализ прежних версий для этих типов проектов недоступен, но вы можете получить динамический анализ кода с помощью [анализаторов кода на основе .NET Compiler Platform](roslyn-analyzers-overview.md). Чтобы отключить предупреждения от анализаторов кода, ознакомьтесь с примечанием в конце этой статьи.

1. Укажите тип сборки в поле **Конфигурация** и Целевая платформа на **платформе**.

1. Чтобы включить или отключить автоматический анализ кода, установите или снимите флажок **Включить анализ кода при сборке** .

> [!NOTE]
> Флажок **Включить анализ кода при сборке** влияет только на прежний анализ. Он не влияет на [анализаторы кода на основе .NET Compiler Platform](roslyn-analyzers-overview.md), которые всегда выполняются при сборке, если они были установлены в качестве пакета NuGet. Если вы хотите удалить ошибки анализатора из **Список ошибок**, можно отключить все текущие нарушения, выбрав **анализировать** > **выполнение анализа кода и подавлять активные проблемы** в строке меню. Дополнительные сведения см. в разделе [подавлять нарушения](use-roslyn-analyzers.md#suppress-violations).
