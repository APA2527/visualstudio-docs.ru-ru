# <a name="c6217"></a>C6217

C6217 предупреждения: Неявное приведение между семантически различными целочисленными типами: проверка HRESULT с "not". Рассмотрите возможность использования макроса " [успешно](/windows/desktop/api/winerror/nf-winerror-succeeded) " или " [сбой](/windows/desktop/api/winerror/nf-winerror-failed) ".

Это предупреждение означает, что `HRESULT` тестируется с помощью оператора NOT (`!`). Значение Success (`S_OK`) в `HRESULT` обозначается значением 0. Однако значение 0 указывает на сбой логического типа. Тестирование `HRESULT` с оператором NOT (`!`) для определения того, какой блок кода следует запустить, может привести к неправильному пути кода. Это приводит к нежелательным результатам.

## <a name="example"></a>Пример

В следующем коде создается это предупреждение, поскольку оператор NOT используется для определения успешности или сбоя значения `HRESULT`. В этом случае выполняется неправильный путь к коду, так как `( !hr )` выполняет код ошибки:

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (!hr)
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

Чтобы устранить это предупреждение, в следующем коде для поиска ошибки используется макрос `FAILED`:

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (FAILED(hr))
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

Для этого предупреждения тип `SCODE` эквивалентен `HRESULT`.

Типичное значение успеха HRESULT (`S_OK`) равно `false` при проверке в качестве логического значения.

Чтобы убедиться, что `HRESULT` является успешным, используйте вместо него макрос `SUCCEEDED`.
