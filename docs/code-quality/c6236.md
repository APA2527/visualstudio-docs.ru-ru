# <a name="c6236"></a>C6236
Warning C6236: (\<expression > &#124; &#124; \<non-нуль Constant >) всегда равно ненулевой константе

 Это предупреждение означает, что в правой части логической операции или, которая выполняется в контексте теста, обнаружено ненулевое значение константы, отличное от единицы. Логически это означает, что тест является избыточным и может быть безопасно удален. Кроме того, предполагается, что программист может использовать другой оператор, например оператор равенства (`==`), побитовый и (`&`) или битовый-XOR (`^`) для проверки конкретного значения или флага.

 Это предупреждение не создается для общей идиомы, если ненулевая константа равна 1, так как она используется для выборочного включения путей кода во время компиляции. Однако предупреждение создается, если ненулевая константа формируется выражением, результатом которого является 1, например 1 + 0.

## <a name="example"></a>Пример
 Этот код показывает, как может выглядеть предупреждение C6236. Поскольку `INPUT_TYPE` не равен 0, выражение `n || INPUT_TYPE` всегда имеет ненулевое значение, а предложение @no__t 2 не выполняется. Однако `INPUT_TYPE` является константой со значением, отличным от единицы, что предполагает, что оно предназначено для сравнения:

```
#define INPUT_TYPE 2
#include <stdio.h>

void f( int n )
{
   if ( n || INPUT_TYPE ) // analysis warning C6236
   {
      puts( "Always gets here" );
   }
   else
   {
      puts( "Never enters here" );
   }
}
```

 В следующем коде вместо этого используется оператор побитового и (`&`), чтобы проверить, установлен ли бит `INPUT_TYPE` входного параметра `n`:

```
#define INPUT_TYPE 2
#include <stdio.h>

void f( int n )
{
   if ( n & INPUT_TYPE ) // no warning
   {
      puts( "Bitwise-AND comparison is true" );
   }
   else
   {
      puts( "Bitwise-AND comparison is false" );
   }
}
```
