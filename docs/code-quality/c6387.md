# <a name="c6387"></a>C6387
Warning C6387: \<argument > может быть \<value >: это не соответствует спецификации функции \<function name >: Строки: x, y

 Это предупреждение возникает, если параметр функции с заметками передается непредвиденным значением. Например, это предупреждение создается при передаче потенциального значения NULL в параметр, помеченный заметкой `_In_`.

## <a name="example"></a>Пример
 В следующем коде создается это предупреждение, поскольку параметр null передается в `f(char *)`:

```cpp

#include <sal.h>

_Post_ _Null_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh); // Warning C6387
}
```

 Чтобы исправить это предупреждение, используйте следующий код:

```cpp

#include <sal.h>

_Post_ _Notnull_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh);
}
```
