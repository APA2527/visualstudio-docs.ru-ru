# <a name="c6239"></a>C6239
предупреждение C6239: (\<non-Zero константа > & & \<expression) всегда вычисляется в результате >-2expression @no__t. Вы планировали использовать оператор "побитовое и"?

 Это предупреждение означает, что в левой части операции логического и, которая выполняется в контексте теста, обнаружено ненулевое значение константы, отличное от единицы. Например, выражение `( 2 && n )` уменьшается до `(!!n)`, что является логическим значением `n`.

 Это предупреждение обычно указывает на попытку проверки битовой маски, в которой должен использоваться оператор побитового и (`&`), и не создается, если ненулевая константа имеет значение 1 из-за его использования для выборочного выбора путей кода.

## <a name="example"></a>Пример
 Это предупреждение создается в следующем коде:

```
#include <stdio.h>
#define INPUT_TYPE 2
void f( int n )
{
   if(INPUT_TYPE && n) // warning 6239
   {
      puts("boolean value of n is true");
   }
   else
   {
      puts("boolean value of n is false");
   }
}
```

 Чтобы исправить это предупреждение, используйте оператор побитового и (`&`), как показано в следующем коде:

```
#include <stdio.h>
#define INPUT_TYPE 2
void f( int n )
{
   if( ( INPUT_TYPE & n ) )
   {
      puts("bitmask true");
   }
   else
   {
      puts("bitmask false");
   }
}
```
