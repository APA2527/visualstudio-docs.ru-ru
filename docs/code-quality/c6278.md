# <a name="c6278"></a>C6278
Warning C6278: \<variable > выделяется массивом new [], но удаляется с скалярным удалением. Деструкторы не будут вызываться

Это предупреждение появляется только в C++ коде и указывает, что вызывающая функция имеет неоднородное выделение памяти с помощью оператора **new []** массива, но освобождает его с помощью скалярного оператора **Delete** . Это поведение не определено в соответствии со C++ стандартом и реализацией Microsoft C++ Visual. По крайней мере три причины могут вызвать проблемы.

- Вызываются конструкторы для отдельных объектов в массиве, но деструкторы не вызываются.

- Если глобальные или специфичные для класса **операторы new** и **Delete** не совместимы с **оператором new []** и **operator delete []** , то, скорее всего, будут возникать непредвиденные результаты.

- Всегда рискованно полагаться на неопределенное поведение.

Последствия этого дефекта сложно предсказать. Это может привести к утечке классов с деструкторами, которые выполняют отмену выделения памяти. несоответствие поведения для классов с деструкторами, которые выполняют некоторую семантическую операцию. или повреждения памяти и сбои при переопределении операторов. В других случаях несоответствие может быть неважным в зависимости от реализации компилятора и его библиотек. Средство анализа не всегда различает эти ситуации.

Если память выделяется с помощью массива **new []** , то, как правило, она должна быть освобождена с помощью **Delete массива []** .

## <a name="example"></a>Пример

Это предупреждение создается в следующем примере кода:

```cpp
class A
{
  // members
};

void f( )
{
  A *pA = new A[5];
  // code ...
  delete pA;
}
```

Чтобы исправить это предупреждение, используйте следующий пример кода:

```cpp
void f( )
{
  A *pA = new A[5];
  // code ...
  delete[] pA;
}
```

Если базовый объект в массиве является примитивным типом, например `int`, `float`, `enum` или указателем, то не будут вызываться деструкторы. В этих случаях вместо этого выводится предупреждение [C6283](../code-quality/c6283.md) .

Использование новых и удаляемых элементов имеет много ловушек с точки зрения утечек памяти и исключений. Чтобы избежать такого рода утечек и проблем с исключениями, используйте механизмы, предоставляемые C++ стандартной библиотекой шаблонов (STL). К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)и [vector](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [смарт-указатели](/cpp/cpp/smart-pointers-modern-cpp) и [ C++ Стандартная библиотека](/cpp/standard-library/cpp-standard-library-reference).
