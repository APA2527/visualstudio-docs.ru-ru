# <a name="c6226"></a>C6226

C6226 предупреждения: Неявное приведение между семантически различными целочисленными типами: присваивание значения-1 значению HRESULT. Вместо этого рекомендуется использовать E_FAIL.

Это предупреждение означает, что `HRESULT` назначено или инициализировано явное значение-1. Это предупреждение часто бывает вызвано случайной путаницой типов integer и `HRESULT`. Чтобы обозначить успешность, используйте вместо нее символьную константу `S_OK`. Чтобы указать на ошибку, используйте символьные константы, начинающиеся с*константы*E_, например `E_FAIL`.

Дополнительные сведения см. в статье об [успешно выполненных](/windows/desktop/api/winerror/nf-winerror-succeeded) и [неудачных](/windows/desktop/api/winerror/nf-winerror-failed) макросах.

## <a name="example"></a>Пример

Это предупреждение создается в следующем коде:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = -1;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Чтобы исправить это предупреждение, используйте следующий код:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = E_FAIL;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Для этого предупреждения тип `SCODE` рассматривается как `HRESULT`.

Использование `malloc` и `free` (и связанных с ним интерфейсов API динамической памяти) связано с множеством ловушек, приводящих к утечкам памяти и исключениям. Чтобы избежать подобных утечек и проблем с исключениями, используйте классы указателей и контейнеров, предоставляемые C++ стандартной библиотекой. К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)и [vector](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [смарт-указатели](/cpp/cpp/smart-pointers-modern-cpp) и [ C++ Стандартная библиотека](/cpp/standard-library/cpp-standard-library-reference).
