# <a name="c6230"></a>C6230

предупреждение C6230: неявное приведение между семантически различными целочисленными типами: использование HRESULT в логическом контексте

Это предупреждение означает, что в контексте, где ожидается логический результат, например в операторе `if`, используется тип `HRESULT`. Этот тест, скорее всего, приведет к неправильным результатам. Например, стандартное значение успеха для `HRESULT` (`S_OK`) равно false, когда оно тестируется как логическое.

## <a name="example"></a>Пример

Это предупреждение создается в следующем коде:

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (hr)
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

В большинстве случаев для проверки значения `HRESULT` следует использовать макрос " [успешно](/windows/desktop/api/winerror/nf-winerror-succeeded) " или " [сбой](/windows/desktop/api/winerror/nf-winerror-failed) ". Чтобы исправить это предупреждение, используйте следующий код:

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

Для этого предупреждения тип `SCODE` рассматривается как `HRESULT`.

Использование `malloc` и `free` (и связанных с ним интерфейсов API динамической памяти) связано с множеством ловушек, приводящих к утечкам памяти и исключениям. Чтобы избежать подобных утечек и проблем с исключениями, используйте классы указателей и контейнеров, предоставляемые C++ стандартной библиотекой. К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)и [vector](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [смарт-указатели](/cpp/cpp/smart-pointers-modern-cpp) и [ C++ Стандартная библиотека](/cpp/standard-library/cpp-standard-library-reference).
