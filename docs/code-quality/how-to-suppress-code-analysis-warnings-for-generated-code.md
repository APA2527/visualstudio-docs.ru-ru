# <a name="how-to-suppress-code-analysis-warnings-for-generated-code"></a>Практическое руководство. Отключение предупреждений анализа для генерируемого кода

Созданный код включает код, который добавляется в проект с Компиляторы управляемого кода или сторонние средства. Может потребоваться нарушения правил, обнаруживаемых при анализе в созданном коде см. в разделе. Тем не менее так как не может просматривать и обслуживать код, который содержит нарушения, может не потребоваться см. в разделе, их.

**Подавлять результаты из созданного кода** флажок на странице свойств анализа кода проекта позволяет выбрать, следует ли отображать предупреждений анализа кода, созданного с помощью средства стороннего кода.

> [!NOTE]
> Этот параметр не отключает ошибки и предупреждения из созданного кода, если ошибки и предупреждения появляются в формах и шаблонах. Вы можете просматривать и обслуживать исходный код для формы или шаблона.

### <a name="to-suppress-warnings-for-generated-code-in-a-project"></a>Чтобы отключить предупреждения для сформированного кода в проекте

1. Щелкните правой кнопкой мыши проект в **обозревателе решений** и нажмите кнопку **свойства**.

2. Выберите **анализа кода** вкладки.

3. Выберите **подавлять результаты из созданного кода** "флажок".

> [!NOTE]
> Можно только отключить предупреждения из статического анализа кода. В настоящее время нельзя было подавить предупреждения при анализе кода из [анализаторы](roslyn-analyzers-overview.md).
