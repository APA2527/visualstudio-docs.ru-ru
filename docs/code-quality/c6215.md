# <a name="c6215"></a>C6215
предупреждение C6215: приведение целочисленных типов с разной семантикой: тип Boolean к типу HRESULT

 Это предупреждение означает, что логическое значение приводится к `HRESULT`. Логические типы указывают на успешность по ненулевому значению, тогда как Success (`S_OK`) в `HRESULT` обозначается значением 0. Приведение логического типа к `HRESULT` и его использование в тестовом выражении приведет к неправильному результату.

 Это предупреждение часто возникает, когда в качестве аргумента для `SUCCEEDED` или `FAILED` используется логическое значение, которое явным образом приводит свои аргументы к `HRESULT`.

## <a name="example"></a>Пример
 Это предупреждение создается в следующем коде:

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

void f( REFGUID riid1, REFGUID riid2 )
{
  if (SUCCEEDED( IsEqual( riid1, riid2 ) )) //warning 6215
  {
    // success code ...
  }
  else
  {
    // failure code ...
  }
}
```

 Как правило, макросы `SUCCEEDED` или `FAILED` следует применять только к `HRESULT`.

 Чтобы исправить это предупреждение, используйте следующий код:

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

void f( REFGUID riid1, REFGUID riid2 )
{
  if (IsEqual( riid1, riid2 ) == TRUE)
  {
    // code for riid1 == riid2
  }
  else
  {
    // code for riid1 != riid2
  }
}
```
