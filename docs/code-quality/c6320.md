# <a name="c6320"></a>C6320
предупреждение C6320: выражение фильтра исключений является константой EXCEPTION_EXECUTE_HANDLER. Это может маскировать исключения, которые не предназначены для обработки

 Это предупреждение указывает на побочный результат использования константы EXCEPTION_EXECUTE_HANDLER в блоке __except. В этом случае инструкция в блоке @no__t -0 _except всегда будет выполняться для устранения исключения, включая исключения, которые не нужно было выполнять в определенной функции. Рекомендуется проверять исключение перед его обработкой.

## <a name="example"></a>Пример
 В следующем коде создается это предупреждение, поскольку блок __except попытается выполнить обработку исключений всех типов:

```
#include <stdio.h>
#include <excpt.h>

void f(int *p)
{
   __try
   {
      puts("in try");
      *p = 13;  // might cause access violation exception
      // code ...
   }
   __except(EXCEPTION_EXECUTE_HANDLER) // warning
   {
      puts("in except");
      // code ...
   }
}
```

 Чтобы исправить это предупреждение, используйте `GetExceptionCode` для проверки конкретного исключения перед его обработкой, как показано в следующем коде:

```
#include <stdio.h>
#include <windows.h>
#include <excpt.h>

void f(int *p)
{
   __try
   {
      puts("in try");
      *p = 13;    // might cause access violation exception
      // code ...
   }
   __except(GetExceptionCode()==EXCEPTION_ACCESS_VIOLATION ?
               EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH)
   {
      puts("in except");
      // code ...
   }
}
```
