# <a name="walkthrough-publish-a-visual-studio-extension"></a>Пошаговая прогулка: Опубликуйте расширение Визуальной студии

Это пошаговое решение показывает, как опубликовать расширение Visual Studio на Visual Studio Marketplace. При добавлении расширения в Marketplace разработчики могут использовать **расширения и обновления** для просмотра новых и обновленных расширений.

## <a name="prerequisites"></a>Предварительные требования

 Для выполнения этого пошагового руководства необходимо установить пакет SDK для Visual Studio. Для получения дополнительной информации, [см.](../extensibility/installing-the-visual-studio-sdk.md)

## <a name="create-a-visual-studio-extension"></a>Создание расширения Visual Studio

В этой статье используется расширение VSPackage по умолчанию, но шаги действительны для любого вида расширения.

1. Создайте VSPackage в `TestPublish` названном с именем, который имеет команду меню. Для получения дополнительной информации, смотрите [Создать свое первое расширение: Привет мир](../extensibility/extensibility-hello-world.md).

## <a name="package-your-extension"></a>Упаковка расширения

1. Обновление расширения *.vsixmanifest* с правильной информацией о названии продукта, авторе и версии.

   ![обновление расширения vsixmanifest](media/update-extension-vsixmanifest.png)

2. Создайте расширение в режиме **выпуска.** Теперь ваше расширение упаковано в качестве VSIX в папку «Бин-Релиз».

3. Вы можете дважды щелкнуть VSIX для проверки установки.

## <a name="test-the-extension"></a>Проверьте расширение

 Перед тем, как распределить расширение, создайте и протестируйте его, чтобы убедиться, что оно правильно установлено в экспериментальном экземпляре Visual Studio.

1. В Visual Studio начните отладку, чтобы открыть экспериментальный экземпляр Visual Studio.

2. В экспериментальном примере перейдите в меню **Инструментов** и нажмите **расширения и обновления.** Расширение TestPublish должно отображаться в центральном стене и быть включенным.

3. В меню **«Инструменты»** убедитесь, что вы видите тестовую команду.

## <a name="publish-the-extension-to-the-visual-studio-marketplace"></a>Опубликовать расширение на рынок визуальной студии

1. Убедитесь, что вы создали версию релиза вашего расширения и что она актуальна.

2. В веб-браузере откройте веб-сайт [Visual Studio Marketplace.](https://marketplace.visualstudio.com/vs)

3. В правом верхнем углу нажмите **Войти в**.

4. Войдите с помощью своей учетной записи Майкрософт. Если у вас нет учетной записи Майкрософт, на этом этапе можно создать ее.

5. Нажмите **Опубликовать расширения**.  Эта опция перемещает вас на страницу управления для всех ваших расширений. Если у вас нет учетной записи издателя, вам предлагается создать ее в это время.

   ![Загрузка на Marketplace](media/upload-to-marketplace.png)

6. Выберите издателя, который вы хотите использовать для загрузки расширения. Вы можете изменить издателей, нажав на имена издателя, перечисленные слева. Нажмите на **новое расширение** и выберите **Visual Studio**.

7. В **1: Загрузите расширение,** вы можете загрузить файл VSIX непосредственно в Visual Studio Marketplace или просто добавить ссылку на свой собственный веб-сайт. В этом примере загружается расширение *TestPublish.vsix.* Перетащите и отбросьте расширение или используйте ссылку **на кнопку** для просмотра файла. Найдите свое расширение в папке «Бин-Release» проекта.  Нажмите кнопку **Продолжить**.

8. В **2: Предоставьте детали расширения,** некоторые поля автоматически заселены от *файла source.extension.vsixmanifest* из вашего расширения. Найти более подробную информацию о каждом из них ниже:

    * **Внутреннее имя** используется в URL-адресе подробной страницы расширения. Например, публикация расширения под названием издателя "myname" и указание внутреннего имени, которое будет "мое расширение" приводит к URL", например, "market.visualstudio\.com/items?itemName-myname.myextension" для страницы детали расширения.

    * **Отобразить имя** вашего расширения. Это имя автоматически заселены из *файла source.extension.vsixmanifest.*

    * Номер **версии** загружаемого расширения. Эта версия автоматически засеелена из файла *source.extension.vsixmanifest.*

    * **VSIX ID** является уникальным идентификатором, который Visual Studio использует для вашего расширения. Этот идентификатор необходим, если вы хотите, чтобы ваше расширение автоматически обновлялось. Этот идентификатор автоматически заседает из файла *source.extension.vsixmanifest.*

    * **Логотип,** который используется для расширения. Этот логотип автоматически заселен из *файла source.extension.vsixmanifest,* если это предусмотрено.

    * **Краткое описание** того, что делает ваше расширение. Это описание автоматически заселены из *файла source.extension.vsixmanifest.*

    * **Обзор** является хорошим местом для включения скриншотов и подробной информации о том, что делает ваше расширение.

    * **Поддерживаемые версии Visual Studio** позволяют выбирать, над какими версиями Visual Studio будет работать ваше расширение. Ваше расширение устанавливается только для этих версий.

    * «Поддерживаемое издание Visual Studio позволяет вам выбирать, какие издания Visual Studio, над которым будет работать ваше расширение. Ваше расширение устанавливается только на эти издания.

    * **Тип**. Наиболее распространенным типом расширений являются **инструменты.**

    * **Категории**. Возьмите до трех, которые лучше всего подходят для вашего расширения.

    * **Теги являются ключевыми словами,** которые помогают пользователям найти ваше расширение. Теги могут помочь повысить актуальность поиска ваших расширений в Marketplace.

    * **Категория ценообразования** — это стоимость вашего расширения.

    * **Репозиторий исходного кода** позволяет совместно использовать ссылку на исходный код с сообществом.

    * **Разрешить&А для вашего расширения** позволяет пользователям оставлять вопросы на странице входа расширения.

9. Нажмите **Сохранить & загрузки**. Эта опция возвращает вас на страницу управления издателем. Ваше расширение еще не опубликовано. Чтобы опубликовать расширение, нажмите правой кнопкой мыши на расширение и выберите **Make Public**. Вы можете просмотреть, как будет выглядеть ваше расширение на Marketplace, выбрав **расширение view.** Для получения номеров, нажмите на **Отчеты**. Чтобы внести изменения в расширение, нажмите на **Edit**.

   ![Меню входа расширения](media/extension-entry-menu.png)

10. После нажатия **Сделать публично,** ваше расширение в настоящее время общественности. Поиск Visual Studio Marketplace для вашего расширения.

## <a name="add-additional-users-to-manage-your-publisher-account"></a>Добавление дополнительных пользователей для управления учетной записью издателя

Marketplace поддерживает предоставление дополнительных разрешений пользователям для доступа к учетной записи издателя и управления ими.

1. Перейдите к учетной записи издателя, в который вы хотите добавить дополнительных пользователей.

2. Выберите **участников** и нажмите на **Добавить**.

   ![Добавление дополнительного пользователя](media/add-users.png)

3. Затем вы можете указать адрес электронной почты пользователя, который вы хотите добавить, и предоставить правильный уровень доступа в **рамках Select a role.**  Вы можете выбрать один из следующих параметров:

   * **Создатель**: Пользователь может публиковать расширения, но не может просматривать или управлять расширениями, опубликованными другими пользователями.

   * **Reader**: Пользователь может просматривать расширения, но не может публиковать или управлять расширениями.

   * **Автор**: Пользователь может публиковать и управлять расширениями, но не может отсеивать настройки издателя или управлять доступом.

   * **Владелец**: Пользователь может публиковать и управлять расширениями, отсеивать настройки издателя и управлять доступом.

## <a name="install-the-extension-from-the-visual-studio-marketplace"></a>Установите расширение с рынка визуальной студии

Теперь, когда расширение опубликовано, установите его в Visual Studio и протестируйте его там.

1. В Visual Studio, в меню **Инструментов,** нажмите **расширения и обновления**.

2. Нажмите **онлайн,** а затем поиск **для TestPublish**.

3. Щелкните элемент **Загрузить**. Затем планируется установить расширение.

4. Для завершения установки закройте все экземпляры Visual Studio.

## <a name="remove-the-extension"></a>Удаление расширения

Вы можете удалить расширение из Visual Studio Marketplace и с вашего компьютера.

### <a name="to-remove-the-extension-from-the-visual-studio-marketplace"></a>Удаление расширения из визуальной студии Marketplace

1. Откройте веб-сайт [Visual Studio Marketplace.](https://marketplace.visualstudio.com/vs)

2. В верхнем правом углу нажмите **Кнопка Публикация** расширений. Выберите издателя, который вы использовали для публикации **TestPublish**. Появляется список для **TestPublish.**

3. Нажмите на вход в расширение и нажмите **Удалить**. Вас просят подтвердить, хотите ли вы удалить расширение. Нажмите кнопку **ОК**.

### <a name="to-remove-the-extension-from-your-computer"></a>Удаление расширения с компьютера

1. В Visual Studio, в меню **Инструментов,** нажмите **расширения и обновления**.

2. Выберите **TestPublish,** а затем нажмите **Uninstall**. Затем продление планируется удалить.

3. Чтобы завершить неустановку, закройте все экземпляры Visual Studio.
