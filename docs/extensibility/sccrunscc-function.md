# <a name="sccrunscc-function"></a>Функция SccRunScc
Эта функция вызывает инструмент управления исходным управлением.

## <a name="syntax"></a>Синтаксис

```cpp
SCCRTN SccRunScc(
   LPVOID  pvContext,
   HWND    hWnd,
   LONG    nFiles,
   LPCSTR* lpFileNames
);
```

#### <a name="parameters"></a>Параметры
 pvКонтекст

(в) Структура контекста управления исходным элементом.

 Hwnd

(в) Ручка к окну IDE, которую плагин управления исходным элементом может использоваться в качестве родительского элемента для любых диалоговых коробок, которые она предоставляет.

 nФайлы

(в) Количество файлов, указанных в массиве. `lpFileNames`

 lpFileNames

(в) Массив выбранных имен файлов.

## <a name="return-value"></a>Возвращаемое значение
 Ожидается, что внедрение этой функции элемента управления исходным элементом вернет одно из следующих значений:

|Значение|Описание|
|-----------|-----------------|
|SCC_OK|Инструмент управления исходным управлением был успешно вызван.|
|SCC_I_OPERATIONCANCELED|Операция была отменена.|
|SCC_E_INITIALIZEFAILED|Не удалось инициализировать систему управления исходом.|
|SCC_E_ACCESSFAILURE|Возникла проблема с доступом к системе управления исходным кодом, вероятно, из-за проблем с сетью или раздором.|
|SCC_E_CONNECTIONFAILURE|Не удалось подключиться к системе управления исходным источником.|
|SCC_E_FILENOTCONTROLLED|Выбранный файл не находится под контролем источника.|
|SCC_E_NONSPECIFICERROR|Неспецифический сбой.|

## <a name="remarks"></a>Примечания
 Эта функция позволяет вызывающему получить доступ к полному спектру функций системы управления исходным кодом с помощью инструмента внешнего администрирования. Если система управления исходным элементом не имеет пользовательского интерфейса, плагин управления исходным управлением может реализовать интерфейс для выполнения необходимых функций администрирования.

 Эта функция вызывается с подсчетом и массивом имен файлов для выбранных в настоящее время файлов. Если инструмент администрирования поддерживает его, список файлов может быть использован для предварительного выбора файлов в интерфейсе администрирования; в противном случае список может быть проигнорирован.

 Эта функция обычно вызывается, когда пользователь выбирает **сервер управления исходным \<>** из меню управления**исходным** **элементом файлов.** ->  Этот вариант меню **запуска** всегда может быть отключен или даже скрыт, установив вход в реестр. Смотрите, [как: Установите подключаемый модуль управления исходным управлением](../extensibility/internals/how-to-install-a-source-control-plug-in.md) для получения подробной информации. Эта функция называется только в том `SCC_CAP_RUNSCC` случае, если [SccInitialize](../extensibility/sccinitialize-function.md) возвращает бит возможностей (см. [Флаги возможностей](../extensibility/capability-flags.md) для получения подробной информации об этом и других битах возможностей).
