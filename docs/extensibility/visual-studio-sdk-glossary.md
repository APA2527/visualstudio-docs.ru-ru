# <a name="visual-studio-sdk-glossary"></a>Визуальная студия SDK глоссарий
Этот глоссарий содержит определения терминов, которые используются в документации. [!INCLUDE[vsipsdk](../extensibility/includes/vsipsdk_md.md)]

## <a name="terms"></a>Термины
 надстройка утилита приложения, драйвера или другого программного обеспечения, добавленного в основное приложение. В интегрированной среде разработки Visual Studio (IDE) надстройка — это приложение на основе автоматизации, расширяя возможности IDE.

 Модель автоматизации Модель автоматизации, известная в предыдущих версиях Visual Studio как модель расширяемости, представляет собой интерфейс программирования, который дает вам доступ к базовым процедурам, которые управляют IDE. Надстройки, мастера и макросы используют объекты в модели автоматизации для управления или расширения функциональности IDE.

 контекстная ассоциация контекста интерфейса GUID с видимостью команды или элемента интерфейса, такого как панель инструментов. Контекст командного uI не похож на контекст выбора, поскольку он не прикреплен к окну.

 Контекст командного uI можно использовать для:

- Назначаем GUID панели инструментов, которая появляется при активации конкретного окна.

- Присвоите GUID доступности команды без необходимости загрузки или запуска VSPackage.

- Назначить GUID, чтобы повлиять на активную связывание ключей.

- Назначай GUID для включаните макрозапись.

- Назначайте GUID для активации режима отладки или переключения между дизайном и режимом запуска в редакторе.

  компонент часть программного обеспечения, который может быть частью функциональности приложения без этого приложения, имеющие какую-либо уже существующую информацию о реализации программного обеспечения. Связь между компонентом и приложением происходит исключительно через интерфейсы стиля OLE.

  менеджер компонента `SOleComponentManager`A service, предоставляющий услуги координации непользовательских интерфейсов для компонентов верхнего уровня. Служба `SOleComponentManager` реализует `IOleComponentManager` интерфейс.

  компонент пользовательского интерфейса `SOleComponentUIManager`менеджер службы , которая обеспечивает услуги координации пользовательского интерфейса. Служба `SOleComponentUIManager` реализует `IOleComponentUIManager` интерфейсы `IOleInPlaceComponentUIManager` и интерфейсы.

  контекстный `IVsUserContext` мешок Объект (объект COM), прикрепленный к компоненту среды. Этот объект содержит ключевые слова поиска, ключевые слова **F1** и атрибуты, относящиеся к компоненту. Контекст мешки дополнительно указать на любые подконтекст мешки, которые связаны с ними.

  контекстный провайдер Компонент в IDE, который имеет контекстный пакет, связанный с ним. К таким компонентам относятся окно инструментов, редактор или иерархия проектов.

  дизайнер интерфейс программирования, который позволяет пользователям манипулировать элементами пользовательского интерфейса (формы, кнопки и другие элементы управления).

  Объект DocData A COM инкапсулирует базовые данные документа в мире, где существует разделение документов/просмотров (например, в случае с текстом редактора это будет текстовый буфер, лежащий в основе всех представлений редактора текста). Если EditorFactory не поставляет этот объект, IDE производит один от его имени. Ответственность этого объекта заключается в управлении сохранением данных и семантики обмена для нескольких представлений по этому же. `DocData` Если `DocData` объект поддерживает `IOleCommandTarget` интерфейс, он включен в командную разгром UIShell.

  Технология DocObject используется для размещения uI в рамках, предоставленной хостом. Более конкретно, этот термин относится к любому `IOleDocument` встраиванию, которое поддерживает и связанных с ними интерфейсов. Эта технология имеет много потенциальных приложений, таких как деталь реализации документов COM, окна инструментов в Visual Basic 5.0, конструкторы ActiveX в Visual Basic 6.0 и так далее.

  документ, используемый для обозначения в общем `DocData` документе `DocView`в целом , как и . Например, DocumentFrame содержит `DocView`, но он также сохраняет `DocData` ссылку на обработку настойчивости.

  DocView The DocObject/Embedding/WindowPane, с которым пользователь взаимодействует для `DocData`просмотра и манипулирования базовым. Пользователи не пользуются преимуществами разделения документов/представлений, которое является частью дизайна интерфейса. `DocObject` Пользователи используют весь DocObject, чтобы выступать в качестве представления вместо использования более абстрактного (и менее формализованного) понятия базовых данных, известных как `DocData`. `DocView`объекты всегда встраиваются в объекты кадра документа (детские окна MDI) IDE.

  Объект DTE The `DTE` (Development Tools Extensibility) является самой высокой точкой доступа в модели автоматизации Visual Studio, которая позволяет программно автоматизировать и расширить IDE.

  Динамическое окно окна справки, которое реализуется IDE и отображает список ключевых слов поиска или тем **F1** Справка.

  код редактора `DocView`(класс, CLSID), который реализует . Он также `DocData` реализуется, если разделение представления и данных поддерживается.

  расширение функция, которая изменяет, настраивает или добавляет к IDE. Вы создаете расширения с помощью модели автоматизации или VSPackages.

  внешний редактор Редактор, не специфичный для IDE, например Microsoft Word. Он был зарегистрирован через свои собственные механизмы и может использоваться за пределами IDE. Если этот редактор может быть встроен, он отображается в окне IDE. Если его невозможно встраить, создается отдельное окно верхнего уровня.

  иерархия Древо узлов, каждый узлы, связанные с набором свойств.

  независимый компонент верхнего уровня А, который использует безспособное окно верхнего уровня и может эффективно работать как отдельное окно приложения, но реализован как объект в процессе. Таким образом, независимый компонент верхнего уровня должен координировать службы модальности и цикла сообщений с IDE. Объекты in-process не имеют своего собственного цикла сообщений.

  поставщик информации Поставщик информации представляет собой модуль, который может искать ключевые `IVsUserContextItem` слова и возвращать список тем, в виде объектов. Чтобы предоставить **F1** и поиск ключевых слов для поставщика информации, зарегистрируйте компилированный файл справки (*. HxS*) с системой. Темы справки в этих файлах предоставляют список тем, отображаемых в окне динамической справки, и показывают, нажимает ли пользователь **на F1.**

  на месте компонент VSPackage объект, `IOleInPlaceComponent` который реализует интерфейс для управления окном, которое визуально содержится в окне документа, принадлежащем IDE. Компоненты на месте не участвуют в стандартном слиянии меню OLE; вместо этого они интегрируют свои элементы пользовательского интерфейса в IDE.

  Существует два типа компонентов на месте: проводные компоненты на месте и элементауправления компонентов.

  Компоненты hardwired на месте имеют меню, панели инструментов и команды, которые плотно интегрированы в пользовательский интерфейс IDE, как будто они были встроены непосредственно в IDE.

  Элементы управления компонентами не имеют элементов пользовательского интерфейса, интегрированных в IDE; вместо этого они используют меню IDE, команды и панели инструментов. Например, команда Bold может быть использована для смазать выбранное слово в богатом текстовом управлении, встроенном в форму. Тем не менее, элементы управления компонентами могут запросить отображение динамически установленных элементов uI для конкретных компонентов.

  языковой сервис Набор объектов, который позволяет разработчикам VSPackage реализовывать функции редакторов компьютерного кода, такие как маркировка текста и раскраска.

  Проект «Разные файлы» используется для размещения открытых файлов, которых нет ни в одном проекте. Список элементов в этом проекте не сохраняется.

  Проектные проекты состоят из объектов иерархии `IVsHierarchy` или объектов COM, реализующих интерфейс.

  проектно-специфический дизайнер или редактор Дизайнер, который не может быть использован независимо от типа проекта. Все проектные дизайнеры должны ввести информацию о своей редакции Factory в реестр. Затем IDE может мгновенно присваивать конструктора всякий раз, когда определенный тип файла открывается в конкретном проекте.

  окно типа проекта A окно, которое постоянно отслеживает активную в настоящее время иерархию проекта и элемент из контекста глобального выбора. Окна типа project `SVsTrackSelectionEx` используют службу для оповещения IDE об изменениях и отображения отзывов пользователей. Solution Explorer является примером окна типа проекта.

  Свойства окна Ранее Собственности Браузер.

  справочные проекты Project, которые не требуют, чтобы файлы для проекта были в том же каталоге. Вместо этого ссылки на файлы из других не связанных каталогов хранятся и поддерживаются самим проектом.

  запуск таблицы документов Внутренняя структура, в которой IDE ведет список всех открытых в настоящее время документов. Список включает в себя все открытые документы в памяти, независимо от того, документы в настоящее время редактируются. Документ — это любой сохраненный элемент, включая сохраненные процедуры, открытые в редакторе, файлы в проекте или основной файл проекта (например, файл s.vcproj).

  данные контекста выбора, которые являются частью детализации каждого окна в IDE и используются для отслеживания активных выборов. Контекст отбора состоит из:

- Указатель на `IVsHierarchy` интерфейс иерархии проекта

- Идентификатор элемента элемента проекта.

- Указатель на `ISelectionContainer` интерфейс, обеспечивающий доступ к свойствам для активных объектов.

- Массив значений элементов.

  обслуживание контракт на набор интерфейсов COM, которые находятся в одном объекте COM. При создании службы, которая идентифицируется GUID, вы определяете набор интерфейсов COM, который выполняет службу. Объекты COM используют службы для связи друг с другом.

  решение группы связанных проектов, с которыми работает пользователь.

  стандартный дизайнер дизайнер, который может быть использован независимо от типа проекта. Все стандартные дизайнеры должны ввести информацию о своей редакции Factory в реестре. Затем IDE может мгновенно присваивать конструктору всякий раз, когда открывается файл с определенным расширением. Данные должны сохраняться в файле.

  стандартный редактор, который может быть использован независимо от какого-либо конкретного типа проекта. Такие редакторы имеют EditorFactories зарегистрированы в реестре. Это позволяет IDE найти и вызвать редактора.

  стандартный редактор ОС Встраивание, которое не является специфюмом Visual Studio. Он зарегистрирован с помощью известных ключей Win32 (например, Win32 Explorer знает, как вызвать). Если такой редактор может быть встроен, редактор по-прежнему появляется на своем месте в IDE. В противном случае для таких редакторов создается отдельное окно верхнего уровня.

  подконтекстмешок `IVsUserContext` объект, связанный с контекстной сумкой. Объект содержит ключевые слова поиска, ключевые слова **F1** и атрибуты для выбора в компоненте IDE. Примеры подконтекста включают команду в окне инструмента или ключевое слово в редакторе.

  Окно инструмента списка задач, которое реализуется IDE и отображает список активных задач.

  текстовый буфер Общее `VSTextBuffer`имя для объекта .

  Текстовое представление Общее `VSTextView`имя для объекта .

  компонент верхнего уровня A компонент, который работает как безспособное всплывающее окно, плотно координируя с пользовательским интерфейсом IDE. Как и независимые компоненты верхнего уровня, компоненты верхнего уровня инструмента также должны координировать службы модальности и цикла сообщений с IDE.

  объект VSPackage верхнего уровня, управляющий безспособным окном верхнего уровня вместо клиентской области окна IDE. Компоненты верхнего `IOleComponent` уровня реализуют интерфейс, чтобы воспользоваться преимуществами служб цикла сообщений, таких как доступ к времени простоя.

  UI активный объект VSPackage, который виден и в настоящее время имеет фокус.

  Иерархия интерфейса A COM объект, который реализует `IVsUIHierarchy` интерфейс, позволяющий отображать иерархию. Окно иерархии интерфейса интерфейса `ISelectionContainer` реализуется для обновления окна Свойств; другие окна типа проекта могут использовать эту реализацию, при желании.

  Командный стол VSCT Visual Studio. Файл .vsct содержит информацию о размещении и поведении меню, панели инструментов и команд в формате XML.

  VSPackage Установить часть программного обеспечения, которое расширяет Visual Studio IDE, внося один или несколько из следующих элементов: пользовательский интерфейс, услуги, типы проектов, или редактор / дизайнер. VSPackage состоит из объекта COM, `IVsPackage` который реализует интерфейс и один или несколько других объектов COM, которые реализуют другие интерфейсы для поддержки выбора и других функций. Кроме того, VSPackage имеет конкретные требования к регистрации.
