# <a name="document-data-and-document-view-in-custom-editors"></a>Данные документов и представление документов в пользовательских редакторах
Пользовательский редактор состоит из двух частей: объекта данных документа и объекта представления документа. Как следует из названий, объект данных документа представляет текстовые данные, которые будут отображаться. Аналогичным образом объект представления документа (или "вид") представляет собой одно или несколько окон, в которых отображается объект данных документа.

## <a name="document-data-object"></a>Объект данных документа
 Объект данных документа — это представление данных текста в текстовом буфере. Это объект COM, который хранит текст документа и другую информацию. Объект данных документа также обрабатывает сохранение документа и позволяет несколько просмотров своих данных. Дополнительные сведения см. в разделе

 <xref:EnvDTE80.Window2.DocumentData%2A>и [документ Windows](../extensibility/internals/document-windows.md).

 Пользовательские редакторы и дизайнеры <xref:Microsoft.VisualStudio.TextManager.Interop.VsTextBuffer> могут использовать объект или свой собственный буфер. <xref:Microsoft.VisualStudio.TextManager.Interop.VsTextBuffer>следует упрощенной модели встраивания для стандартного редактора, поддерживает несколько представлений и предоставляет интерфейсы событий, которые используются для управления несколькими представлениями.

## <a name="document-view-object"></a>Объект представления документов
 Окно, отображаемый кодом и другим текстом, известно как представление документа или представление. При создании редактора можно выбрать одно представление, в котором текст отображается в одном окне. Или вы можете выбрать несколько представлений, в которых текст отображается в более чем одном окне. Ваш выбор зависит от вашего приложения. Например, если вам нужно боковое редактирование, вы выберете несколько представлений. Каждое представление связано с записью в таблице с помощью исходной таблицы документов с интегрированной средой разработки (IDE) (RDT). Окна представления принадлежат проекту <xref:Microsoft.VisualStudio.Shell.Interop.IVsHierarchy> или объекту.

 Если редактор поддерживает несколько представлений объекта данных документа, данные документа и объекты представления документа должны быть разделены. В противном случае они могут быть сгруппированы вместе. Для получения дополнительной информации [см.](../extensibility/supporting-multiple-document-views.md)

 IDE уведомляет представления о событиях (например, когда решение, содержащее документ закрыт), сопоставляя идентификатор элемента (ItemID) для каждой записи в таблице запущенных документов. Для получения дополнительной информации об этом [см.](../extensibility/internals/running-document-table.md)

 Существует два варианта создания представления для пользовательского редактора. Одним из них является модель активации на месте, где представление размещается в окне с использованием либо элемента управления ActiveX, либо объекта данных документа. Во-вторых, это упрощенная модель встраивания, где представление размещается [!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] и <xref:Microsoft.VisualStudio.Shell.Interop.IVsWindowPane> реализуется для обработки команд окон. Для получения информации о модели [In-place activation](/visualstudio/misc/in-place-activation?view=vs-2015)активации на месте см. Для получения информации об упрощенной модели встраивания [см.](../extensibility/simplified-embedding.md)
