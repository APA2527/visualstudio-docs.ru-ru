# <a name="test-area-1-add-toopen-from-source-control"></a>Область тестирования 1. Добавление и открытие элементов в системе управления версиями
Эта область тестирования подключаемого ключа охватывает размещение решений или проектов под контролем источников и их извлечение из-под управления исходом.

## <a name="command-menu-access"></a>Доступ к меню командования
 В [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] тестовых случаях используются следующие пути меню интегрированной среды разработки:

- Для [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)], открытый от управления исходным кодом: **Файл**, **Открыть**, **Проект**/**ноsolution**; посмотрите [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] в месте.

- Для других плагинов управления исходным кодом, открытый от управления исходным кодом: **Файл**, **Управление исходом**, **Открыть от управления исходом**.

- Добавьте в элемент управления исходным элементом: **Файл**, **Управление источником,** **Добавление решения в файл управления исходным элементом,** контроль **источника,** **добавление выбранных проектов в source Control.**

- Меню ярлыка (проект/решение), **добавить решение в управление исходом**.

- Добавить из элемента управления исходом: **Файл**, **Управление источником**, **Добавить проект из управления исходом**.

- Для, [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]добавить из элемента управления также доступна из **файла,** **Добавить**, **Существующий проект**; посмотрите [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] в месте.

  > [!NOTE]
  > В этом тесте можно использовать путь локального файла или локального IIS (веб-сервера).

## <a name="expected-behavior"></a>Ожидаемое поведение

- Для каждого поддерживаемого типа проекта пользователь должен иметь возможность "Добавить" и "Открыть от" Source Control.

- При добавлении проекта в Source \<Control создается соответствующий файл *ProjectName*>.vspscc (файл подсказки проекта). Он содержит список файлов исключений и информацию о связи. Не удаляйте этот файл, поскольку он содержит информацию, относясь к проекту.

- При добавлении решения в элемент \<управления исходным ресурсом создается соответствующий файл *SolutionName*>.vssscc (тройной S). Текстовый файл содержит информацию о связи и список файлов исключения, похожий на файл подсказки проекта. Этот файл является временным и существует только в базе данных управления исходным источником.

- При открытии решения из исходного элемента \<файл *SolutionName*>.vsscc (double S), который существует только в базе данных управления исходным элементом, создается локально во временном файле. Этот файл содержит путь от папки соединения решения к файлу решения. Этот файл является временным, и локальная копия удаляется после завершения операции "Open from Source Control".

- После добавления проекта в элемент управления исходным ресурсом можно выполнять любые действия управления исходным источником (Проверьте, Получите и так далее).
