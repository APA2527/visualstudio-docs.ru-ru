# <a name="test-area-1-add-toopen-from-source-control"></a>Область тестирования 1. Добавление и открытие из системы управления версиями
Этой системы управления версиями подключаемого модуля тестов описание области размещения решения или проекты в системе управления версиями и их извлечение из системы управления версиями.

## <a name="command-menu-access"></a>Доступ к меню команды
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] в тестовых случаях используются пути меню среды разработки:

- Для [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]откройте из системы управления версиями: **Файл**, **откройте**, **проекта**/**решение**; поиск в [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] расположение.

- Для других модулей системы управления версиями plug-откройте из системы управления версиями: **Файл**, **система управления версиями**, **открыть из системы управления версиями**.

- Добавьте в систему управления версиями: **Файл**, **система управления версиями**, **добавить решение в файл управления версиями**, **система управления версиями**, **добавить выбранные проекты в систему управления версиями**.

- Контекстное меню (решение или проект), **добавить решение в систему управления версиями**.

- Добавьте из системы управления версиями: **Файл**, **система управления версиями**, **добавьте проект из системы управления версиями**.

- Для [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)], добавьте из источника элемента управления также доступна из **файл**, **добавить**, **существующий проект**; поиск в [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] расположение.

  > [!NOTE]
  > В этом тесте может использоваться путь локального файла или локальный сервер IIS (веб-сервер).

## <a name="expected-behavior"></a>Ожидаемое поведение

- Для каждого проекта поддерживаемого типа пользователь должен быть «Добавить» и «Открыть из» системы управления версиями.

- При добавлении проекта к системе управления версиями, соответствующий \< *имя_проекта*> создается файл .vspscc (указание файла проекта). Он содержит исключения файла списка и сведений о соединении. Не удаляйте этот файл, так как она содержит сведения, относящиеся к проекту.

- При добавлении решения в систему управления версиями, соответствующий \< *SolutionName*> создается файл .vssscc (triple S). Текстовый файл содержит сведения о подключении и в файл список исключений, аналогичную указание файла проекта. Этот файл является временным и существует только в базе данных управления версиями.

- При открытии решения из системы управления версиями, \< *SolutionName*> файл .vsscc (double S), который существует только в базе данных управления версиями, создается локально во временном файле. Этот файл содержит путь к файлу решения из папки подключения решения. Этот файл является временным и локальную копию удаляется при завершении операции «Открыть из системы управления версиями».

- После добавления проекта в систему управления версиями, вы может выполнять любые действия управления источника на нем (извлечение, Get и т. д.).
