# <a name="test-area-4-check-in"></a>Область тестирования 4. Запись после изменения
Эта область тестирования подключаемый модуль системы управления версиями охватывает, отправка обновленных элементов в хранилище версий с помощью **вернуть** команды.

## <a name="command-menu-access"></a>Доступ к меню команды
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] в тестовых случаях используются пути меню в среде разработки.

##### <a name="check-in"></a>Зарегистрируйся:
 **Файл**, **система управления версиями**, **возврата**.

 **Файл**, **возврата**.

 Контекстное меню **вернуть**.

## <a name="common-expected-behavior"></a>Распространенное ожидаемое поведение

- Проекты и файлы, добавленные в решение или проект в системе управления версиями отображаются в **вернуть** диалоговое окно и **Отложенные возвраты** окна.

- После возврата добавленные элементы отображаются в системе управления версиями.

- После проверки в обновленных элементов, правильно управляются в хранилище.

## <a name="test-cases"></a>Тестовые случаи
 Ниже приведены определенные тестовые случаи для области проверки возврата.

### <a name="case-4a-modified-items"></a>Case 4а. Элементы, измененные
 Описывается использование проверки в действие для обновления файла в системе управления версиями, которые были изменены.

|Действие|Шаги теста|Ожидаемые результаты для проверки|
|------------|----------------|--------------------------------|
|Изменения в файл текст, который был извлечен, проверьте в файле только (**вернуть** диалоговое окно)|1.  Создайте новый проект с помощью текстового файла.<br />2.  Чтобы добавьте решение в систему управления версиями.<br />3.  Извлечение и изменение текстового файла.<br />4.  Возврат через диалоговое окно «Возврат» (**файл**, **системы управления версиями**, **вернуть**).|Распространенное ожидаемое поведение.|
|Изменения в файл текст, который был извлечен, проверьте в файле только (**Отложенные возвраты** окно)|1.  Создайте новый проект с помощью текстового файла.<br />2.  Чтобы добавьте решение в систему управления версиями.<br />3.  Извлечение и изменение текстового файла.<br />4.  Возврат через **Отложенные возвраты** окна.|Распространенное ожидаемое поведение.|

### <a name="case-4b-adding-files"></a>Case 4б. Добавление файлов
 При добавлении файла в проект или элемент в решение, проект или решение, необходимо изменить также. Таким образом родительский файл извлечен также и должны быть возвращены для завершения добавления.

|Действие|Шаги теста|Ожидаемые результаты для проверки|
|------------|----------------|--------------------------------|
|Добавьте в текстовый файл и вернуть все (**вернуть** диалоговое окно)|1.  Создайте новый проект.<br />2.  Чтобы добавьте решение в систему управления версиями.<br />3.  Добавьте в текстовый файл в проект.<br />4.  Извлечение проекта следует примите при появлении соответствующего запроса.<br />5.  Выберите решение в **обозревателе решений**.<br />6.  Регистрация из **вернуть** диалоговое окно.|Распространенное ожидаемое поведение.|
|Добавьте в текстовый файл и вернуть все (**Отложенные возвраты** окно)|1.  Создайте новый проект.<br />2.  Чтобы добавьте решение в систему управления версиями.<br />3.  Добавьте в текстовый файл в проект.<br />4.  Извлечение проекта следует примите при появлении соответствующего запроса.<br />5.  Возврат решения из **Отложенные возвраты** окна.|Распространенное ожидаемое поведение|
