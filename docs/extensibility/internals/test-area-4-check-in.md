# <a name="test-area-4-check-in"></a>Область тестирования 4. Возврат
Эта тестовая зона подключаемого ключа охватывает отправку обновленных элементов в магазин версий через команду **Check In.**

## <a name="command-menu-access"></a>Доступ к меню командования
 В [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] тестовых случаях используются следующие пути меню интегрированной среды разработки.

##### <a name="check-in"></a>Вернуть:
 **Файл**, **Управление источником,** **Регистрация .**

 **Файл**, **Регистрация в**.

 Меню ярлыка, **Регистрация .**

## <a name="common-expected-behavior"></a>Общее ожидаемое поведение

- Проекты и файлы, добавленные в решение или проект под контролем источника, отображаются в диалоговом окне **Check In** и в окне **ожидающих проверки.**

- После регистрации добавленные элементы отображаются в элементе исходного элемента.

- После регистрации обновленные элементы должным образом отожносятся в магазине.
