# <a name="test-area-7-share"></a>Область тестирования 7. Предоставление общего доступа
Эта область тестирования охватывает совместное использование элементов между местами через команду **Share.**

 Операция hhare — это явное дублирование файлов и элементов папок между двумя или более местами в иерархии файла управления исходным управлением. Дублирование на самом деле не происходит на сервере, но пользователь видит один и тот же файл в двух или более указанных местах. Всякий раз, когда в какие-либо общие элементы вносятся изменения, эти изменения появляются во всех других общих местах.

 Совместное использование папок работает, если вы выбираете папку с по крайней мере одним файлом под контролем источника в нем. Команда общего обмена отключена при следующих условиях:

- Если выбранная папка является пустой папкой.

- Если есть реальная папка, но она не содержит файлов управления исходным источником.

- Если есть виртуальная папка, будь то файлы под контролем источника в нем или нет.

- Если есть веб-проект удаленного сайта.

## <a name="command-menu-access"></a>Доступ к меню командования
 В [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] тестовых случаях используются следующие пути меню интегрированной среды разработки.

 Доля:**Доля****управления**->источником **файлов**->.

## <a name="expected-behavior"></a>Ожидаемое поведение

- Общий файл отображается в общем местоположении.

- Просмотр истории магазина исходного управления показывает, что файл (ы) являются общими.

- Редактирование общего файла редактирует оба местоположения файла.
