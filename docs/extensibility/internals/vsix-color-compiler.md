# <a name="vsix-color-compiler"></a>Компилятор цветов VSIX
Инструмент Visual Studio Extension Color Compiler представляет собой консольное приложение, которое использует файл .xml, представляющий цвета для существующих тем Visual Studio, и закатит его в файл .pkgdef, чтобы эти цвета могли быть использованы в Visual Studio. Поскольку легко сравнивать различия между файлами .xml, этот инструмент полезен для управления пользовательскими цветами в элементах управления исходными данными. Он также может быть подключен к средам сборки, так что выход сборки является действительным файлом .pkgdef.

 **Тема XML схема**

 Полный файл .xml выглядит следующим образом:

```xml
<Themes>
      <!—one or Theme elements -->
      <Theme>
        <!-- one or more Category elements -->
        <Category>
          <!-- one or more Color elements -->
          <Color>
            <!-- zero or one Background element -->
            <Background />
            <!-- zero or one Foreground element -->
            <Foreground />
          </Color>
        </Category>
      </Theme>
</Themes>
```

 **Тема**.

 Элемент \<«Тема> определяет всю тему. Тема должна содержать по \<крайней мере один элемент категории>. Тематические элементы определяются следующим образом:

```xml
<Theme Name="name" GUID="guid">
      <!-- one or more Category elements -->
</Theme>
```

|||
|-|-|
|**Атрибут**|**Определение**|
|name|(Обязательно) Название темы|
|GUID|(Обязательно) GUID темы (должен соответствовать форматированию GUID)|

 При создании пользовательских цветов для Visual Studio эти цвета должны быть определены для следующих тем. Если цвета не существуют для конкретной темы, Visual Studio пытается загрузить недостающие цвета из темы Света.

|||
|-|-|
|**Название темы**|**Тема GUID**|
|Светлый|(de3dbbcd-f642-433c-8353-8f1df4370aba)|
|Dark|1ded0138-47ce-435e-84ef-9ec1f439b749|
|Синий|(a4d6a176-b948-4b29-8c66-53c97a1ed7d0)|
|Высокая контрастность|(a4d6a176-b948-4b29-8c66-53c97a1ed7d0)|

 **Категория**

 Элемент \<категории> определяет набор цветов в теме. Названия категорий обеспечивают логические группировки и должны быть определены как можно более узко. Категория должна содержать по \<крайней мере один элемент> цвета. Элементы категории определяются следующим образом:

```xml
<Category Name="name" GUID="guid">
      <!-- one or more Color elements -->
 </Category>
```

|||
|-|-|
|**Атрибут**|**Определение**|
|name|(Обязательно) Название категории|
|GUID|(Обязательно) GUID категории (должен соответствовать форматированию GUID)|

 **Цвет**

 Элемент \<Color> определяет цвет для компонента или состояния uI. Предпочтительной схемой именования цвета является «тип UI» (государство). Не используйте слово "цвет", так как оно является излишним. Цвет должен четко указывать тип элемента и ситуации, или "состояние", для которого цвет будет применяться. Цвет не должен быть пустым и должен содержать \<один или \<оба элемента фонового> и переднего плана> элемент. Цветовые элементы определяются следующим образом:

```xml
<Color Name="name">
        <Background /> <!-- zero or one Background element -->
        <Foreground /> <!-- zero or one Foreground element -->
 </Color>
```

|||
|-|-|
|**Атрибут**|**Определение**|
|name|(Обязательно) Название цвета|

 **Фон и/или передний план**

 Фоновая \<> \<и элементы переднего плана> определяют значение и тип цвета для фона или переднего плана элемента uI. Эти элементы не имеют детей.

```xml
<Background Type="type" Source="int" />
<Foreground Type="type" Source="int" />
```

|||
|-|-|
|**Атрибут**|**Определение**|
|Type|(Обязательно) Тип цвета. Может принимать одно из следующих значений:<br /><br /> *CT_INVALID:* Цвет недействителен или не установлен.<br /><br /> *CT_RAW:* Сырое значение ARGB.<br /><br /> *CT_COLORINDEX:* НЕ ИСПОЛЬЗУЙТЕ.<br /><br /> *CT_SYSCOLOR:* Цвет системы Windows от SysColor.<br /><br /> *CT_VSCOLOR:* Цвет визуальной студии от __VSSYSCOLOREX.<br /><br /> *CT_AUTOMATIC:* Автоматический цвет.<br /><br /> *CT_TRACK_FOREGROUND:* НЕ ИСПОЛЬЗУЙТЕ.<br /><br /> *CT_TRACK_BACKGROUND:* НЕ ИСПОЛЬЗУЙТЕ.|
|Источник|(Обязательно) Значение цвета, представленного в гексадецимного|

 Все значения, поддерживаемые __VSCOLORTYPE перечисления, поддерживаются схемой в атрибуте типа. Тем не менее, мы рекомендуем использовать только CT_RAW и CT_SYSCOLOR.

 **Все вместе**

 Это простой пример действительной темы .xml файла:

```xml
<Themes>
  <Theme Name="Light" GUID="{de3dbbcd-f642-433c-8353-8f1df4370aba}">
    <Category Name="MyCategory" GUID="{0A96238B-70CE-4479-9170-EECEAA3FCD58}">
      <Color Name="MyActiveBorder">
        <Background Type="CT_RAW" Source="FFCCCEDB" />
      </Color>
    </Category>
  </Theme>
</Themes>
```

## <a name="notes"></a>Примечания

- Этот инструмент требует установки последней версии времени выполнения VC.

- Поддерживаются только отдельные файлы. Массовое преобразование через пути папки не поддерживается.
