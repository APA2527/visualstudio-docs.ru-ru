# <a name="project-priority"></a>Приоритет проекта
Элемент проекта обычно является участником только одного проекта в решении. Таким образом, IDE может легко определить, какой проект используется для открытия элемента. Однако, если элемент является участником более чем одного проекта, IDE использует схему приоритетов для определения наилучшего проекта для открытия элемента.

 В следующем списке показана схема приоритета проекта:

- IDE называет <xref:Microsoft.VisualStudio.Shell.Interop.IVsProject2.IsDocumentInProject%2A> метод для каждого проекта в решении, чтобы определить, является ли документ участником этого проекта.

- Если документ является участником проекта, проект отвечает приоритетом, который проект присваивает в соответствии с его обработкой этого документа. Например, проект языка отвечает с высоким приоритетом для исходных файлов языка, но отвечает с более низким приоритетом для нераспознаванного типа файла, который не используется как часть процесса сборки.

- Проекты, предоставляющие пользовательские, конкретные редакторы или дизайнеры для документа, также получают высокий приоритет.

- В <xref:Microsoft.VisualStudio.Shell.Interop.VSDOCUMENTPRIORITY> перечислении приводятся значения приоритета документа.

- Проекту, в рамках которого указывается наивысший приоритет, дается контекст для открытия документа. Если два проекта возвращают равные значения приоритета, предпочтение отдается активному проекту. Если ни один проект в решении не отвечает, что он может открыть документ, IDE помещает документ в проект «Разные файлы». Для получения дополнительной информации [см.](../../extensibility/internals/miscellaneous-files-project.md)
