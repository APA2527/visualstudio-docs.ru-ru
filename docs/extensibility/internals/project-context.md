# <a name="project-context"></a>Контекст проекта
Когда пользователь добавляет или работает с проектами и элементами проекта, IDE использует понятие контекста проекта для определения того, как должны выполняться различные операции.

 Как правило, файлы являются стандартными объектами проекта, которые пользователь явно создает, выбрав команду **Нового Проекта** или сделать доступными, выбрав команду **Open Project** в меню **файла.** В этих случаях файлы создаются и открываются в контексте проекта, а тип проекта определяет контекст для редактирования документа.

 Некоторые проекты обеспечивают очень богатый контекст. Например, проект управляет проектным, программным пространством имен или подключением базы данных к проекту для связывания данных. Пользователь часто может открывать файлы или соединения баз данных непосредственно с помощью определенного объекта проекта, например элемента проекта, отображаемого в Solution Explorer.

 В других случаях контекст проекта элемента прямо не указан. Например, контекст элемента недоступен, когда пользователь открывает файл, выбрав команду **Open Existing File** в меню **файла,** когда отладчик работает в файле, или когда пользователь нажимает на команду **«Найти в файлах»** в поле поиска **и замены** диалогов. Для обработки этих ситуаций <xref:Microsoft.VisualStudio.Shell.Interop.IVsUIShellOpenDocument> IDE призывает управлять процессом поиска наилучшего проекта для открытия документа.
