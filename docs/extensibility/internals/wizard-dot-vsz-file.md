# <a name="wizard-vsz-file"></a>Файл мастера (VSZ-файл)

Интегрированная среда разработки (IDE) использует файлы .vsz для запуска колдунов. Эти файлы .vsz содержат информацию, которую IDE использует для определения того, какой мастер вызывает вызов и какую информацию передать мастеру.

Файл .vsz — это версия текстового файла.ini-formatted, в нее нет разделов. Информация, известная IDE, хранится в начале файла. Это обеспечивает связь между мастером, который вызывает IDE, и параметрами, которые находятся в файле .vsz, который будет передан IDE. Остальная часть файла содержит параметры, которые специфичны для мастера и которые должны быть собраны IDE и переданы конкретному мастеру.

В следующем примере показано содержимое файла .vsz.

```
VSWizard 8.0
Wizard=VsWizard.CBlankSiteWizard -or- {123-1234556-123334}
Param="WIZARDNAME = Wizard One"
Param="WIZARDUI = FALSE"
```

Ниже приведены части в файле .vsz.

|Часть|Описание|
|----------|-----------------|
|VSWizard|Первым параметром в файле является номер версии формата файла шаблона. Этот номер версии должен быть 6.0, 7.0, 7.1, или 8.0. Другие номера не могут быть запущены и вызывают ошибку недействительного формата.|
|Мастер|Это поле содержит OLE ProgID мастера, или же GUID строку представление CLSID мастера, который создан IDE.|
|Параметр|Эти части не являются обязательными. Вы можете добавить столько, сколько необходимо.|

Параметры позволяют файлу .vsz передавать мастеру дополнительные пользовательские параметры. Каждое значение передается мастеру в виде элемента строки в массиве вариантов. Для получения дополнительной [информации см.](../../extensibility/internals/custom-parameters.md)

Чтобы добавить идентификатор локального значения `FALLBACK_LCID`по умолчанию в файл .vsz, укажите, например, что xxxx является идентификатором локера, например, 1033 для английского языка. При `FALLBACK_LCID` определении параметра мастер использует идентификатор локального резервного использования, если текущий идентификатор не найден.
