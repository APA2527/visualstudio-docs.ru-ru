# <a name="test-area-3-check-outundo-checkout"></a>Область тестирования 3. Извлечение и отмена извлечения
Эта область тестирования управления исходным элементом охватывает редактирование и возврат элементов из магазина версий с помощью команд **Check Out** и **Undo Checkout.**

**Отъезд**: Отметки пункта в магазине версии, как проверили, изменяет локальную копию для чтения / записи.

**Отменить**оформление: Отметки элемента в магазине версии, как зарегистрированный, возвращает локальную копию к государству до выезда (в зависимости от вариантов).

## <a name="command-menu-access"></a>Доступ к меню командования

В [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] тестовых случаях используются следующие пути меню интегрированной среды разработки.

##### <a name="check-out"></a>Извлечь:

- **Файл**, **Управление источником**, **Проверьте**.

- **Файл**, **Проверьте**.

- Shortcut Меню, **Проверьте**.

- Отменить оформление: **Файл**, **Управление источником**, **Отменить оформление**.

## <a name="common-expected-behavior"></a>Общее ожидаемое поведение

- После операции выезда целевой файл (ы) и/или папка (ы) помечаются как проверенные в магазине версий.

- Магазин версий приписывает кассу правильному пользователю.

- Время и дата оформления являются правильными (в настройках пользователя).

### <a name="case-3b-disconnected-check-out"></a>Дело 3b: Отключенный отъезд

Работа в отключенном режиме позволяет пользователям на определенном уровне непрерывной поддержки управления исходным источником, когда они не подключены непосредственно к магазину версий. Это делается путем локального кэширования всей соответствующей информации о завербованном решении и проектах.

Эксклюзивные операции проверки могут происходить только при подключении к магазину управления исходным источником. Общие операции выезда могут происходить в любое время, независимо от того, подключены или отключены. Поэтому при отключении от хранилища версий включена только общая команда **Check Out (COS).** При отключении **отключите, Отключите от записи,** поскольку старая версия не может быть восстановлена для замены изменений, внесенных пользователем.

При повторном подключении пользователя к магазину версий состояния возврата всех завербованных решений и проектов синхронизируются. Это делает необходимые обновления в магазин для проверки, что пользователь выполнил. После синхронизации пользователь может продолжать работать в обычном режиме (подключен).

#### <a name="expected-behavior"></a>Ожидаемое поведение

- Не удается использовать команду **Check Out Exclusively** при отключении от магазина версий.

- Не удается использовать команду **Undo Checkout** при отключении от магазина версий.

- Общие работы команды **Check Out.**

|Действие|Тестовые шаги|Ожидаемые результаты для проверки|
|------------|----------------|--------------------------------|
|При отключении, проверить файл, а затем подключитьдля синхронизации|1. ### <a name="case-3e-undo-check-out"></a>Дело 3e: Отменить выезд
