# <a name="sccdirdiff-function"></a>Функция SccDirDiff
Эта функция отображает различия между текущим локальным каталогом на клиентском диске и соответствующим проектом под контролем источника.

## <a name="syntax"></a>Синтаксис

```cpp
SCCRTN SccDirDiff(
   LPVOID    pContext,
   HWND      hWnd,
   LPCSTR    lpDirName,
   LONG      dwFlags,
   LPCMDOPTS pvOptions
);
```

### <a name="parameters"></a>Параметры
 pContext

(в) Структура контекста управления исходным элементом.

 Hwnd

(в) Ручка к окну IDE, которую плагин управления исходным элементом может использоваться в качестве родительского элемента для любых диалоговых коробок, которые она предоставляет.

 lpDirName

(в) Полностью квалифицированный путь к местному каталогу, для которого можно показать визуальную разницу.

 dwFlags

(в) Флаги командования (см. раздел Замечания).

 pvOptions

(в) Опции управления исходным управлением.

## <a name="return-value"></a>Возвращаемое значение
 Ожидается, что внедрение этой функции элемента управления исходным элементом вернет одно из следующих значений:

|Значение|Описание|
|-----------|-----------------|
|SCC_OK|Каталог на диске такой же, как и проект в управлении исходным кодом.|
|SCC_I_FILESDIFFER|Каталог на диске отличается от проекта в управлении исходным кодом.|
|SCC_I_RELOADFILE|Файл или проект необходимо перезагрузить.|
|SCC_E_FILENOTCONTROLLED|Каталог не находится под контролем исходного кода.|
|SCC_E_NOTAUTHORIZED|Пользователю не разрешается выполнять эту операцию.|
|SCC_E_ACCESSFAILURE|Возникла проблема с доступом к системе управления исходным кодом, вероятно, из-за проблем с сетью или раздором. Рекомендуется повторная попытка.|
|SCC_E_NONSPECIFICERROR<br /><br /> SCC_E_UNKNOWNERROR|Неспецифический сбой.|
|SCC_E_FILENOTEXIST|Местный каталог найти не удалось.|

## <a name="remarks"></a>Примечания
 Эта функция используется для инструктажа плагина управления исходным элементом для отображения пользователю списка изменений в определенном каталоге. Плагин открывает собственное окно в формате по своему выбору, чтобы отобразить различия между каталогом пользователя на диске и соответствующим проектом под управлением версии.

 Если плагин поддерживает сравнение каталогов вообще, он должен поддерживать сравнение каталогов на основе файловых имен, даже если параметры "быстрого диффа" не поддерживаются.

|`dwFlags`|Интерпретация|
|---------------|--------------------|
|SCC_DIFF_IGNORECASE|Сравнение без чувственного случая (может быть использовано для быстрого диффа или визуального).|
|SCC_DIFF_IGNORESPACE|Игнорирует белое пространство (может быть использовано для быстрого диффа или визуального).|
|SCC_DIFF_QD_CONTENTS|Если поддерживается плагином управления исходным ключом, бесшумно сравнивает каталог, байт байтом.|
|SCC_DIFF_QD_CHECKSUM|Если поддерживается плагином, бесшумно сравнивает каталог через чексум, или, если не поддерживается, возвращается к SCC_DIFF_QD_CONTENTS.|
|SCC_DIFF_QD_TIME|Если поддерживается плагином, бесшумно сравнивает каталог через его отметку времени, или, если не поддерживается, падает обратно на SCC_DIFF_QD_CHECKSUM или SCC_DIFF_QD_CONTENTS.|

> [!NOTE]
> Эта функция использует те же флаги команды, что и [SccDiff.](../extensibility/sccdiff-function.md) Тем не менее, плагин управления исходным кодом может отказаться от поддержки операции "быстрый дифф" для каталогов.
