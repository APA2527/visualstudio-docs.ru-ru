# <a name="scchistory-function"></a>Функция SccHistory
Эта функция отображает историю указанных файлов.

## <a name="syntax"></a>Синтаксис

```cpp
SCCRTN SccHistory(
   LPVOID    pvContext,
   HWND      hWnd,
   LONG      nFiles,
   LPCSTR*   lpFileNames,
   LONG      fOptions,
   LPCMDOPTS pvOptions
);
```

#### <a name="parameters"></a>Параметры
 `pvContext`

(в) Структура контекста управления исходным элементом.

 `hWnd`

(в) Ручка к окну IDE, которую плагин управления исходным элементом может использоваться в качестве родительского элемента для любых диалоговых коробок, которые она предоставляет.

 `nFiles`

(в) Количество файлов, указанных в массиве. `lpFileName`

 `lpFileName`

(в) Массив полностью квалифицированных имен файлов.

 `fOptions`

(в) Флаги командования (в настоящее время не используются).

 `pvOptions`

(в) Опции управления исходным управлением.

## <a name="return-value"></a>Возвращаемое значение
 Ожидается, что внедрение этой функции элемента управления исходным элементом вернет одно из следующих значений:

|Значение|Описание|
|-----------|-----------------|
|SCC_OK|История версии была успешно получена.|
|SCC_I_RELOADFILE|Система управления исходным элементом фактически изменила файл на диске, изнавив историю (например, получив старую версию), поэтому IDE должен перезагрузить этот файл.|
|SCC_E_FILENOTCONTROLLED|Файл не находится в системе управления версиями.|
|SCC_E_OPNOTSUPPORTED|Система управления исходным источником не поддерживает эту операцию.|
|SCC_E_NOTAUTHORIZED|Пользователю не разрешается выполнять эту операцию.|
|SCC_E_ACCESSFAILURE|Возникла проблема с доступом к системе управления исходным кодом, вероятно, из-за проблем с сетью или раздором. Рекомендуется повторная попытка.|
|SCC_E_PROJNOTOPEN|Проект не открыт.|
|SCC_E_NONSPECIFICERROR|Неспецифический сбой. История файлов не может быть получена.|

## <a name="remarks"></a>Примечания
 Плагин управления исходным элементом может отображать свой собственный диалоговый `hWnd` ящик, чтобы показать историю каждого файла, используя в качестве родительского окна. Кроме того, может использоваться дополнительная функция обратного вызова вывода текста, поставляемая [в SccOpenProject,](../extensibility/sccopenproject-function.md) если она поддерживается.

 Обратите внимание, что при определенных обстоятельствах исследуемый файл может изменяться во время выполнения этого вызова. Например, [!INCLUDE[vsvss](../extensibility/includes/vsvss_md.md)] команда истории дает пользователю возможность получить старую версию файла. В таком случае, источник управления плагинвозвращает возвращается, `SCC_I_RELOAD` чтобы предупредить IDE, что он должен перезагрузить файл.

> [!NOTE]
> Если плагин управления исходным элементом не поддерживает эту функцию для массива файлов, можно отобразить только историю файлов для первого файла.
