# <a name="images-and-icons-for-visual-studio"></a>Изображения и значки для Visual Studio
## <a name="image-use-in-visual-studio"></a><a name="BKMK_ImageUseInVisualStudio"></a>Использование изображения в Visual Studio
 Перед созданием произведений искусства, рассмотреть возможность использования 1000 "изображений в [библиотеке изображений Visual Studio](https://www.microsoft.com/download/details.aspx?id=35825).

### <a name="types-of-images"></a>Типы изображений

- **Иконки**. Небольшие изображения, которые отображаются в командах, иерархиях, шаблонах и так далее. Размер значка по умолчанию, используемый в Visual Studio, — это PNG 16x16. Значки, производимые службой изображений, автоматически генерируют формат XAML для поддержки HDPI.

    > [!NOTE]
    > В то время как изображения используются в системе меню, не следует создавать значок для каждой команды. Проконсультируйтесь с [меню и командами для Visual Studio,](../../extensibility/ux-guidelines/menus-and-commands-for-visual-studio.md) чтобы узнать, должна ли ваша команда получить значок.

- **Миниатюры.** Изображения, используемые в области предварительного просмотра диалога, такие как диалог нового проекта.

- **Диалогизображения.** Изображения, которые отображаются в диалогах или мастеров, либо в качестве описательной графики или индикаторов сообщений. Используйте нечасто и только тогда, когда это необходимо, чтобы проиллюстрировать сложную концепцию или привлечь внимание пользователя (предупреждение, предупреждение).

- **Анимированные изображения.** Используется в индикаторах выполнения, полосках состояния и диалогах операций.

- **Курсоры.** Используется для указания того, разрешена ли операция с помощью мыши, где объект может быть удален, и так далее.

## <a name="icon-design"></a><a name="BKMK_IconDesign"></a>Дизайн иконок

### <a name="overview"></a>Обзор
 Visual Studio использует иконы в современном стиле, которые имеют чистую геометрию и баланс 50/50 положительных/отрицательных (светлых/темных) и используют прямые, понятные метафоры. Критические точки проектирования значок центр вокруг ясности, упрощения и контекста.

- **Ясность:** сосредоточьтесь на основной метафоре, которая придает иконе ее значение и индивидуальность.

- **Упрощение:** уменьшить значок до его основного значения - получить тему через только с необходимым элементом (ы) и без излишеств.

- **Контекст:** рассмотрим все аспекты роли значка во время разработки концепции, что имеет решающее значение при принятии решения о том, какие элементы составляют основную метафору иконы.

  С иконами, Есть ряд точек проектирования, чтобы избежать:

- Не используйте значки, обозначающие элементы uI, за исключением случаев, когда это необходимо. Выберите более абстрактный или символический подход, когда элемент uI не является ни общим, ни очевидным, ни уникальным.

- Не злоупотребляйте общими элементами, такими как документы, папки, стрелки и увеличительное стекло. Используйте такие элементы только тогда, когда необходимо значение значка. Например, в правом увеличительное стекло должно быть указано только поиск, просмотр и поиск.

- Хотя некоторые элементы значка устаревшего сохраняют использование перспективы, не создавайте новые значки с перспективой, если элементне не имеет ясности без него.

- Не втиснуть слишком много информации в значок. Простое изображение, которое можно легко распознать или узнать как узнаваемый символ, гораздо полезнее, чем слишком сложное изображение. Икона не может рассказать всю историю.

### <a name="production-details"></a>Детали производства
 Все новые элементы uI должны быть созданы с помощью Windows Presentation Foundation (WPF), а все новые значки для WPF должны быть в 32-битном формате PNG. 24-разрядный PNG является устаревшим форматом, который не поддерживает прозрачность и поэтому не рекомендуется для иконок.

 Сохранить разрешение на 96 DPI.

#### <a name="file-types"></a>Типы файлов

- **32-битный PNG:** предпочтительный формат для иконок. Без потерь формат сжатия данных файл, который может хранить один raster (пиксель) изображение. 32-разрядные PNG-файлы поддерживают прозрачность альфа-канала, гамма-коррекцию и переплетение.

- **32-битная БМП:** для элементов управления, не относясь к WPF. Также называется XP или высокого цвета, 32-битный БМП является Формат RGB / A изображения, истинного цвета изображения с альфа-канал прозрачности. Альфа-канал представляет собой слой прозрачности, обозначенный в Adobe Photoshop, который затем сохраняется в биткарте в качестве дополнительного (четвертого) цветного канала. Черный фон добавляется во время производства произведений искусства для всех 32-разрядных файлов BMP, чтобы обеспечить быстрый визуальный сигнал о глубине цвета. Этот черный фон представляет область, которая будет замаскирована в uI.

- **32-битное ICO:** для значков проекта и добавления товара. Все файлы ICO имеют 32-битный истинный цвет с прозрачностью альфа-канала (RGB/A). Поскольку файлы ICO могут хранить несколько размеров и цветовых глубин, значки Vista часто находятся в формате ICO, содержащем размеры изображений 16x16, 32x32, 48x48 и 256x256. Для правильного отображения в Windows Explorer файлы ICO должны быть сохранены до 24-битной и 8-битной цветной глубины для каждого размера изображения.

- **XAML:** для дизайнерских поверхностей и украшений Windows. Значки XAML — это векторные файлы изображений, которые поддерживают масштабирование, вращение, подачу и прозрачность. Они не являются общими в Visual Studio сегодня, но становятся все более популярными из-за их гибкости.

- **SVG**

- **24-битная БМП:** для командной панели Visual Studio. Формат изображения RGB, 24-битная БМП — это конвенция значка, которая создает слой прозрачности с помощью пурпурного (R-255, G'0, B'255) в качестве цветного ключа для слоя прозрачности нокаута. В 24-разрядной БМП все пурпурные поверхности отображаются с использованием фонового цвета.

- **24-разрядный GIF:** для панели команд Visual Studio. Истинный цвет RGB формат изображения, который поддерживает прозрачность. Файлы GIF часто используются в художественных работах Мастера и gIF-анимациях.

### <a name="icon-construction"></a>Конструкция иконы
 Самый маленький размер значка в Visual Studio 16x16. Самый большой в общем использовании 32x32. Имейте в виду, чтобы не заполнить весь 16x16, 24x24, или 32x32 кадр при проектировании значка. Разборчивые, равномерные конструкции значка имеет важное значение для распознавания пользователей. Придерживайтесь следующих точек при построении значков.

- Иконы должны быть четкими, понятными и последовательными.

- Элементы уведомления о статусе лучше использовать в качестве отдельных иконок, а не складывать их поверх базового элемента значка. В некоторых контекстах uI может потребовать, чтобы элемент состояния был сопряжен с базовым элементом.

- Значки проекта обычно представляют файлы .ico, которые содержат несколько размеров. Обновляются только иконки 16x16, 24x24 и 32x32. Большинство икон оконь як и 16x16 и 24x24 будут содержать те же элементы. Иконки 32x32 содержат более подробную информацию, включая тип языка проекта, когда это применимо.

- Для иконок 32x32 базовые элементы обычно имеют 2-пиксельный вес линии. 1- или 2-пиксельная линия вес может быть использован для деталей элементов. Используйте ваши лучшие суждения, чтобы определить, что является более подходящим.

- Ите хотя бы 1-пиксельное расстояние между элементами для иконок 16x16 и 24x24. Для значков 32x32 используйте 2-пиксельное расстояние между элементами и между модификатором и базовым элементом.

  ![Расстояние между элементами для иконок размером 16x16, 24x24 и 32x32](../../extensibility/ux-guidelines/media/0404-47_elementspacing.png "0404-47_ElementSpacing")<br />Расстояние между элементами для иконок размером 16x16, 24x24 и 32x32

### <a name="overview"></a>Обзор
 Иконы в Visual Studio в основном монохромны. Цвет зарезервирован для передачи конкретной информации и никогда не для украшения. Цвет используется:

- для указания действия

- чтобы предупредить пользователя об уведомлении о статусе

- для обозначения языковой принадлежности

- дифференцировать элементы в IntelliSense

### <a name="accessibility"></a>Возможности доступа
 Руководящие принципы соответствия Visual Studio требуют, чтобы все значки, зарегистрированные в продукте, проходили требования к доступности цвета и контрастности. Цвета в визуальной языковой палитре были протестированы и отвечают этим требованиям.

Ограничьте использование цвета в этой палитре, или вы получите непредсказуемые результаты, когда инверсия применяется.
