# <a name="layout-for-visual-studio"></a>Макет для Visual Studio
Большинство диалогов Visual Studio являются [утилита диалог макет](../../extensibility/ux-guidelines/layout-for-visual-studio.md#BKMK_UtilityDialogLayout), которые являются unthemed диалогов, которые следуют стандартным [Windows Desktop диалог макет принципов.](/windows/desktop/uxguide/win-dialog-box) По мере того как Visual Studio движется к обновлению своего uI, некоторые из наиболее известных диалогов имеют новый дизайн, который устанавливает их как продукт определяя опыт. Эти [тематические макет диалога](../../extensibility/ux-guidelines/layout-for-visual-studio.md#BKMK_ThemedDialogLayout) имеют тематический внешний вид.

## <a name="utility-dialog-layout"></a><a name="BKMK_UtilityDialogLayout"></a>Макет диалога утилиты

- Все элементы управления в диалоге утилиты должны начинаться в верхней /левой части и течь вниз.

- Никогда не центр управления на диалогдля для заполнения большой площади.

- Используйте шрифт среды для всего диалогового текста. При написании визуальной спецификации укажите шрифт среды вместо выбора конкретного шрифта и размера. Смотрите [шрифт среды](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md#BKMK_TheEnvironmentFont).

- Используйте последовательное расстояние контроля и размещение для поддержки цели качества в мастерстве.

- Диалоги могут стать более сложными из большего числа элементов управления, уникального сопоставления элементов управления или и того, и другого. Для этих сложных ситуаций, позволяют достаточное пространство между контрольными группировками, чтобы дать пользователю логический поток для разбора.

### <a name="layout-details"></a>Детали планировки

#### <a name="margins"></a>Поля

- Все диалоги должны иметь 12-пиксельную границу по всем краям.

- Поля в групповом кадре должны быть на уровне 9 пикселей от края кадра.

- Поля в элементе управления вкладкой должны быть на 6 пикселей от края управления вкладкой.

#### <a name="command-buttons"></a>Кнопки командования

- Кнопки командной команды работают на рамке диалога, а не на содержимом. Они должны быть размещены в правом нижнем углу и должны иметь достаточно переменного пространства выше, чтобы установить кнопки отчетливо отдельно.

- Если в диалоге есть горизонтальные кнопки, то альтернативная конфигурация кнопки команды представляет собой вертикальный стек в правом верхнем углу. Смотрите [кнопки команды интерьера](../../extensibility/ux-guidelines/layout-for-visual-studio.md#BKMK_InteriorCommandButtons) ниже.

- Пространство слева от кнопок команды (в левом нижнем/ центре диалога) считается частью "полосы" управления работой диалога. Единственное, что должно вторгаться в это пространство, это ссылка справки, которая имеет отношение к общей задаче или диалогу.

- Кнопки команддолжных команд должны быть 75х23 пикселей.

- Кнопки команддолжных команд должны быть на 6 пикселей друг от друга.

  ![Выравнивание основных кнопок](../../extensibility/ux-guidelines/media/0801-c_buttonalign.png "0801-c_ButtonAlign")

  **Рисунок 08.01-c: Выравнивание базовой кнопки**

#### <a name="labels"></a>Метки

- Левые выровнять все этикетки.

- Для меток, которые сидят над элементом управления, они должны выровнять сявра именно с управлением под ним, а нижняя часть метки должна быть на 5 пикселей выше верхней части другого элемента управления (например, комбо-бокс).

- Для меток, которые расположены слева от элементов управления, минимальная ширина между меткой и элементом управления ввода составляет 10 пикселей. Подразумеваемый второй столбец должен быть установлен для выравнивания текстовых коробок, комбо-коробок или других элементов управления.

- Этикетки приговор случае и следуют толстой кишки. Смотрите [текстовый стиль](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md#BKMK_TextStyle).

#### <a name="distance-between-controls"></a>Расстояние между элементами управления
 Управление стеком разумно. Нет абсолютного ориентира для интервала между сложенными элементами управления. Герметичность между элементами управления может незначительно отличаться между диалогами. Рекомендуемое расстояние составляет 20 пикселей для вертикальных пар управления/метки и 9 пикселей для горизонтальных пар управления/метки. Минимальный интервал управления для горизонтальных пар составляет 6 пикселей.

#### <a name="control-indentation"></a>Контроль отступов
 Когда элементы управления вложены, выровнять внутренние элементы управления горизонтально с левым краем управления выше, как правило, этикетки.

#### <a name="control-width"></a>Ширина управления
 Ширина текстового поля или других аналогичных элементов управления должна быть не больше среднего ввода для поля. Среднее английское слово состоит из пяти символов. Например, текстовое поле, требующее длинного имени пути, должно быть до тех пор, пока позволяет горизонтальная компоновка, в то время как выпадение для названий платформы должно быть только длиной, которая позволяет для самого длинного входа.

#### <a name="helper-text"></a>Текст помощника

- Диалог может отображать текст помощника, который предоставляет больше информации о цели диалога. Это обычно сидит в верхней части и может быть 1-2 предложения.

- Длина линии должна быть удобной шириной для пользователя, чтобы разобрать и прочитать. Средний диалог должен быть не более 550 пикселей в ширину.

#### <a name="interior-command-buttons"></a><a name="BKMK_InteriorCommandButtons"></a>Кнопки командования интерьера
 В более сложных диалогах внутренний элемент управления может иметь свои собственные связанные кнопки, которые могут повлиять на то, где находятся кнопки коммита диалога.

- Используйте вертикальное выравнивание (колонку) внутренних кнопок, когда **OK**/**Cancel** горизонтально ориентированы в правом нижнем углу.

- Используйте горизонтальное выравнивание (ряд) внутренних кнопок, когда **OK**/**Cancel** вертикально ориентированы в правом верхнем углу. Такая ситуация встречается реже.

- Размер внутренней кнопки должен быть нацелен на стандартный размер кнопки 75x23 пикселей, соответствующий размерков кнопок **OK**/**Cancel,** когда это возможно. Если метка кнопки делает кнопку превышать стандартный размер кнопки, другие кнопки в этом наборе должны соответствовать этому более широкому размеру.

#### <a name="browse-button"></a>«Просмотрите...» Кнопку
 **«Просмотрите...»** кнопки, которые следуют текстовый ящик должен изложить "Обзор ..." в полном объеме, в том числе эллипсис. Если пространство плотно или есть несколько кнопок **«Просмотр...»** на экране, кнопка может быть уменьшена до просто эллипсиса.

## <a name="themed-dialog-layout"></a><a name="BKMK_ThemedDialogLayout"></a>Тематический макет диалога
 Тематические диалоги в Visual Studio имеют более легкий внешний вид и предлагают больше белого пространства. Типография обеспечивает больший акцент и интерес, предлагая более открытые интервалы между линиями и изменение размеров шрифтов и весов. Там, где это возможно, хром и титульные батончики были уменьшены или удалены. Расположение этих диалогов должно следовать этой основной схеме:

1. Фон диалога белый.

2. Граница правил 1-пикселя имеет серый среднего значения.

3. Название диалога больше не находится в заглавной панели, но обеспечивает визуальный интерес и акцент в большем размере точки. (См. раздел размер шрифта в [стиле Текст](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md#BKMK_TextStyle).)

4. Этикетки в сочетании с дополнительным текстом, таким как описание, должны быть **шрифтом окружающей среды и смелейшим**.

5. Внутренние колонны разделены 1-пиксельным правилом в светло-серый цвет.

6. Ссылки по умолчанию не подчеркивают. Нависают и нажатые состояния имеют изменение цвета плюс подчеркивайте.

7. Кнопки фиксации (например, **OK**/**Cancel)** сидят в правом нижнем углу.
