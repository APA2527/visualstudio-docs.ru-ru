# <a name="msbuild-properties-supported-by-sharepoint"></a>Свойства MsBuild, поддерживаемые SharePoint
  Любое свойство [!INCLUDE[vstecmsbuild](../sharepoint/includes/vstecmsbuild-md.md)], определенное в файле Microsoft. VisualStudio. SharePoint. targets, файле проекта или файле проекта, может использоваться в [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)] проектах SharePoint. В дополнение к общим свойствам [!INCLUDE[vstecmsbuild](../sharepoint/includes/vstecmsbuild-md.md)], предоставляемым проектом, SharePoint определяет дополнительные свойства, относящиеся к проектам SharePoint.

 Список общих свойств [!INCLUDE[vstecmsbuild](../sharepoint/includes/vstecmsbuild-md.md)] см. в разделе [Общие свойства проекта MSBuild](/previous-versions/dotnet/netframework-4.0/bb629394(v=vs.100)). Полный список свойств, поддерживаемых языком программирования, можно найти в файле *Targets* , файле проекта ( *. csproj* или *. vbproj*) или в файле пользователя проекта (*csproj. User* или *. vbproj. User*).

## <a name="msbuild-properties-specific-to-sharepoint"></a>Свойства MsBuild, относящиеся к SharePoint
 В следующей таблице перечислены [!INCLUDE[vstecmsbuild](../sharepoint/includes/vstecmsbuild-md.md)] свойства, применяемые к проектам SharePoint в [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)]. Существуют и другие свойства, но они предназначены для внутреннего использования.

|Имя свойства|Описание|
|-------------------|-----------------|
|шарепоинтситеурл|Строка, представляющая [!INCLUDE[TLA2#tla_url](../sharepoint/includes/tla2sharptla-url-md.md)] сайта SharePoint.|
|сандбокседсолутион|Логическое значение, указывающее, является ли решение изолированным решением.|
|активедеплойментконфигуратион|Активная конфигурация развертывания.|
|инклудеассемблинпаккаже|Логическое значение, указывающее, включена ли сборка в файл пакета.|
|предеплойменткомманд|Строковое значение, представляющее команду, выполняемую на шаге команды перед развертыванием.|
|постдеплойменткомманд|Строковое значение, представляющее команду, выполняемую на шаге команды после развертывания.|
|кустомбефорешарепоинттаржетс|Строка, представляющая путь к файлу целевых объектов [!INCLUDE[vstecmsbuild](../sharepoint/includes/vstecmsbuild-md.md)]. Если файл целей существует и определен, он импортируется перед любыми данными целевых объектов SharePoint. Это свойство позволяет настроить процесс пакета, предварительно определив свойства, связанные с упаковкой, не изменяя поставляемый файл целевых объектов SharePoint, но файл целей по-прежнему применяется ко всем проектам SharePoint.|
|кустомафтершарепоинттаржетс|Строка, представляющая путь к файлу целевых объектов [!INCLUDE[vstecmsbuild](../sharepoint/includes/vstecmsbuild-md.md)]. Если файл целей существует и определен, он импортируется после всех данных целевых объектов SharePoint. Это свойство позволяет настроить процесс пакета путем переопределения свойств и целевых объектов, связанных с упаковкой, без изменения поставляемого файла целевых объектов SharePoint, однако файл целевых объектов по-прежнему применяется ко всем проектам SharePoint.|
|LayoutPath|Строка, представляющая корневой каталог, в котором каждый из упакованных файлов временно помещается перед добавлением в *WSP* -файл. Этот путь может быть полезен при переопределении целевых объектов Бефорелайаут и Афтерлайаут для добавления, удаления или изменения файлов, которые должны быть упакованы, так как их можно использовать для изменения содержимого *WSP* -файла.|
|басепаккажепас|Строка, представляющая папку, в которую помещается пакет. Это значение использует выходной каталог проекта, например Бин\дебуг.|
|паккажеекстенсион|Строка, представляющая расширение имени файла, добавляемого к пакету. Значение по умолчанию — WSP.|
|ассемблидеплойменттаржет|Строка, представляющая расположение, в котором развернута сборка проекта на сервере SharePoint. Его значение — GlobalAssemblyCache (по умолчанию) или «приложение». Это свойство также может быть задано в окно свойств.|
|паккажевисвалидатион|Логическое значение, указывающее, выполняется ли проверка перед упаковкой. Это свойство позволяет пропускать ошибки проверки при построении пакетов.|
|валидатепаккажедепендсон|Строка, определяющая дополнительные целевые объекты для выполнения перед целевым объектом Валидатепаккаже.|
|токенреплацементфиликсенсионс|Строка, определяющая файлы, токены которых были заменены во время упаковки.|

## <a name="use-msbuild-properties-in-the-properties-page"></a>Использование свойств MsBuild на странице свойств
 Для гибкости вместо использования жестко заданных строк в полях Командная **строка перед развертыванием** и **Командная строка после развертывания** на странице свойств SharePoint можно использовать свойства SharePoint в качестве аргументов. Например, вместо указания конкретной строки [!INCLUDE[TLA2#tla_url](../sharepoint/includes/tla2sharptla-url-md.md)] для сайта SharePoint можно использовать `$(SharePointSiteUrl)`.

> [!NOTE]
> Для указания свойства можно использовать синтаксис переменной [!INCLUDE[vstecmsbuild](../sharepoint/includes/vstecmsbuild-md.md)] `$(`*propertyname*`)` или синтаксис переменной среды `%`*PropertyName*`%`.
