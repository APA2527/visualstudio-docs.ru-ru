# <a name="sandboxed-solution-considerations"></a>Замечания об изолированных решениях
  *Изолированные решения* — это функция в Microsoft SharePoint 2010, позволяет пользователям коллекции сайта отправлять свои собственные решения с пользовательским кодом. Распространенные изолированное решение — пользователи, отправка собственные веб-частей.

 Изолированное приложение SharePoint работает в безопасном, отслеживаемом процессе, который имеет доступ к ограниченной части веб-фермы. Microsoft SharePoint 2010 использует сочетание функций, коллекции решений, контроля решений и структуры проверки для включения изолированных решениях.

## <a name="specify-project-trust-level"></a>Укажите уровень доверия проекта
 [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)] поддерживает изолированные решения через свойство логическое проект называется *изолированное решение*. Это свойство можно задать в любое время в проекте, или могут быть указаны при создании проекта в **мастер настройки SharePoint**.

> [!NOTE]
> Изменение *изолированное решение* свойства проекта после его создания может вызвать ошибки проверки.

 Решение считается решением из области фермы, если *изолированное решение* свойству **false** или выбрать **развернуть как решение фермы** параметр. Тем не менее, решение обрабатывается по-разному решение фермы Если *изолированное решение* свойству **true** или выбрать **развернуть как изолированное решение** параметр в мастере.

## <a name="sharepoint-site-hierarchy"></a>Иерархия сайта SharePoint
 Чтобы понять, как изолированные решения работы, полезно знать, узлов SharePoint являются иерархическими в области видимости. Верхний элемент называется веб-фермы, а остальные элементы являются подчинен:

 Веб-фермы

 Веб-приложение A

 Семейство сайтов A1

 A1a сайта

 Веб-приложение Б

 Семейство сайтов B1

 B1a сайта

 B1b сайта

 Семейство сайтов B2

 Сайт B2a

 Как вы видите, веб-фермы может содержать один или несколько веб-приложений, которые в свою очередь, могут содержать один или несколько семейств веб-сайтов, которые могут иметь дочерние узлы и т. д. Изменения, внесенные в один узел коллекции затрагивают коллекции и никакие другие. Однако изменения, внесенные на уровне веб-фермы влияют на всех семейств веб-сайтов на ферме.

 Службы Windows SharePoint Services (WSS) 3.0 позволяет развертывать решения только на уровне фермы, но [!INCLUDE[wss_14_long](../sharepoint/includes/wss-14-long-md.md)] позволяет развертывать их на уровне фермы (решения фермы), так и на уровне семейства узлов (песочница).

## <a name="why-sandboxed-solutions"></a>Почему изолированные решения?
 В WSS 3.0 решения можно установить только на уровне фермы. Это означало, что потенциально вредоносных или нестабильных позволяют развертывать решения, затронутых всей веб-фермы и всех других семейств веб-сайтов и приложений, работающих под ним. Тем не менее с помощью изолированные решения, можно развернуть свои решения для дочерней области фермы, данного семейства узлов. Для дополнительной защиты решения сборка не загружается в главный [!INCLUDE[TLA2#tla_iis5](../sharepoint/includes/tla2sharptla-iis5-md.md)] процесса (*w3wp.exe*). Вместо этого он загружается в отдельном процессе (*SPUCWorkerProcess.exe*). Этот процесс отслеживается и реализует квоты и регулирование для защиты фермы из изолированных решениях, которые выполняют вредоносных действий, таких как запуск сплошных циклах, которые потребляют циклы ЦП.

## <a name="site-collection-solution-gallery"></a>Коллекция решений семейств сайтов
 В [!INCLUDE[sharepointShort](../sharepoint/includes/sharepointshort-md.md)] 2010 входит компонент под названием "коллекция решений семейств сайтов". Эту функцию можно открыть на странице центра администрирования SharePoint 2010 также можно открыть **действия сайта** меню, выбор **параметры сайта**, а затем выбрав **решения** ссылку в списке **галереи** на сайте SharePoint. Коллекции решений — это репозитории решений, которые позволяют Администраторы семейства сайтов для управления решениями в этих семействах сайтов.

 Коллекция решений — это библиотека документов, хранящихся в корневом веб-сайте SharePoint. В solution gallery заменяет шаблонов узлов и поддерживает пакеты решений. Когда пакета решения SharePoint (*.wsp*) отправить файл, он обрабатывается как изолированное решение.

## <a name="sandboxed-solution-limitations"></a>Ограничения изолированных решений
 При развертывании изолированного решения с целью сокращения всех уязвимостей системы безопасности, которые он может иметь ограничено массив функциональных возможностей SharePoint к нему. Ниже приведены некоторые из этих ограничений:

- Изолированные решения имеют ограниченное подмножество элементов развертываемых решений, доступным ему. Потенциально уязвимые шаблоны проектов SharePoint, такие как определения узла и рабочие процессы, недоступны.

- SharePoint выполняется код изолированного решения в процессе (*SPUCWorkerProcess.exe*) отдельно от главного [!INCLUDE[TLA2#tla_iis5](../sharepoint/includes/tla2sharptla-iis5-md.md)] пула приложений (*w3wp.exe*) процесс.

- Невозможно добавить сопоставленные папки в проект.

- Типы в [!INCLUDE[moss_14_long](../sharepoint/includes/moss-14-long-md.md)] сборки Microsoft.Office.Server не может использоваться в изолированных решениях. Кроме того, только типы из [!INCLUDE[wss_14_long](../sharepoint/includes/wss-14-long-md.md)] сборки Microsoft.SharePoint может использоваться в изолированных решениях.

  Важно отметить, что при указании решения SharePoint как изолированное решение не оказывает влияния на сервер SharePoint. оно только определяет, как проект SharePoint развертывается в SharePoint из [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)] и какие сборками он связывается. Он не влияет на созданного *.wsp* файл и *.wsp* файл не содержит данных, непосредственно связано с *изолированное решение* свойство.

## <a name="capabilities-and-elements-in-sandboxed-solutions"></a>Возможности и элементы в изолированных решениях
 Изолированные решения поддерживают следующие возможности и элементы:

- Поля и типы содержимого

- Настраиваемые действия

- Декларативные рабочие процессы

- приемники событий;

- Вызовы компонентов

- Определения списков

- Экземпляры списка

- Файлы/модулей

- Навигация

- *файл onet.XML*

- SPItemEventReceiver

- SPListEventReceiver

- SPWebEventReceiver

- Поддержка для всех веб-частей, которые являются производными от `System.Web.UI.WebControls.WebParts.WebPart`

- Веб-части

- Элементы компонента WebTemplate (а не *Webtemp.xml*)

- Визуальные веб-части

  Изолированные решения не поддерживают следующие возможности и элементы:

- Страницы приложения

- Настраиваемая группа действий

- Компоненты уровня фермы

- Элемент `HideCustomAction`

- Веб-приложения функции

- Рабочие процессы с кодом
