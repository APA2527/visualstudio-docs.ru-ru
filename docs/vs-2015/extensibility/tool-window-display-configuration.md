# <a name="tool-window-display-configuration"></a>Конфигурация отображения окна инструментов
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Когда VSPackage регистрирует окно инструментов, в необязательных значениях указывается положение по умолчанию, размер, стиль закрепления и другие сведения о видимости. Дополнительные сведения о регистрации окна инструментов см. в разделе [окна инструментов в реестре](../extensibility/tool-windows-in-the-registry.md) .  
  
## <a name="window-display-information"></a>Сведения о отображении окна  
 Базовая конфигурация экрана окна инструментов хранится в не более шести необязательных значениях:  
  
```  
HKEY_LOCAL_MACHINE\  
  Software\  
    Microsoft\  
      VisualStudio\  
        <Version>\  
          ToolWindows\  
            <Tool Window GUID>\  
              (Default)       = reg_sz: <Package GUID>Name            = reg_sz: <name of tool window>Float           = reg_sz: <position>Style           = reg_sz: <dock style>Window          = reg_sz: <window GUID>Orientation     = reg_sz: <orientation>DontForceCreate = reg_dword: 0x00000000  
```  
  
|Имя|Тип|Данные|Описание|  
|----------|----------|----------|-----------------|  
|Имя|REG_SZ|"Краткое имя здесь"|Краткое имя, описывающее окно инструментов. Используется только для справки в реестре.|  
|Float|REG_SZ|X1, Y1, X2, Y2|Четыре значения, разделенные запятыми. X1, Y1 — это координата левого верхнего угла окна инструментов. X2, Y2 — это координата правого нижнего угла. Все значения отображаются в экранных координатах.|  
|Стиль|REG_SZ|ИНТЕРФЕЙСА<br /><br /> Сделать<br /><br /> Установлен<br /><br /> С вкладками<br /><br /> "Алвайсфлоат"|Ключевое слово, указывающее начальное состояние отображения окна инструментов.<br /><br /> "MDI" = закреплено в окне MDI.<br /><br /> "Float" = плавающий.<br /><br /> "Linked" = связан с другим окном (указанное в записи окна).<br /><br /> "С вкладками" — вместе с другим окном инструментов.<br /><br /> "Алвайсфлоат" = невозможно закрепить.<br /><br /> Дополнительные сведения см. в разделе комментариев ниже.|  
|Окно|REG_SZ|*\<GUID>*|Идентификатор GUID окна, к которому может быть привязано окно инструментов, или вкладка. Идентификатор GUID может принадлежать к одному из окон или одному из окон в [!INCLUDE[vsprvs](../includes/vsprvs-md.md)] интегрированной среде разработки.|  
|Ориентация|REG_SZ|Слева<br /><br /> Справа<br /><br /> Лучшая<br /><br /> Нижний|См. раздел комментариев ниже.|  
|донтфорцекреате|REG_DWORD|0 или 1|Если эта запись имеется и ее значение не равно нулю, окно загружается, но не отображается сразу.|  
  
### <a name="comments"></a>Комментарии  
 Запись ориентации определяет положение, где окно инструментов закрепляется при двойном щелчке на строке заголовка. Это расположение относительно окна, указанного в записи окна. Если для записи стиля задано значение «Linked», то для записи ориентации может быть «Left», «Right», «Top» или «Bottom». Если элемент стиля имеет значение «с вкладками», то запись ориентации может иметь значение «Left» или «right» и указывает место добавления вкладки. Если элемент стиля имеет значение float, окно инструментов сначала будет перемещаться. При двойном щелчке строки заголовка применяются ориентация и элементы окна, а в окне используется стиль с вкладками. Если элемент стиля имеет значение «Алвайсфлоат», окно инструментов не может быть закреплено. Если элемент стиля имеет значение "MDI", окно инструментов связывается с областью MDI, а запись окна игнорируется.  
  
### <a name="example"></a>Пример  
  
```  
HKEY_LOCAL_MACHINE\  
  Software\  
    Microsoft\  
      VisualStudio\  
        8.0Exp\  
          ToolWindows\  
            {A0C5197D-0AC7-4B63-97CD-8872A789D233}\  
              (Default)       = reg_sz: {DA9FB551-C724-11D0-AE1F-00A0C90FFFC3}  
              DontForceCreate = reg_dword: 0x00000000  
              Float           = reg_sz: 100,100,450,300  
              Name            = reg_sz: Bookmarks  
              Orientation     = reg_sz: Left  
              Style           = reg_sz: Tabbed  
              Window          = reg_sz: {34E76E81-EE4A-11D0-00A0C90FFFC3}  
```  
  
## <a name="tool-window-visibility"></a>Видимость окна инструментов  
 Значения в необязательном подразделе видимости определяют параметры видимости окна инструментов. Имена значений используются для хранения идентификаторов GUID команд, которым требуется видимость окна. Если команда выполняется, интегрированная среда разработки гарантирует, что окно инструментов будет создано и сделано видимым.  
  
```  
HKEY_LOCAL_MACHINE\  
  Software\  
    Microsoft\  
      VisualStudio\  
        <Version>\  
          ToolWindows\  
            <Tool Window GUID>\  
              Visibility\  
                (Default) = reg_sz:  
                <GUID>    = reg_dword:  
                <GUID>    = reg_dword:  
                <GUID>    = reg_sz:  
```  
  
|Имя|Тип|Данные|Описание|  
|----------|----------|----------|-----------------|  
|(по умолчанию)|REG_SZ|Нет|Оставьте пустым.|  
|*\<GUID>*|REG_DWORD или REG_SZ|0 или описательную строку.|Необязательный элемент. Имя записи должно быть идентификатором GUID команды, для которой требуется видимость. Значение просто содержит информативную строку. Как правило, значением является значение `reg_dword` 0.|  
  
### <a name="example"></a>Пример  
  
```  
HKEY_LOCAL_MACHINE\  
  Software\  
    Microsoft\  
      VisualStudio\  
        8.0Exp\  
          ToolWindows\  
            {EEFA5220-E298-11D0-8F78-00A0C9110057}\  
              Visibility\  
                (Default) = reg_sz:  
                {93694fa0-0397-11d1-9f4e-00a0c911004f} = reg_dword: 0x00000000  
                {9DA22B82-6211-11d2-9561-00600818403B} = reg_dword: 0x00000000  
                {adfc4e66-0397-11d1-9f4e-00a0c911004f} = reg_dword: 0x00000000  
```
