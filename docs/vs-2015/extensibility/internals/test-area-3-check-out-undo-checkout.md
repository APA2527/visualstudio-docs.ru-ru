# <a name="test-area-3-check-outundo-checkout"></a>Область тестирования 3. Ознакомьтесь с / отменить извлечение
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

Эта область тестирования подключаемый модуль системы управления версиями охватывает редактирования и выполняется Отмена изменений элементов из хранилища версий с помощью **извлечь** и **отменить извлечение** команды.  
  
 **Ознакомьтесь с**: Элемент в хранилище версий, как извлечь отметок изменяет локальную копию для чтения и записи.  
  
 **Отменить извлечение**: Метки элемента в хранилище версий, как в базу данных, возвращается локальной копии состояния перед извлечением (в зависимости от параметров).  
  
## <a name="command-menu-access"></a>Доступ к меню команды  
 Следующие [!INCLUDE[vsprvs](../../includes/vsprvs-md.md)] в тестовых случаях используются пути меню в среде разработки.  
  
##### <a name="check-out"></a>Извлечь:  
  
- **Файл**, **система управления версиями**, **извлечь**.  
  
- **Файл**, **извлечь**.  
  
- Контекстное меню **извлечь**.  
  
- Отмена извлечения: **Файл**, **система управления версиями**, **отменить извлечение**.  
  
## <a name="common-expected-behavior"></a>Распространенное ожидаемое поведение  
  
- После извлечения операции удаляемые файлы или папки, помечаются как извлечен в хранилище версий.  
  
- Хранилище версий атрибутов извлечение соответствующему пользователю.  
  
- Время и Дата извлечения верны (согласно параметрам пользователя).

### <a name="case-3b-disconnected-check-out"></a>Case 3b. Отключенный извлечение  
 В отключенном режиме позволяет пользователям определенный уровень поддержки системы управления версиями постоянной, если не присоединен непосредственно в хранилище версий. Это делается путем локального кэширования вся информация о прикрепленной решение и проекты.  
  
 Монопольное извлечение операций могут возникать только в том случае, подключенных к системе управления версиями. Общий извлечение операций может произойти в любое время ли подключен или отключен. Таким образом при отключении от хранилища версий, только **проверки для совместного использования** (COS) команда включена. При отсутствии подключения, **отменить извлечение** отключено, так как невозможно получить старую версию, чтобы заменить изменения, внесенные пользователем.  
  
 При повторном подключении пользователя к версии хранения и извлечения состояния всех включенный в список решений и проектов, синхронизируются. Это так и для извлечения, которые пользователь выполнил необходимые обновления в хранилище. После синхронизации был выполнен, пользователь не сможет продолжить работу в обычном режиме (подключено).  
  
#### <a name="expected-behavior"></a>Ожидаемое поведение  
  
- Нельзя использовать **проверьте Out исключительно** команды при отсутствии подключения из хранилища версий.  
  
- Нельзя использовать **отменить извлечение** команды при отсутствии подключения из хранилища версий.  
  
- **Общие извлечения** работу команды.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|------------|----------------|--------------------------------|  
|При отсутствии подключения, извлечь файл, а затем соедините синхронизации|1.  Отключить управляемую проект с помощью диалогового окна Изменение системы управления версиями (**файл**, **системы управления версиями**, **система управления версиями изменений**l).<br />2. ### <a name="case-3c-query-editquery-save-qeqs"></a>Вариант 3c. Запрос редактирования/запроса сохранить (QEQS)  
 Элементы в системе управления версиями, отслеживаются для редактирования, изменения, и сохраняет, чтобы помочь пользователям легко управлять их файлы. При редактировании элемента управляемой, «установлен в» QEQS перехватывает попытки изменить и запрашивает у пользователя, если он хочет проверить его для редактирования. В зависимости от **средства**, **параметры** параметры пользователя — либо принудительно проверяемый выходной файл для изменения или можно разрешить изменить копию в памяти и проверить позже. Если пользователя **средства**, **параметры** задан параметр для отображения диалоговое окно извлечения и просто извлечь его, то когда пользователь делает его изменить, файл автоматически проверяет, когда это возможно.

### <a name="case-3e-undo-check-out"></a>Case 3Д. Отменить извлечение  
 **Отменить извлечение** используется для отмены файла проверяется состояние извлечения и избежать изменения, внесенные в файл.  
  
#### <a name="expected-behavior"></a>Ожидаемое поведение  
  
- Значение по умолчанию создается на основе пользователя **извлечь локальную версию** параметр. Если пользователь выбрал извлечь локальную версию, то по умолчанию операция отмены извлечения будет всегда вернуться к версии, извлечен.  
  
- После принятия отмены, значки в **обозревателе решений** обновлены для затронутых файлов и элемент удаляется из **Отложенные возвраты** окна.
