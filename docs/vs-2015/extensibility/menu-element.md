# <a name="menu-element"></a>Элемент Menu
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Определяет один пункт меню. Это шесть видов меню: контекст, меню, Менуконтроллер, Менуконтроллерлатчед, панель инструментов и Тулвиндовтулбар.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
<Menu guid=”guidMyCommandSet” id=”MyCommand” priority=”0x100” type=”button”>  
  <Parent>... </Parent>  
  <CommandFlag>... </CommandFlag>  
  <Strings>... </Strings>  
</Menu>  
```  
  
## <a name="attributes-and-elements"></a>Атрибуты и элементы  
 В следующих разделах описаны атрибуты, дочерние и родительские элементы.  
  
### <a name="attributes"></a>Атрибуты  
  
|Атрибут|Описание|  
|---------------|-----------------|  
|guid|Обязательный. Идентификатор GUID идентификатора команды GUID/ID.|  
|идентификатор|Обязательный. Идентификатор идентификатора команды GUID/ID.|  
|priority|Необязательный элемент. Числовое значение, указывающее относительное расположение меню в группе меню.|  
|тулбарприоритинбанд|Необязательный элемент. Числовое значение, указывающее относительное положение панели инструментов в полосе при закреплении окна.|  
|type|Необязательный элемент. Перечислимое значение, указывающее тип элемента.<br /><br /> Если он отсутствует, по умолчанию используется тип Menu.<br /><br /> Контекст<br /> Контекстное меню, отображаемое, когда пользователь щелкает правой кнопкой мыши по окну. Контекстное меню имеет следующие характеристики.<br /><br /> — Не использует поля родительский и приоритет, если меню отображается в виде контекстного меню.<br />— Может использоваться как подменю, а также как контекстное меню. В этом случае учитываются и идентификатор группы, и поля приоритета.<br />— Не всегда доступно.<br /><br /> Контекстное меню отображается только в том случае, если выполняются следующие условия.<br /><br /> — Отображается окно, в котором он размещен.<br />— Обработчик мыши в VSPackage обнаруживает щелчком правой кнопкой мыши по окну, а затем вызывает метод, обрабатывающий команду.<br />— Контекстное меню отображается путем вызова <xref:Microsoft.VisualStudio.Shell.Interop.IOleComponentUIManager.ShowContextMenu%2A> метода (рекомендуемый подход) или <xref:Microsoft.VisualStudio.Shell.Interop.IVsUIShell.ShowContextMenu%2A> метода.<br /><br /> Меню<br /> Предоставляет раскрывающееся меню. Раскрывающееся меню имеет следующие характеристики.<br /><br /> — Учитывает родительский элемент в его определении.<br />— Должен иметь родительскую группу или CommandPlacement для группы.<br />— Может быть подменю любого другого вида меню.<br />— Автоматически отображается при отображении его родительского меню.<br />— Не требует реализации какого-либо кода VSPackage, чтобы сделать его отображаемым.<br /><br /> менуконтроллер<br /> Предоставляет раскрывающееся меню с разворачивающимися кнопками, которое обычно используется на панелях инструментов. Меню Менуконтроллер имеет следующие характеристики.<br /><br /> — Должен содержаться в другом меню через родительский объект или CommandPlacement.<br />— Учитывает родительский элемент в его определении.<br />— Может иметь любое меню в качестве родительского элемента.<br />— Автоматически предоставляется при каждом отображении родительского меню.<br />— Не требует программной поддержки для отображения меню.<br /><br /> Команда из меню разбиения на кнопки отображается на кнопке меню. Отображаемая команда имеет одну из следующих характеристик.<br /><br /> — Это последняя команда, которая использовалась, если команда все еще отображается и включена.<br />— Это Первая отображаемая команда.<br /><br /> менуконтроллерлатчед<br /> Предоставляет раскрывающееся меню кнопок разделения, для которого команда может быть указана по умолчанию путем пометки команды как кратковременной.<br /><br /> Кратковременная команда — это команда, которая отмечена в меню как выбранная, как правило, с галочкой. Команда может быть помечена как кратковременная, если для нее установлен флаг OLECMDF_LATCHED в реализации `QueryStatus` метода <xref:Microsoft.VisualStudio.OLE.Interop.IOleCommandTarget> интерфейса. Меню Менуконтроллерлатчед имеет следующие характеристики.<br /><br /> — Должно находиться в другом меню через родительскую группу или CommandPlacement.<br />— Учитывает родительский элемент в его определении.<br />— Может иметь любое меню в качестве родительского элемента.<br />— Становится доступным при каждом отображении родительского меню.<br />— Не требует программной поддержки для отображения меню.<br /><br /> Команда из меню разбиения на кнопки отображается на кнопке меню. Отображаемая команда имеет одну из следующих характеристик.<br /><br /> — Это Первая отображаемая команда, которая заблокируется.<br />— Это Первая отображаемая команда.<br /><br /> Панель инструментов<br /> Предоставляет панель инструментов. Панель инструментов имеет следующие характеристики.<br /><br /> — Пропускает родительский элемент в его определении.<br />— Нельзя сделать подменю ни одной группы, даже с помощью CommandPlacement.<br />— Всегда может отображаться, если в меню **вид** выбрать пункт **панели инструментов** .<br />— Может отображаться с помощью [висибилититем](../extensibility/visibilityitem-element.md).<br />— Не требует какого-либо кода создавать его. Пример создания панели инструментов см. в разделе [Добавление панели инструментов](../extensibility/adding-a-toolbar.md).<br /><br /> тулвиндовтулбар<br /> Предоставляет панель инструментов, присоединенную к конкретному окну инструментов, так же как и панель инструментов, присоединенная к среде разработки.<br /><br /> — Пропускает родительский элемент в его определении.<br />— Нельзя сделать подменю ни одной группы, даже с помощью CommandPlacement.<br />— Отображается, только если отображается окно инструментов, в котором размещается панель, и пакет VSPackage явно добавляет панель инструментов в окно инструментов. Обычно это делается при создании окна инструментов путем получения свойства узла панели инструментов (представленного <xref:Microsoft.VisualStudio.Shell.Interop.IVsToolWindowToolbarHost> интерфейсом) из рамки окна инструментов и последующего вызова <xref:Microsoft.VisualStudio.Shell.Interop.IVsToolWindowToolbarHost.AddToolbar%2A> метода.|  
|Условие|Необязательный элемент. См. раздел [Условные атрибуты](../extensibility/vsct-xml-schema-conditional-attributes.md).|  
  
### <a name="child-elements"></a>Дочерние элементы  
  
|Элемент|Описание|  
|-------------|-----------------|  
|Parent|Необязательный элемент. Родительский элемент пункта меню.|  
|CommandFlag|Обязательный. См. раздел [флаг команды](../extensibility/command-flag-element.md). Допустимые значения CommandFlag для меню:<br /><br /> -   **AlwaysCreate**<br />-   **дефаултдоккед**<br />-   **Дефаултинвисибле** — этот флаг не влияет на отображение панелей инструментов.<br />-   **донткаче**<br />-   **Динамиквисибилити** — этот флаг не влияет на отображение панелей инструментов.<br />-   **иконандтекст**<br />-   **Не настраивать**<br />-   **нотинтблист**<br />-   **нотулбарклосе**<br />-   **текстчанжес**<br />-   **текстисанчоркомманд**|  
|Строки|Обязательный. См. [элемент strings](../extensibility/strings-element.md). Необходимо определить дочерний `ButtonText` элемент.|  
|Заметка|Необязательный комментарий.|  
  
### <a name="parent-elements"></a>Родительские элементы  
  
|Элемент|Описание|  
|-------------|-----------------|  
|[Элемент Menus](../extensibility/menus-element.md)|Определяет все меню, которые реализует VSPackage.|  
  
## <a name="example"></a>Пример  
  
```  
<Menu guid="cmdGuidWidgetCommands" id="menuIDEditWidget"  
  priority="0x0002" type="Menu">  
  <Parent guid="cmdSetGuidWidgetCommands" id="groupIDFileEdit">  
    <CommandFlag>AlwaysCreate</CommandFlag>  
    <Strings>  
      <ButtonText>Edit Widget</ButtonText>  
    </Strings>  
    </Parent>  
</Menu>  
```
