# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>Изменение изолированной оболочки с помощью файла .Pkgdef
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Файл pkgdef поддерживает параметры, которые можно использовать для настройки изолированного приложения оболочки. В нем указываются значения, которые создаются при установке приложения на компьютере и на которые ссылается оболочка Visual Studio при запуске приложения. Параметры упорядочены в файле на основе соответствующих разделов реестра.  

> [!WARNING]
> Обратите внимание, что файлы. pkgdef, не объявленные в файле. vsixmanifest пакета VSPackage, не проверяются при запуске Visual Studio.  

 Pkgdef-файл содержит разделы, каждый из которых идентифицируется ключом `[$RootKey$]` или `[$RootKey$\` *подразделом* `]` , где $RootKey $ является корневым ключом для приложения.  

 Каждый раздел содержит пары "имя-значение", имеющие следующий формат: `"` *valueName* `"=` *Value*.  

 Значения — это либо строка, заключенная в кавычки, либо 32-разрядное целое число, представленное в виде типа DWORD. Значения имеют следующие ограничения.  

- Все значения DWORD имеют шестнадцатеричный формат, например `dword:00000001` .  

   Для логических значений 1 представляет true, а 0 — значение false.  

- Все строки GUID имеют формат реестра, например `"{00000000-0000-0000-0000-000000000000}"` .  

- Все локализуемые идентификаторы ресурсов имеют форму "@*ResourceId*" или "#*ResourceId*", где *ResourceId* — идентификатор ресурса в пакете пользовательского интерфейса приложения, например `"@102"` . Пакет пользовательского интерфейса приложения — это пакет, на который ссылается параметр Апплокализатионпаккаже.  

  Например, примененная к объекту директива  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 Комментарии можно добавить в файл pkgdef. Однострочный комментарий содержит две косые черты в качестве первых двух символов.  

 Список строк подстановки см [. в разделе строки подстановки, используемые в. Pkgdef и. Pkgundef файлы](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md).  

 В следующих разделах описываются конкретные значения реестра, влияющие на поведение оболочки Visual Studio в изолированном режиме. Кроме того, в этом файле можно определить дополнительные значения реестра для приложения.  

> [!NOTE]
> Если параметр не указан в pkgdef-файле, соответствующая запись в реестре не вносится.

## <a name="binding-path-settings"></a>Параметры пути привязки  
 Раздел [$RootKey $ \Биндингпасс \\ {00000000-0000-0000-0000-000000000000} ] содержит список каталогов, которые оболочка проверяет для сборок. Эти каталоги добавляются в список каталогов, которые оболочка проверяет для закрытых сборок приложения.  

 По умолчанию в pkgdef-файл не добавляются записи о пути привязки. Однако следующие подкаталоги каталога установки Visual Studio автоматически добавляются в список привязки приложений в реестре.  

- Common7\IDE  

- Common7\IDE \\ \приватеассемблиес  

- Common7\IDE \\ \публикассемблиес  

  Чтобы добавить каталог к пути привязки, добавьте запись в формате "*директоринаме*" = "", где *директоринаме* — это абсолютный путь. Например, примененная к объекту директива  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```

## <a name="package-menu-item-settings"></a>Параметры пункта меню "пакет"  
 Раздел реестра [$RootKey $ \Менус] определяет файлы ресурсов пользовательского интерфейса для приложения.  

 Значения пунктов меню имеют форму "{*всуипаккажегуид*}" = ", *resourceId*, *versionNumber*", где *всуипаккажегуид* является идентификатором GUID пакета ПОЛЬЗОВАТЕЛЬСКОГО интерфейса приложения, *resourceId* — идентификатор ресурса CTMENU ресурса, содержащего элементы пользовательского интерфейса, а *versionNumber* — номер виртуальной версии для ресурса CTMENU. Дополнительные сведения см. в разделе [Регистрация обработчиков команд сборки взаимодействия](../extensibility/internals/registering-interop-assembly-command-handlers.md).  

 По умолчанию запись пункта меню создается в pkgdef-файле для пакета пользовательского интерфейса приложения.  

 Для каждого пакета, предоставляющего пункты меню и распространяемых в составе приложения, добавьте запись пункта меню для пакета.
