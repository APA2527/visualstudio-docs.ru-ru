# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>Изменение изолированной оболочки с помощью. Файл pkgdef
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Файл pkgdef поддерживает параметры, которые можно использовать для настройки приложений изолированной оболочки. Он указывает значения, которые создаются при установке приложения на компьютере и которые ссылается оболочки Visual Studio при запуске приложения. Параметры организованы в файле, в соответствии с применимым разделам реестра.  

> [!WARNING]
> Обратите внимание на то, что файлах pkgdef, которые не объявлены в файле VSIXMANIFEST VSPackage не проверяются при запуске Visual Studio.  

 Файл pkgdef содержит разделы, каждый из которых идентифицируется по ключу, либо `[$RootKey$]` или `[$RootKey$\` *подраздел*`]`, где $RootKey$ — это корневой ключ для приложения.  

 Каждый раздел содержит пары "имя значение", которые имеют следующий формат: `"`*ValueName*`"=`*значение*.  

 Значения: строкой, заключенной в кавычки, или 32-разрядное целое число, которое представляется как dword. Значения имеют следующие ограничения:  

- Все значения dword заключаются в шестнадцатеричном формате, например `dword:00000001`.  

   Для логических значений 1 представляет значение true, а 0 соответствует false.  

- Все строки GUID, находятся в формате реестра, например, `"{00000000-0000-0000-0000-000000000000}"`.  

- Все идентификаторы локализуемый ресурс, имеют форму «@*resourceID*"или «#*resourceID*«, где *resourceID* указывается идентификатор ресурса в пакете приложения пользовательского интерфейса, например `"@102"`. Пакет пользовательского интерфейса приложения — пакет, который указывается в параметре AppLocalizationPackage.  

  Например, примененная к объекту директива  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 Можно добавить комментарии в pkgdef-файл. Однострочный комментарий имеет две косые черты, что первые два символа.  

 Список строки подстановки, см. в разделе [использовать строки подстановки в. Pkgdef и. Файлы Pkgundef](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md).  

 В следующих разделах описаны определенные разделы реестра, влияющие на поведение оболочки Visual Studio в изолированном режиме. В этом файле можно также определить дополнительными значениями реестра для приложения.  

> [!NOTE]
> Если параметр не указан в pkgdef-файл, выполняется без соответствующей записи в реестре.

## <a name="binding-path-settings"></a>Параметры пути привязки  
 [$RootKey$ \BindingPaths\\{00000000-0000-0000-0000-000000000000}] ключ содержит список каталогов, оболочка проверяет наличие сборки. Эти каталоги добавляются в список каталогов, оболочка проверяет наличие закрытых сборок для приложения.  

 По умолчанию без пути привязки элементов добавляются в pkgdef-файл. Тем не менее следующие подкаталоги каталога установки Visual Studio автоматически добавляются в список пути привязки приложений в реестре.  

- Common7\IDE\  

- Common7\IDE\\\PrivateAssemblies  

- Common7\IDE\\\PublicAssemblies  

  Чтобы добавить каталог путь привязки, добавьте запись в формате "*имя_каталога*«=»», где *имя_каталога* является абсолютным путем. Например, примененная к объекту директива  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```

## <a name="package-menu-item-settings"></a>Параметры элемента меню пакета  
 Раздел реестра [$RootKey$ \Menus] определяет файлы ресурсов пользовательского интерфейса для приложения.  

 Значений пунктов меню имеют форму «{*vsUiPackageGuid*} «=» *resourceId*, *versionNumber*«, где *vsUiPackageGuid* – идентификатор GUID пакет пользовательского интерфейса приложения, *resourceId* указывается идентификатор ресурса CTMENU ресурса, который содержит элементы пользовательского интерфейса, и *versionNumber* — это номер версии виртуального для CTMENU ресурс. Дополнительные сведения см. в разделе [регистрации обработчиков команд сборки взаимодействия](../extensibility/internals/registering-interop-assembly-command-handlers.md).  

 По умолчанию в pkgdef-файл для пакета приложения пользовательского интерфейса создается запись элемента меню.  

 Для каждого пакета, предоставляющий элементы меню и распространяемом как часть приложения добавьте запись элемента меню пакета.
