# <a name="options-text-editor-cc-formatting"></a>"Параметры", "Текстовый редактор", C/C++, "Форматирование"
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

Позволяет изменять поведение по умолчанию редактора кода при написании программ на языках C и C++.

 Чтобы открыть эту страницу, в диалоговом окне **Параметры** в левой области разверните **Текстовый редактор**, **C/C++** и щелкните **Форматирование**.

> [!NOTE]
> Отображаемые на компьютере имена или расположения некоторых элементов пользовательского интерфейса Visual Studio могут отличаться от указанных в следующих инструкциях. Это зависит от имеющегося выпуска Visual Studio и используемых параметров. Дополнительные сведения см. в статье [Настройка параметров разработки в Visual Studio](https://msdn.microsoft.com/22c4debb-4e31-47a8-8f19-16f328d7dcd3).

## <a name="cc-options"></a>Параметры C/C++
 **Включить автоматические подсказки с краткими сведениями** Включает или отключает функцию IntelliSense для быстрой информации.

## <a name="inactive-code"></a>Неактивный код
 **Показывать неактивные блоки кода** Код, который неактивен из-за объявлений `#ifdef`, расцветом отличается для упрощения его определения.

 **Отключить непрозрачность неактивного кода** Неактивный код можно определить с помощью цвета, а не прозрачности.

 **Процент прозрачности неактивного кода** Степень непрозрачности для неактивных блоков кода может быть настроена.

## <a name="indentation"></a>Indentation;
 **Отступ для фигурных скобок** Можно настроить способ согласования фигурных скобок при нажатии клавиши ВВОД после начала блока кода, например функции или цикла `for`. Фигурные скобки можно выровнять по первому знаку блока кода или с отступом.

 **Автоматический отступ на вкладке** Вы можете настроить, что происходит с текущей строкой кода при нажатии клавиши TAB. Строка смещается на шаг отступа или вставляется знак табуляции.

## <a name="miscellaneous"></a>Прочее
 **Перечисление комментариев в окне Список задач** Редактор может проверять файлы с открытым исходным кодом на наличие предустановленных слов в комментариях. Он создает запись в окне **Список задач** для любых обнаруженных ключевых слов.

 **Выделить совпадающие токены** Если курсор находится рядом с фигурной скобкой, редактор может выделить парную фигурную скобку, чтобы можно было легко увидеть содержащийся в ней код.

## <a name="outlining"></a>Структуризация
 **Вход в режим структуры при открытии файлов** При переводе файла в текстовый редактор можно включить функцию структурирования. Дополнительные сведения см. в разделе [Структура](../../ide/outlining.md). Если выбран этот параметр, возможность структурирования будет включаться при открытии файла.

 **Автоматическое структурирование блоков #pragma регионов** Если выбран этот параметр, включается автоматическое структурирование [директив pragma](https://msdn.microsoft.com/library/9867b438-ac64-4e10-973f-c3955209873f) . Это позволяет разворачивать и сворачивать блоки области директивы pragma в режиме создания структуры.

 **Автоматическое структурирование блоков операторов** Если выбран этот параметр, автоматическое структурирование включается для следующих конструкций операторов:

- [if-else](https://msdn.microsoft.com/library/d9a1d562-8cf5-4bd4-9ba7-8ad970cd25b2)

- [Оператор switch (C++)](https://msdn.microsoft.com/library/6c3f3ed3-5593-463c-8f4b-b33742b455c6)

- [Оператор while (C++)](https://msdn.microsoft.com/library/358dbe76-5e5e-4af5-b575-c2293c636899)
