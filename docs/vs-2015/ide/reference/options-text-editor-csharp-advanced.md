# <a name="options-text-editor-c-advanced"></a>"Параметры", "Текстовый редактор", C#, "Дополнительно"
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

Это диалоговое окно позволяет изменять параметры для форматирования в редакторе, рефакторинга кода и комментариев XML-документации в Visual C#. Для доступа к этому диалоговому окну щелкните элемент **Параметры** в меню **Сервис**, разверните папки **Текстовый редактор** и **C#** , а затем выберите **Дополнительно**.

> [!NOTE]
> Отображаемые диалоговые окна и команды меню могут отличаться от описанных в справке в зависимости от текущих параметров или выпуска. Чтобы изменить параметры, выберите в меню **Сервис** пункт **Импорт и экспорт параметров** . Дополнительные сведения см. в статье [Настройка параметров разработки в Visual Studio](https://msdn.microsoft.com/22c4debb-4e31-47a8-8f19-16f328d7dcd3).

## <a name="outlining"></a>Структуризация
 Введите режим структуры при открытии файлов при выборе, автоматически определяет файл кода, который создает свертываемые блоки кода. При первом открытии файла блоки #regions и неактивные блоки кода свертываются.

## <a name="editor-help"></a>Справка по редактору
 Ошибки подчеркивания в редакторе идентифицируют ошибки сборки в коде. При выборе этого параметра появляются волнистые линии разных цветов, которые имеют особое значение:

- Красный — ошибки синтаксического анализа.

- Синий — ошибки сборки.

- Зеленый — предупреждения сборки.

- Недопустимые изменения [Изменить и продолжить](../../debugger/edit-and-continue.md) выделяются сиреневым цветом.

  Наведите указатель на подчеркнутый сегмент кода, чтобы увидеть подсказку со сведениями об ошибке.

  Показывать динамические семантические ошибки определяет определенные ошибки компиляции без явной компиляции, например объявление и использование неизвестного типа или ссылка на неизвестное свойство.

  Выделение ссылок на символ под курсором при размещении курсора внутри символа или при щелчке символа все экземпляры этого символа в файле кода выделяются.

## <a name="refactoring"></a>Рефакторинг
 Проверка результатов рефакторинга открывает диалоговое окно **результаты проверки** при попытке выполнить рефакторинг кода, содержащего ошибки сборки, или если рефакторинг кода вызовет привязку к чему-то другому из исходной привязки.

 Предупреждать о членах с созданными компилятором ссылка отображает диалоговое окно с предупреждением при попытке выполнить рефакторинг элемента с тем же именем, что и ссылка, созданная компилятором.
