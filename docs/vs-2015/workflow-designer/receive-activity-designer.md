# <a name="receive-activity-designer"></a>Конструктор действия Receive
Для создания и настройки <xref:System.ServiceModel.Activities.Receive> действия используется разработчик активности **Receive.** Действие <xref:System.ServiceModel.Activities.Receive> получает сообщение - либо встроенное, например <xref:System.ServiceModel.Channels.Message>, <xref:System.IO.Stream> или <xref:System.Xml.Linq.XElement>, либо определяемое контрактом данных приложения, контрактом сообщений или классом XML, поддерживающим сериализацию.

## <a name="the-receive-activity"></a>Действие Receive
 Действие <xref:System.ServiceModel.Activities.Receive> может получать один или несколько элементов в зависимости от используемого типа приема содержимого. Действие <xref:System.ServiceModel.Activities.SendReply> может быть привязано к действию <xref:System.ServiceModel.Activities.Receive>, которое получает сообщение в процессе обмена сообщениями по шаблону «запрос-ответ» на стороне службы.

### <a name="using-the-receive-activity-designer"></a>Использование конструктора действия Receive
 Дизайнер активности **Receive** можно найти в категории **«Сообщения»** **в наборе инструментов,** доступ к которой можно получить, нажав на вкладку **Toolbox** на [!INCLUDE[wfd2](../includes/wfd2-md.md)] (Альтернативно выберите панель **инструментов** из меню **View** или CTRL-ALT-X.)

 Конструктор активности **Receive** может быть вытащен из [!INCLUDE[wfd2](../includes/wfd2-md.md)] **Toolbox** и высажен на поверхность везде, где обычно размещаются действия. При этом создается действие <xref:System.ServiceModel.Activities.Receive> со значением по умолчанию <xref:System.Activities.Activity.DisplayName%2A> для Receive. Он <xref:System.Activities.Activity.DisplayName%2A> может быть отредактирован в заголовке дизайнера активности **Receive** или в поле **DisplayName** сетки свойств.

 Чтобы создать <xref:System.ServiceModel.Activities.SendReply> действие и привязать <xref:System.ServiceModel.Activities.Receive> его к выбранной деятельности, нажмите правой кнопкой **кнопку Получить** деятельности дизайнера, нажмите **На создание SendReply** пункт в контекстном меню и **SendReplyToReceive** дизайнер появляется ниже **Получить** дизайнера. Действие <xref:System.ServiceModel.Activities.SendReply> отправляет ответное сообщение в процессе обмена сообщениями по шаблону «запрос-ответ» на стороне службы. Он может быть настроен с **конструктором SendReplyToReceive.**

 Кроме того, конструктор шаблона **ReceiveAndSendReply** в категории **«Сообщения»** **Toolbox** может быть <xref:System.ServiceModel.Activities.Receive> <xref:System.ServiceModel.Activities.SendReply> использован для создания пары предварительно настроенных и активности. [!INCLUDE[crabout](../includes/crabout-md.md)]использование шаблона **ReceiveAndSendReply** и **SendReplyToReceive** [ReceiveAndSendReply](../workflow-designer/receiveandsendreply-template-designer.md) см.
