# <a name="c6308"></a>C6308
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Предупреждение C6308: «realloc» может возвратить пустой указатель: Присвоение пустого указателя \<переменной >, который передается в качестве аргумента «realloc», приведет к утечке исходного блока памяти  
  
 Это предупреждение указывает на утечку памяти, которая является результатом неправильного использования функции перераспределения. Функции перераспределения кучи не освобождать переданного буфера, если перераспределение завершится неудачно. Чтобы устранить неполадку, присвоить его результат функции перераспределения во временный и затем заменить исходный указатель после успешного перераспределения.  
  
## <a name="example"></a>Пример  
 Это предупреждение возникает в следующем примере кода:  
  
```cpp  
#include <malloc.h>  
#include <windows.h>  
  
void f( )  
{  
  char *x;  
  x = (char *) malloc(10);  
  if (x != NULL)  
  {  
    x = (char *) realloc(x, 512);  
    // code...  
    free(x);  
  }     
}  
```  
  
 Чтобы устранить это предупреждение, используйте следующий код:  
  
```cpp  
#include <malloc.h>  
#include <windows.h>  
  
void f()  
{  
  char *x, *tmp;  
  
  x = (char *) malloc(10);  
  
  if (x != NULL)  
  {  
    tmp = (char *) realloc(x,512);  
    if (tmp != NULL)   
    {  
      x = tmp;  
    }  
    free(x);  
  }  
}  
```  
  
 Это предупреждение может привести шума, если буфер к--перераспределить действующего псевдонима во время присвоения результата функции перераспределения.  
  
 Чтобы избежать возникновения проблемы такого рода, используйте механизмы, предоставляемые по C++ стандартной библиотеки шаблонов (STL). К ним относятся [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a), и [вектор](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Дополнительные сведения см. в разделе [интеллектуальные указатели](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) и [стандартной библиотеки C++](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
