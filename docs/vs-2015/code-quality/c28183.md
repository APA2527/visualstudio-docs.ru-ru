# <a name="c28183"></a>C28183
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Warning C28183: аргумент может иметь одно значение и является копией значения, найденного в указателе  
  
 Это предупреждение означает, что это значение является неожиданным в текущем контексте. Это предупреждение обычно появляется, когда `NULL` значение передается в качестве аргумента в функцию, которая не допускают ее. Значение фактически найдено в указанной переменной, а аргумент является копией этой переменной.  
  
 Средство анализа кода сообщает об этом предупреждении на первой точке, где он может точно определить, что указатель `NULL` или содержит недопустимое значение. Однако часто возникает ошибка, которая может произойти ранее в трассировке. В этом случае средство анализа кода также выдает номер строки первого возможного экземпляра, обычно в месте, где не удается точно определить, что предупреждение является соответствующим. В таких случаях к предупреждающее сообщение добавляется предыдущее расположение, где это может произойти. Как правило, изменение кода должно выполняться по номеру строки или до него, а не на момент отчета.  
  
## <a name="example"></a>Пример  
 В следующем примере средство анализа кода определяет, что в операторе *s* имеет значение NULL `if` , а сам текст `if` принимается. Указатель *s* затем присваивается *t* , а затем *t* используется способом, где `NULL` может возникнуть разыменование.  
  
```  
#include <windows.h>  
  
int fun2(char *s)  
{  
    char *t;  
    if (s == NULL) {  
       //... but s is unchanged   
    }  
  
    t = s;  
  
    return lstrlenA(t);  
}  
```  
  
 В этом примере кода средство анализа кода сообщает следующее предупреждение:  
  
 **d:\sample\testfile.cpp (38): предупреждение C28183; "t" может быть "0" и копией значения, приведенного в "s": это не соответствует спецификации для функции 'lstrlenA'.: Строки: 31, 32, 36, 38**
