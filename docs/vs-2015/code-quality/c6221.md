# <a name="c6221"></a>C6221
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Предупреждение C6221: Неявное приведение между семантически различными целочисленными типами: сравнение HRESULT с целым. Попробуйте вместо этого использовать макрос SUCCEEDED или FAILED  
  
 Это предупреждение указывает, что `HRESULT` сравнивается в целое число, отличное от нуля. Об успешном выполнении в `HRESULT` (`S_OK`) представляется 0. Поэтому неявное приведение из `HRESULT` будет создавать неверное значение в целое число и может привести к неверному результату. Это часто случается с по ошибке, что функция возвращать целое число, если он фактически возвращает `HRESULT`.  
  
## <a name="example"></a>Пример  
 В следующем коде создается данное предупреждение, сравнивая `HRESULT` с целым числом:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == 4)  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Чтобы устранить это предупреждение, в следующем коде используется `FAILED` макрос:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Для данного предупреждения `SCODE` эквивалентен типу `HRESULT`.  
  
 Дополнительные сведения см. в разделе [макрос SUCCEEDED](http://go.microsoft.com/fwlink/?LinkId=92738) и [макрос FAILED](http://go.microsoft.com/fwlink/?LinkId=180875).  
  
 Обратите внимание, что использование из malloc и free (и связанных динамическое выделение памяти API-интерфейсы) множество ловушек с точки зрения утечки памяти и исключения. Чтобы избежать такого рода утечки и исключение проблемы, используйте механизмы, предоставляемые по C++ стандартной библиотеки шаблонов (STL). К ним относятся [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a), и [вектор](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Дополнительные сведения см. в разделе [интеллектуальные указатели](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) и [стандартной библиотеки C++](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
