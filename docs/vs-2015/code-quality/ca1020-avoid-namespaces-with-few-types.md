# <a name="ca1020-avoid-namespaces-with-few-types"></a>CA1020. Не используйте пространства имен с несколькими типами
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

|||
|-|-|
|TypeName|AvoidNamespacesWithFewTypes|
|CheckId|CA1020|
|Категория|Microsoft.Design|
|Критическое изменение|Критическое|

## <a name="cause"></a>Причина
 Пространство имен, отличном от глобального пространства имен содержит меньше пяти типов.

## <a name="rule-description"></a>Описание правила
 Убедитесь, что каждый из используемых пространств имен имеет логическую организацию, и что существует веская причина для помещения типов в разреженное пространство имен. Пространства имен должен содержать типы, которые используются совместно в большинстве сценариев. Когда их приложения являются взаимоисключающими, типы должна находиться в отдельном пространстве имен. Например <xref:System.Web.UI> пространство имен содержит типы, используемые в веб-приложениях и <xref:System.Windows.Forms> пространство имен содержит типы, используемые в [!INCLUDE[TLA#tla_mswin](../includes/tlasharptla-mswin-md.md)]-приложений на основе. Несмотря на то, что оба пространства имен содержат типы, управляющие аспектов пользовательского интерфейса, эти типы не предназначены для использования в одном приложении. Таким образом они находятся в отдельных пространствах имен. Организовывать пространства имен можно также полезно, так как он возрастает понятность функций. При проверке иерархии пространств имен, потребители библиотеки должны иметь возможность найти необходимые типы, реализующие компонент.

> [!NOTE]
> Типы во время разработки и разрешения не должен быть объединен в других пространствах имен в соответствии с этой рекомендации. Эти типы принадлежат собственные пространства имен ниже основного пространства имен и пространства имен должна завершиться в `.Design` и `.Permissions`, соответственно.

## <a name="how-to-fix-violations"></a>Устранение нарушений
 Чтобы устранить нарушение этого правила, попробуйте объединить пространств имен, содержащих несколько типов в одном пространстве имен.

## <a name="when-to-suppress-warnings"></a>Отключение предупреждений
 Его можно безопасно подавить предупреждение из этого правила, если пространство имен содержит типы, используемые с типами из других пространств имен.
