# <a name="manage-exceptions-with-the-debugger-in-visual-studio"></a>Управление исключениями с помощью отладчика в Visual Studio

Исключение указывает на состояние ошибки, возникающее при выполнении программы. Можно указать отладчику, какие исключения или наборы исключений следует прерывать, и в какой момент нужно, чтобы отладчик был прерван (то есть приостанавливать в отладчике). Когда отладчик прерывает работу, он показывает, где было создано исключение. Можно также добавлять или удалять исключения. Откройте решение в Visual Studio, используя **параметры отладки > Windows > исключения** , чтобы открыть окно **параметры исключений** .

Предоставление обработчиков, реагирующих на наиболее важные исключения. Сведения о том, как добавить обработчики для исключений, см. в разделе [Устранение ошибок C# путем написания лучшего кода](../debugger/write-better-code-with-visual-studio.md). Кроме того, Узнайте, как настроить отладчик для постоянного прерывания выполнения некоторых исключений.

При возникновении исключения отладчик записывает его сообщение в окно **Вывод**. Это может привести к нарушению выполнения в следующих случаях:

- Создается исключение, которое не обрабатывается.
- Отладчик настроен на прерывание выполнения до вызова любого обработчика.
- Вы установили [только мой код](../debugger/just-my-code.md), и отладчик настроен на прерывание на любом исключении, которое не обрабатывается в пользовательском коде.

> [!NOTE]
> В ASP.NET существует обработчик исключений верхнего уровня, отображающий станицы ошибок в браузере. Он не нарушает выполнение, если не включен **только мой код** . Пример см. в разделе [сообщить отладчику о необходимости продолжения для необработанных пользовательских исключений](#BKMK_UserUnhandled) ниже.

<!-- Two consecutive notes are intentional here...-->

> [!NOTE]
> В приложениях, написанных на Visual Basic, отладчик управляет всеми ошибками как исключениями, даже при использовании обработчиков ошибок типа On Error.

## <a name="tell-the-debugger-to-break-when-an-exception-is-thrown"></a>Сообщить отладчику о прерывании при возникновении исключения

Отладчик может прерывать выполнение в точке, где создается исключение, поэтому вы можете проверить исключение перед вызовом обработчика.

В окне **параметры исключений** (**Отладка > Windows > Параметры исключений**) разверните узел для категории исключений, например **исключения среды CLR**. Затем установите флажок для конкретного исключения в этой категории, например **System. AccessViolationException**. Можно также выбрать всю категорию исключений.

![Проверено AccessViolationException](../debugger/media/exceptionsettingscheckaccess.png "ексцептионсеттингсчеккакцесс")

> [!TIP]
> Конкретные исключения можно найти в окне **Поиск** на панели инструментов **параметры исключений** или с помощью кнопки Поиск для фильтрации конкретных пространств имен (например, **System.IO**).

Если в окне " **параметры исключений** " выбрать исключение, выполнение отладчика будет прерываться везде, где возникает исключение, независимо от того, обрабатывается ли оно. Теперь исключение называется первым шансом исключения. Ниже приведено несколько примеров.

- В следующем консольном приложении C# метод Main создает исключение **AccessViolationException** внутри блока `try/catch`.

  ```csharp
  static void Main(string[] args)
  {
      try
      {
          throw new AccessViolationException();
          Console.WriteLine("here");
      }
      catch (Exception e)
      {
          Console.WriteLine("caught exception");
      }
      Console.WriteLine("goodbye");
  }
  ```

  Если вы вернули **параметры исключения** **AccessViolationException** , выполнение будет прерываться на `throw` строке при выполнении этого кода в отладчике. После этого выполнение можно продолжить. В консоли должны отображаться обе строки.

  ```cmd
  caught exception
  goodbye
  ```

  но строка `here` не отображается.

- C# Консольное приложение ссылается на библиотеку классов с классом, имеющим два метода. Один метод создает исключение и обрабатывает его, в то время как второй метод создает такое же исключение, но не обрабатывает его.

  ```csharp
  public class Class1
  {
      public void ThrowHandledException()
      {
          try
          {
              throw new AccessViolationException();
          }
          catch (AccessViolationException ave)
          {
              Console.WriteLine("caught exception" + ave.Message);
          }
      }

      public void ThrowUnhandledException()
      {
          throw new AccessViolationException();
      }
  }
  ```

  Далее приводится метод Main() консольного приложения:

  ```csharp
  static void Main(string[] args)
  {
      Class1 class1 = new Class1();
      class1.ThrowHandledException();
      class1.ThrowUnhandledException();
  }
  ```

  Если **AccessViolationException** вернул **параметры исключений**, выполнение будет прерываться на `throw` строке в **сровхандледексцептион ()** и **сровунхандледексцептион ()** при выполнении этого кода в отладчике.

Чтобы восстановить параметры исключений до значений по умолчанию, нажмите кнопку **восстановить список до кнопки Параметры по умолчанию** :

![Восстановить значения по умолчанию в параметрах исключений](../debugger/media/restoredefaultexceptions.png "ресторедефаултексцептионс")

## <a name="BKMK_UserUnhandled"></a>Сообщить отладчику об продолжении обработки необработанных пользовательских исключений

При отладке кода .NET или JavaScript с [только мой код](../debugger/just-my-code.md)можно указать отладчику, чтобы избежать нарушения исключений, которые не обрабатываются в пользовательском коде, но обрабатываются в других местах.

1. В окне **параметры исключений** откройте контекстное меню, щелкнув правой кнопкой мыши метку столбца, а затем выберите пункт **Показывать столбцы > дополнительных действий**. (Если вы отключили **только мой код**, эта команда отображаться не будет.) Появится третий столбец с именем **Дополнительные действия** .

   ![Столбец "дополнительные действия"](../debugger/media/additionalactionscolumn.png "аддитионалактионсколумн")

   Для исключения, которое показывает, **продолжить, когда необработанный код пользователя** в этом столбце, отладчик продолжает работу, если это исключение не обрабатывается в пользовательском коде, но обрабатывается извне.

2. Чтобы изменить этот параметр для конкретного исключения, выберите исключение, щелкните правой кнопкой мыши, чтобы открыть контекстное меню, и выберите пункт **Продолжить при необработанном коде пользователя**. Вы также можете изменить параметр для всей категории исключений, например для всех исключений среды CLR.

   ![* * Продолжить при необработанном значении в пользовательском коде * *](../debugger/media/continuewhenunhandledinusercodesetting.png "континуевхенунхандлединусеркодесеттинг")

Например, веб-приложения ASP.NET обрабатывают исключения, преобразуя их в код состояния HTTP 500 ([обработка исключений в веб-API ASP.NET](/aspnet/web-api/overview/error-handling/exception-handling)), что может не помочь определить источник исключения. В следующем примере пользовательский код вызывает метод `String.Format()` , который создает <xref:System.FormatException>. Выполнение прерывается следующим образом.

![Прерывания при необработанном исключении пользователя&#45;](../debugger/media/exceptionunhandledbyuser.png "ексцептионунхандледбюсер")

## <a name="add-and-delete-exceptions"></a>Добавление и удаление исключений

Исключения можно добавлять и удалять. Чтобы удалить тип исключения из категории, выберите исключение и нажмите кнопку **Удалить выбранное исключение из списка** (знак "минус") на панели инструментов " **параметры исключения** ". Можно также щелкнуть исключение правой кнопкой мыши и выбрать в контекстном меню команду **Удалить** . Удаление исключения приводит к тому же результату, что и Непроверяемое исключение, что заключается в том, что отладчик не будет прерываться при возникновении исключения.

Чтобы добавить исключение, сделайте следующее:

1. В окне **параметры исключений** выберите одну из категорий исключений (например, **Среда CLR**).

2. Нажмите кнопку **Добавить исключение в выбранную категорию** (знак "плюс").

   ![* * Добавить исключение в выбранную категорию * *](../debugger/media/addanexceptiontotheselectedcategorybutton.png "адданексцептионтосеселектедкатегорибуттон")

3. Введите имя исключения (например, **System. UriTemplateMatchException**).

   ![Введите имя исключения](../debugger/media/typetheexceptionname.png "типесиксцептионнаме")

   Исключение добавляется в список (в алфавитном порядке) и автоматически проверяется.

Чтобы добавить исключение к исключениям доступа к памяти GPU, исключениям среды выполнения JavaScript или категориям исключений Win32, добавьте код ошибки и описание.

> [!TIP]
> Проверьте правильность написания! В окне **Параметры исключений** не проверяется существование добавленного исключения. Поэтому при вводе **Sytem.UriTemplateMatchException** появится запись для этого исключения (а не для **System.UriTemplateMatchException**).

Параметры исключения сохраняются в файл SUO решения и таким образом применяются к конкретному решению. Параметры конкретного исключения нельзя повторно использовать в решениях. Теперь сохраняются только добавленные исключения. Удаленные исключения отсутствуют. Вы можете добавить исключение, закрыть и снова открыть решение, и исключение останется там. Однако при удалении исключения, закрытии и повторном открытии решения исключение появится снова.

В окне **Параметры исключений** поддерживаются универсальные типы исключений на C#, но не на Visual Basic. Чтобы делать останов при возникновении таких исключений, как `MyNamespace.GenericException<T>`, необходимо добавить исключение в виде **MyNamespace.GenericException'1**. То есть, если создано исключение, такое как код:

```csharp
public class GenericException<T> : Exception
{
    public GenericException() : base("This is a generic exception.")
    {
    }
}
```

Исключение можно добавить в **параметры исключения** с помощью предыдущей процедуры:

![Добавление универсального исключения](../debugger/media/addgenericexception.png "аддженерицексцептион")

## <a name="add-conditions-to-an-exception"></a>Добавление условий к исключению

Окно **параметры исключений** используется для задания условий исключений. В настоящее время поддерживаются следующие условия: имена модулей, которые следует включить в исключение или исключить из него. При задании имен модулей в качестве условий можно приостановить исключение только для определенных модулей кода. Вы также можете избежать нарушения работы определенных модулей.

> [!NOTE]
> Добавление условий в исключение поддерживается начиная с [!include[vs_dev15](../misc/includes/vs_dev15_md.md)].

Чтобы добавить условные исключения, выполните следующие действия.

1. Нажмите кнопку **изменить условия** в окне Параметры исключений или щелкните исключение правой кнопкой мыши и выберите команду **изменить условия**.

   ![Условия для исключения](../debugger/media/dbg-conditional-exception.png "дбгкондитионалексцептион")

2. Чтобы добавить дополнительные условия, необходимые для исключения, выберите **Добавить условие** для каждого нового условия. Отобразятся дополнительные строки условий.

   ![Дополнительные условия для исключения](../debugger/media/extraconditionsforanexception.png "екстракондитионсфоранексцептион")

3. Для каждой строки условия введите имя модуля и измените список оператор сравнения на **равно** или **не равно**. В имени можно указать символы-шаблоны ( **\\\*** ), чтобы указать более одного модуля.

4. Если необходимо удалить условие, выберите **X** в конце строки условия.
