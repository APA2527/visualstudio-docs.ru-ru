# <a name="whats-new-for-the-debugger-in-visual-studio-2017"></a>Новые возможности отладчика в Visual Studio 2017

Отладчик включает следующие новые функции:

- Новое в версии 15,5 **snapshot Debugger** создает моментальный снимок ваших рабочих приложений при выполнении кода, который вас интересует. Чтобы указать отладчику на необходимость создать моментальный снимок, следует установить точки прикрепления и точки ведения в коде. Отладчик позволяет увидеть источник ошибки, не затрагивая трафик рабочего приложения. Средство Snapshot Debugger позволяет значительно сократить затраты времени на устранение проблем, возникающих в рабочих средах.

    Коллекция моментальных снимков доступна для следующих веб-приложений, выполняющихся в службе приложений Azure App:

  * Приложения ASP.NET, выполняющиеся на платформе .NET Framework 4.6.1 или более поздней версии.
  * Приложения ASP.NET Core, выполняющиеся на платформе .NET Core 2.0 или более поздней версии под управлением Windows.

    Дополнительные сведения см. в разделе [Отладка интерактивных приложений ASP.NET с использованием Snapshot Debugger](../debugger/debug-live-azure-applications.md).

- Новые в версии 15,5 только в Visual Studio Enterprise, **Шаг с заходом IntelliTrace** автоматически создает моментальный снимок приложения на каждой точке останова и на шаге отладчика. Используя записанные моментальные снимки, вы можете возвращаться к этим точкам останова и шагам, просматривая предыдущее состояние приложения. Возможность возврата на шаг назад в IntelliTrace позволяет сэкономить время в тех случаях, когда вам нужно просмотреть предыдущее состояние приложения, но не требуется перезапускать отладку или воссоздавать необходимое состояние приложения.

    Для просмотра моментальных снимков и перехода между ними используйте кнопки **На шаг назад** и **На шаг вперед** на панели инструментов отладки. С помощью этих кнопок можно перейти к событиям, которые отображаются на вкладке **События** в окне **Средства диагностики**.

    ![Кнопки "Назад" и "Вперед"](../debugger/media/intellitrace-step-back-icons-description.png  "Кнопки "Назад" и "Вперед"")

    Дополнительные сведения см. в разделе [Проверка предыдущих состояний приложения с помощью IntelliTrace](view-historical-application-state.md).

- **Вспомогательный метод исключения** заменяет помощник по исключениям и появляется в немодальном диалоговом окне, в котором произошла ошибка. **Модуль поддержки исключений** обеспечивает более быстрый доступ к любым внутренним исключениям, дополнительному анализу отладчика (если он доступен) и мгновенному доступу к **параметрам исключений** для исключения. Кроме того, вспомогательное приложение исключения можно перетащить в плавающее представление, если оно блокирует что-либо, которое нужно увидеть.

    Например, в **NullReferenceException** теперь отображается переменная, имеющая пустую ссылку (Дополнительные сведения).

    ![Вспомогательное приложение исключения отладчика](../debugger/media/dbg-exception-helper.png "дбжексцептионхелпер")

    Дополнительные сведения см. в публикации [Using the New Exception Helper in Visual Studio](https://devblogs.microsoft.com/devops/using-the-new-exception-helper-in-visual-studio-15-preview/) (Использование нового помощника по исправлению ошибок в Visual Studio).

- Теперь можно выполнять команду в строке кода, приостанавливаясь к отладчику, выбрав значок **выполнить выполнение до** значка зеленой стрелки (значок выполнения), который отображается при наведении указателя мыши на строку кода. Это устраняет необходимость в установке временных точек останова.

    ![Запуск отладчика для щелчка](../debugger/media/dbg-run-to-click.png "дбгрунтокликк")

- Условия для исключений можно задать в диалоговом окне **параметры исключений** (это можно сделать с помощью значка **изменить условие** в диалоговом окне Параметры исключений или щелкнув правой кнопкой мыши в контекстном меню исключения). В настоящее время поддерживаются следующие условия: имена модулей, которые следует включить в исключение или исключить из него.

    ![Условия для исключения](../debugger/media/dbg-conditional-exception.png "дбгкондитионалексцептион")

- Диалоговое окно «Присоединение к процессу» содержит новую функцию поиска, которая позволяет быстрее определить процесс, к которому необходимо присоединиться.

    ![Поиск в присоединение к процессу](../debugger/media/dbg-attach-to-process-search.png "дбгаттачтопроцесссеарч")

Дополнительные сведения об этих новых функциях см. в [заметках о Выпуске [!include[vs_dev15](../misc/includes/vs_dev15_md.md)]](/visualstudio/releasenotes/vs2017-relnotes).
