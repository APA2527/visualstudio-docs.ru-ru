# <a name="visual-studio-graphics-diagnostics"></a>Диагностика графики в Visual Studio
Visual Studio*Диагностика графики* — это набор средств для записи и анализа проблем с визуализацией и производительностью в приложениях Direct3D. Диагностику графики можно использовать для приложений, которые выполняются локально на компьютере под управлением Windows, в эмуляторе устройства Windows либо на удаленном компьютере или устройстве.

 Рабочий процесс диагностики графики начинается с регистрации того, как приложение использует Direct3D — в оперативном режиме непосредственно во время выполнения. Это позволяет сразу проанализировать поведение приложения, предоставить соответствующие сведения для общего доступа либо сохранить их для использования в дальнейшем. Сеансы отслеживания можно инициировать и контролировать вручную из Visual Studio или с помощью средства записи командной строки **dxcap. exe**. Сеансы отслеживания также можно инициировать и контролировать программно с помощью диагностика графики API-интерфейсов записи.

 После записи сеанса регистрации его содержимое можно в любое время воспроизвести с помощью *анализатора графики* Visual Studio, воссоздавая записанные кадры с применением точно таких же ресурсов и команд отрисовки, которые использовались в приложении. Затем с помощью средств, предоставляемых в окне анализатора графики, можно подробно анализировать все захваченные кадры. С помощью этих инструментов можно проверить любой вызов Direct3D API, ресурс, объект состояния конвейера, этап конвейера или даже полный журнал всех пикселей в записанном кадре. Благодаря совместному использованию этих инструментов проблемы с отрисовкой можно анализировать интуитивно, начиная с ее проявления на записанном кадре и выполняя детализацию до первопричины в исходном коде приложения, шейдерах или графических ресурсах.

 Для диагностики проблем с производительностью записанный кадр можно проанализировать с помощью инструмента *Анализ кадров*. Этот инструмент анализирует потенциальные оптимизации производительности, автоматически изменяя способ использования Direct3D приложением и оценивая все отклонения. Возможно, раньше вам приходилось вручную вносить и оценивать такие изменения просто для того, чтобы подобрать нужное. Благодаря анализу кадров вы можете вносить только те изменения, вы обязательно дадут результат.

 Диагностика графики помогает обогатить графическую составляющую приложения Direct3D и оптимизировать его выполнение.

 Перейдите к [обзору](overview-of-visual-studio-graphics-diagnostics.md) , чтобы узнать, какие предложения Visual Studio диагностика графики.

## <a name="in-this-section"></a>Содержание
 [Общие сведения](overview-of-visual-studio-graphics-diagnostics.md) Вводит диагностика графики рабочий процесс и средства.

 [Начало работы](getting-started-with-visual-studio-graphics-diagnostics.md) В этом разделе вы узнаете, как установить Visual Studio диагностика графики и как начать использовать диагностика графики с приложением Direct3D.

 [Запись сведений о графике](capturing-graphics-information.md) Чтобы использовать диагностика графики для изучения проблемы отрисовки в приложении, вы сначала записываете сведения о том, как приложение использует DirectX. Во время сеанса записи приложение работает в обычном режиме, а вы *захватываете* (т. е. выбираете) интересующие вас кадры. Захваченные данные содержат подробные сведения о том, как отрисовываются кадры. Собранные данные можно сохранить в виде документа журнала графики, чтобы просмотреть его позже или показать другим членам команды.

 [Использование GPU](../../profiling/gpu-usage.md) Чтобы использовать диагностика графики для профилирования приложения, используйте средство использования GPU. Этот инструмент можно использовать в сочетании с другими инструментами профилирования, такими как «Загрузка ЦП», для сопоставления активности ЦП и GPU, которая может вызывать появление проблем с производительностью в приложении.

 [Документ журнала графики](graphics-log-document.md) Чтобы начать изучение записанного журнала графики, используйте окно документа журнала графики для выбора захваченного кадра или даже определенного пикселя, чтобы можно было подробно изучить *события* (то есть вызовы API DirectX), влияющие на него.

 [Анализ кадров](graphics-frame-analysis.md) После выбора кадра Используйте анализ кадров графики для проверки и настройки производительности отрисовки.

 [Список событий](graphics-event-list.md) После выбора кадра используйте **список событий графики** , чтобы проверить его события, чтобы определить, связаны ли они с проблемой подготовки к просмотру.

 [Состояние](graphics-state.md) Окно состояния помогает понять состояние графики, которая активна во время текущего события.

 [Этапы конвейера](graphics-pipeline-stages.md) В окне **Этапы графического конвейера** вы узнаете, как текущее выбранное событие обрабатывается на каждом этапе графического конвейера, чтобы можно было определить, где отображается сообщение о проблеме отрисовки. Анализ этапов конвейера особенно полезен, когда объект не отображается из-за неверного преобразования или когда один из этапов выдает результат, не соответствующий ожиданиям следующего этапа.

 [Стек вызовов событий](graphics-event-call-stack.md) **Стек вызовов событий Graphics** используется для проверки стека вызовов текущего выбранного события, чтобы можно было переходить к коду приложения, связанному с проблемой визуализации.

 [Журнал пикселей](graphics-pixel-history.md) С помощью окна « **Журнал пикселов графики** » для анализа того, как затронутые в данный момент Пиксели зависят от событий, влияющих на него, можно вычислить событие или сочетание событий, вызывающих определенные виды проблем отрисовки. Журнал пикселей особенно полезен, когда объект обрабатывается неправильно из-за того, что выходные данные пиксельного шейдера неверны или были неправильно объединены с буфером кадров, или когда объект даже не появляется, поскольку его пиксели были удалены до попадания в буфер кадров.

 [Таблица объектов](graphics-object-table.md) **Таблица объект Graphics** используется для проверки свойств и содержимого конкретных объектов и ресурсов Direct3D, действующих для текущего выбранного события. Таблица объектов может помочь определить контекст графического устройства, который активен во время события, и изучить содержимое графических ресурсов, например буферов констант, буферов вершин и текстур.

 [Отладчик HLSL](hlsl-shader-debugger.md) Чтобы узнать, как работает код шейдера для выбранного этапа конвейера событий и графики, используйте **отладчик HLSL** для пошагового выполнения кода, проверки содержимого переменных и выполнения других типичных задач отладки. Отладчик HLSL можно также использовать для проверки кода вычислительного шейдера независимо от того, обрабатываются ли его результаты далее конвейером графики или просто считываются приложением.

 [Средство захвата командной строки](command-line-capture-tool.md) Используйте средство захвата командной строки, чтобы быстро записывать и воспроизводить графические данные без использования Visual Studio или программного захвата. В частности, программу командной строки можно использовать в целях автоматизации или в тестовой среде.

 [Примеры](graphics-diagnostics-examples.md) В нескольких примерах показано, как использовать инструменты диагностика графики вместе для диагностики различных типов проблем отрисовки.
