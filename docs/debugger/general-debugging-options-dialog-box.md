# <a name="general-debugging-options"></a>Общие параметры отладки

Чтобы установить параметры отладки Visual Studio, выберите **Параметры инструментов** > **Options**и при **отладке** выберите или отберите коробки рядом с **общими** опциями. Можно восстановить все настройки по умолчанию с помощью > **настроек импорта и экспорта инструментов,** > **сбросив все настройки.** **Tools** Чтобы сбросить подмножество настроек, сохраните настройки с **помощью Мастера параметров импорта и экспорта,** прежде чем вносить изменения, которые вы хотите протестировать, а затем импортируйте сохраненные настройки.

Вы можете установить следующие **общие** параметры:

**Спросите, прежде чем удалить все точки разрыва:** Требуется подтверждение перед завершением команды **Удалить все точки разрыва.**

**Перерыв все процессы, когда один процесс перерывов**: Одновременно перерывы всех процессов, к которым отладчик прилагается, когда перерыв происходит.

**Перерыв, когда исключения пересекают AppDomain или управляемые/родные границы:** В управляемой или смешанной отладке языка время выполнения общего языка может поймать исключения, которые пересекают границы домена приложения или управляемые/родные границы, когда следующие условия верны:

1. Машинный код вызывает управляемый код с использованием COM-взаимодействия; при этом в управляемом коде возникает исключение. Смотрите [Введение в COM Interop](/dotnet/articles/visual-basic/programming-guide/com-interop/introduction-to-com-interop).

2. Управляемый код, выполняемый в домене приложения 1, вызывает управляемый код домена приложения 2; при этом в управляемом коде домена приложения 2 возникает исключение. См. [Программирование с использованием доменов приложений и сборок](/dotnet/articles/framework/app-domains/index).

3. Когда код вызывает функцию с помощью отражения, и эта функция бросает исключение. Смотрите [Отражение](/dotnet/framework/reflection-and-codedom/reflection).

В условиях 2 и 3 исключение иногда `mscorlib` захваливается управляемым кодом, а не общим временем выполнения языка. Этот параметр не влияет на прерывание по исключениям, перехватываемым с помощью библиотеки `mscorlib`.

**Включить отладку адресного уровня**: Включает расширенные функции для отладки на уровне адреса (окно **разборки,** окно регистров и точки разрыва **адресов).**

- **Отображение, если источник недоступен:** Автоматически отображаетокно окно **Разборки** при отладке кода, источник которого недоступен.

**Включить фильтры точки разрыва**: Позволяет устанавливать фильтры на точках разрыва, чтобы они могли влиять только на конкретные процессы, потоки или компьютеры.

**Используйте новый помощник исключения**: Включает в себя помощника по исключению, который заменяет помощника исключения. (Помощник исключения поддерживается начиная с Visual Studio 2017)

> [!NOTE]
> Для управляемого кода эта опция ранее называлась **«Включить помощник исключения».**

**Включить просто мой код**: отладчик отображает и шаги в пользовательский код ("Мой код") только, игнорируя код системы и другой код, который оптимизирован или который не имеет отладки символов.

- **Предупредите, если нет кода пользователя при запуске (только управляется):** Когда отладка начинается с включенного просто моего кода, эта опция предупреждает вас, если нет кода пользователя ("Мой код").

**Включить шаг амбиотчатого источника .NET:** позволяет отладчику войти в источник .NET Framework. Включение этой опции автоматически отключит Just My Code. Символы рамочного соглашения .NET будут загружены в место кэша. Измените местоположение кэша с помощью диалогового окна **Options,** категория **"Отладка",** страница **"Символы".**

**Перешагните через свойства и операторов (только управляемые)**: Предотвращает переход от отладчика в свойства и операторов в управляемый код.

**Включить оценку свойств и другие неявные вызовы функции:** Включает автоматическую оценку свойств и неявные вызовы функций в окнах переменных и в диалоговом окне **quickWatch.**

- **Функция преобразования строки вызова на объектах в окнах переменных (только c и JavaScript)**: выполняет неявный вызов преобразования строки при оценке объектов в окнах переменных. Результат отображается как строка вместо имени типа. Применимо только при отладке кода C#. Эта настройка может быть отменена атрибутом DebuggerDisplay [(см. Использование атрибута DebuggerDisplay).](../debugger/using-the-debuggerdisplay-attribute.md)

**Включить поддержку исходного сервера**: Говорит Visual Studio отладчика, чтобы получить`srcsrv.dll`исходные файлы из исходных серверов, которые реализуют Протокол SrcSrv . Team Foundation Server и инструменты отладки для Windows — два исходных сервера, которые реализуют этот протокол. Для получения дополнительной информации о настройке [SrcSrv](/windows-hardware/drivers/debugger/srcsrv) SrcSrv см. Кроме того, [см. Символ уточнения (.pdb) и исходные файлы.](../debugger/specify-symbol-dot-pdb-and-source-files-in-the-visual-studio-debugger.md)

> [!IMPORTANT]
> Поскольку чтение файлов *.pdb* может выполнять произвольный код в файлах, убедитесь, что вы доверяете серверу.

- **Печать диагностических сообщений сервера исходного кода в окне вывода:** Когда поддержка исходного сервера включена, эта настройка включает диагностический дисплей.

- **Разрешить исходный сервер для частичных сборок доверия (только управляемых):** Когда поддержка исходного сервера включена, эта настройка перекрывает поведение по умолчанию, не извлекающее источники для частичных сборок доверия.

- **Всегда запускайте ненадежные команды сервера исходного кода без запроса:** Когда поддержка исходного сервера включена, эта настройка перекрывает поведение по умолчанию запроса при запуске ненадежной команды.

**Поддержка ссылки источника**: Говорит Visual Studio отладчик для загрузки исходных файлов для *.pdb* файлов, которые содержат информацию об источнике ссылки. Для получения дополнительной информации об источнике ссылки, [см.](/dotnet/standard/library-guidance/sourcelink)

> [!IMPORTANT]
> Поскольку Source Link будет загружать файлы с помощью http или https, убедитесь, что вы доверяете файлу *.pdb.*

- **Отступить к проверке подлинности Git Credential Manager для всех запросов Source Link:** Когда включена поддержка ссылки на источник, а запрос Source Link не выполняет проверку подлинности, Visual Studio вызывает менеджера учетных данных Git.

**Выделите всю строку для точек разрыва и текущей оператора (только ск)**: Когда отладчик выделяет точку разрыва или текущее заявление, он выделяет всю строку.

**Требуйте исходных файлов, чтобы точно соответствовать исходной версии:** Говорит отладчику проверить, соответствует ли исходному файлу версию исходного кода, используемого для создания отладки. Когда версия не совпадает, вам предлагается найти соответствующий источник. Если соответствующий исходный код не найден, в процессе отладки исходный код не отображается.

**Перенаправьте весь текст окна вывода в окно Немедленное:** Отправляет все сообщения отладчика, которые обычно отображаются в окне **вывода** в окно **Immediate** вместо этого.

**Отображайте необработанную структуру объектов в окнах переменных:** отключает все настройки представления структуры объекта. Для получения дополнительной информации о настройках представления [см.](../debugger/create-custom-views-of-managed-objects.md)

**Подавление оптимизации JIT на загрузке модуля (только управляемый)**: отклоняет оптимизацию управляемого кода JIT при загрузке модуля и компилируется JIT при подключении отладчика. Отключение оптимизации позволяет упростить процесс отладки некоторых проблем, однако может отрицательно сказаться на производительности. В режиме "Только мой код" при отключении JIT-оптимизации в коде пользователя ("Мой код") может отображаться посторонний код. Для получения дополнительной [JIT optimization and debugging](../debugger/jit-optimization-and-debugging.md)информации см.

**Включить отладку JavaScript для ASP.NET (Chrome, Microsoft Edge и IE)**: Позволяет отладить скрипт для ASP.NET приложений. При первом использовании в Chrome, возможно, потребуется войти в браузер, чтобы включить Chrome расширения, которые вы установили. Отсоедините эту опцию, чтобы вернуться к устаревшему поведению.

**Включить инструменты для разработчиков Edge для приложений UWP JavaScript (экспериментальные)**: Включает в себя инструменты для разработчиков для приложений UWP JavaScript в Microsoft Edge.

**Включите устаревший отладчик Chrome JavaScript для ASP.NET:** Позволяет отлавить устаревший сценарий Chrome JavaScript для ASP.NET приложений. При первом использовании в Chrome, возможно, потребуется войти в браузер, чтобы включить Chrome расширения, которые вы установили.

**Используйте экспериментальный способ запуска отладки Chrome JavaScript при запуске Visual Studio в качестве администратора:** Говорит Visual Studio, чтобы попробовать новый способ запуска Chrome во время отладки JavaScript.

**Загрузка dll экспорта (только родной)**: Загрузок dll экспортных таблиц. Символьные данные из таблиц экспорта библиотеки DLL можно использовать при работе с сообщениями Windows, процедурами Windows (WindowProcs), COM-объектами, при маршалинге или при работе с любой библиотекой DLL, для которой нет символов. Считывание данных экспорта библиотеки DLL связано с определенными дополнительными затратами. Поэтому данная возможность по умолчанию отключена.

Чтобы посмотреть, какие символы доступны в таблице экспорта библиотеки DLL, воспользуйтесь командой `dumpbin /exports`. Символы доступны для любой 32-разрядной системной библиотеки DLL. В выходных данных команды `dumpbin /exports` можно увидеть точное имя функции, включая символы, отличные от буквенно-цифровых. Это полезно при задании точки останова в функции. Имена функций из таблиц экспорта библиотеки DLL могут отображаться в отладчике в сокращенном виде. Вызовы функций перечисляются в том порядке, в котором эти функции вызываются, при этом текущая функция (наиболее глубоко вложенная) располагается наверху. Дополнительные сведения см. в разделе [dumpbin /exports](/cpp/build/reference/dash-exports).

**Отображение параллельных стеков диаграммы снизу вверх**: Контролирует направление, в котором стеки отображаются в окне **параллельных стеков.**

**Не обращайте внимания на исключения доступа к памяти GPU, если записанные данные не изменили значение:** Игнорировать условия гонки, которые были обнаружены во время отладки, если данные не изменились. Для получения дополнительной [информации](../debugger/debugging-gpu-code.md)см.

**Используйте режим управляемой совместимости:** Заменяет движок отладки по умолчанию на устаревшую версию, чтобы включить следующие сценарии:

- Вы используете язык .NET, отличный от языка СЗ, Visual Basic или F, который предоставляет свой собственный оценщик экспрессии (в том числе и C-CLI).

- Необходимо включить отодевание и продолжение проектов c-образного режима во время отладки смешанного режима.

> [!NOTE]
> Выбор режима управляемой совместимости отсутствовает некоторые функции, которые реализуются только в двигателе отладки по умолчанию. Устаревший отладка двигателя был заменен в Visual Studio 2012.

**Используйте устаревшие оценщики выражений C и VB:** отладчик будет использовать оценщиков визуальных выражений Visual Studio 2013 или Visual Basic expression, а не оценщиков выражений на основе Visual Studio 2015 Roslyn.

**Предупредите при использовании пользовательских визуализаторов отладчика против потенциально небезопасных процессов (только управляемых):** Visual Studio предупреждает вас, когда вы используете пользовательский визуализатор отладчика, который работает код в отладительном процессе, потому что он может быть запущен небезопасный код.

**Включить отладку Windows кучи разляжитель (только родной)**: Включает в себя отладку окна кучу для улучшения диагностики кучи. Включение этого параметра повлияет на производительность отладки.

**Включить UI Отладка Инструменты для XAML**: Live Visual Tree и Live Свойство Исследуйте окна будут отображаться, когда вы начинаете отладку (**F5**) поддерживаемый тип проекта. Для получения дополнительной информации смотрите [свойства Inspect XAML во время отладки.](../xaml-tools/inspect-xaml-properties-while-debugging.md)

- **Предварительный просмотр выбранных элементов в Live Visual Tree**: Элемент XAML, контекст которого выбран, также выбран в окне Live Visual **Tree.**

- **Отображение инструментов выполнения в приложении:** Отображается команды **Live Visual Tree** в панели инструментов на главном окне приложения XAML, которое отлажено. Эта опция была введена в Visual Studio 2015 Update 2.

- **Включить XAML Горячая перезагрузка**: Позволяет использовать функцию XAML Горячая перезагрузка с кодом XAML, когда ваше приложение работает. (Эта функция ранее называлась "XAML Edit and Continue")

- **Выключите режим выбора при выборе элемента** Начиная с visual Studio 2019 версия 16.4, в приложении панель инструментов селектор кнопки (**Включить выбор**) выключается при выборе элемента. Если вы отделаете эту опцию, выбор элементов остается на кнопке панели инструментов в приложении.

**Включить диагностические инструменты во время отладки**: Окно **диагностических инструментов** появляется во время отладки.

**Отображение прошедшего времени PerfTip во время отладки:** Окно кода отображает прошедшее время вызова данного метода при отладке.

**Включить исключать и продолжать:** позволяет отсрочение и продолжить функциональность во время отладки.

- **Включить native Edit and Continue**: Вы можете использовать функцию «Оторит и продолжить» при отладке родного кода C.'. Для получения более подробной информации [см.](../debugger/edit-and-continue-visual-cpp.md)

- **Применяйте изменения на continue (только native)**: Visual Studio автоматически компилирует и применяет любые невыполненные изменения кода, внесенные при продолжении процесса из состояния перерыва. Если вы не выбраны, вы можете применить изменения, используя элемент **Apply Code Changes** в меню **Debug.**

- **Предупреждение о черствый код (только родной)**: Получить предупреждения о устаревший код.

**Показать кнопку Run to Click в редакторе во время отладки:** При выборе этой опции во время отладки будет отображаться кнопка [«Выполнить кнопку «Нажмите».](../debugger/debugger-feature-tour.md#run-to-a-point-in-your-code-quickly-using-the-mouse)

**Автоматически закрывай консоль при остановке отладки:**«Говорит Visual Studio о закрытии консоли в конце сеанса отладки.

::: moniker range=">= vs-2019"
**Включить быструю оценку выражения (только управляемый)**: Позволяет отладчику попытаться ускорить оценку, имитируя выполнение простых свойств и методов.
::: moniker-end

## <a name="options-available-in-older-versions-of-visual-studio"></a>Варианты, доступные в старых версиях Visual Studio

Если вы используете старую версию Visual Studio, могут присутствовать некоторые дополнительные опции.

**Включить помощник исключения:** Для управляемого кода, позволяет помощник уисключения. Начиная с Visual Studio 2017, Помощник исключения заменил помощника исключения.

**Раскрутите стек вызовов на необработанных исключениях:** Вызывает окно **Call Stack,** чтобы откатить стек вызова до того, как произошло необработанное исключение.

**Предупредите, если нет символов на запуске (только родной)**: Отображает предупреждающий диалоговый ящик, когда вы отлажаете программу, для которой отладчик не имеет информации о символах.

**Предупредите, если отладка скрипта отключена при запуске:** Отображает предупреждающий диалоговый ящик при запуске отладчика с отключением скрипта.

**Используйте режим нативной совместимости:** Когда эта опция выбрана, отладчик использует нативный отладчик Visual Studio 2010 вместо нового родного отладчика.

- Используйте эту опцию при отладке кода .NET C, потому что новый движок отладки не поддерживает оценку выражений .NET C. . Однако включение режима совместимости машинного кода отключает множество функций, которые зависят от работы текущей реализации отладчика. Например, в устаревшем движке не `std::string` хватает многих визуализаторов для встроенных типов, как в проектах Visual Studio 2015.   Используйте проекты Visual Studio 2013 для оптимального опыта отладки в этих случаях.
