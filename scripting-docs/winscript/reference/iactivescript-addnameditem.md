# <a name="iactivescriptaddnameditem"></a>IActiveScript::AddNamedItem
Добавляет имя элемента корневого уровня в пространство имен обработчика скриптов. Элемент корневого уровня — это объект со свойствами и методами, источником события или всеми тремя.  
  
## <a name="syntax"></a>Синтаксис  
  
```cpp
HRESULT AddNamedItem(  
    LPCOLESTR pstrName,  // address of item name  
    DWORD dwFlags          // item flags  
);  
```  
  
#### <a name="parameters"></a>Параметры  
 `pstrName`  
 окне Адрес буфера, который содержит имя элемента, просматриваемого из скрипта. Имя должно быть уникальным и постоянным.  
  
 `dwFlags`  
 окне Флаги, связанные с элементом. Может быть сочетанием следующих значений:  
  
|значения|Смысл|  
|-----------|-------------|  
|SCRIPTITEM_CODEONLY|Указывает, что именованный элемент представляет объект, содержащий только код, и что узел не имеет `IUnknown`, чтобы быть связанным с этим объектом только для кода. Узел имеет только имя для этого объекта. В объектно-ориентированных языках C++, таких как, этот флаг создает класс. Этот флаг поддерживается не для всех языков.|  
|SCRIPTITEM_GLOBALMEMBERS|Указывает, что элемент является коллекцией глобальных свойств и методов, связанных с скриптом. Как правило, обработчик сценариев будет игнорировать имя объекта (за исключением его использования в качестве файла cookie для метода [IActiveScriptSite:: GetItemInfo](../../winscript/reference/iactivescriptsite-getiteminfo.md) или разрешения явного определения области) и предоставить его членам в качестве глобальных переменных и методов. Это позволяет узлу расширить библиотеку (функции времени выполнения и т. д.), доступные для скрипта. Обработчик скриптов работает с конфликтами имен (например, если два SCRIPTITEM_GLOBALMEMBERS элемента имеют методы с одинаковыми именами), хотя ошибка не должна возвращаться из-за этой ситуации.|  
|SCRIPTITEM_ISPERSISTENT|Указывает, что элемент должен быть сохранен при сохранении обработчика скриптов. Аналогичным образом, установка этого флага означает, что переход к инициализированному состоянию должен хранить сведения об имени и типе элемента (обработчик сценариев должен, однако, освободить все указатели на интерфейсы на фактическом объекте).|  
|SCRIPTITEM_ISSOURCE|Указывает, что элемент источника содержит события, которые могут быть приемниками скрипта. Дочерние объекты (свойства объекта, находящихся в самих объектах) также могут быть источником событий в скрипте. Это не является рекурсивным, но предоставляет удобный механизм для общего случая, например, для создания контейнера и всех его элементов управления.|  
|SCRIPTITEM_ISVISIBLE|Указывает, что имя элемента доступно в пространстве имен скрипта, что позволяет получить доступ к свойствам, методам и событиям элемента. По соглашению свойства элемента включают дочерние элементы. Таким образом, все свойства и методы дочернего объекта (и их дочерние элементы, рекурсивно) будут доступны.|  
|SCRIPTITEM_NOCODE|Указывает, что элемент является просто именем, добавляемым в пространство имен скрипта, и не должен обрабатываться как элемент, с которым должен быть связан код. Например, если не задать этот флаг, VBScript создаст отдельный модуль для именованного элемента и C++ может создать отдельный класс-оболочку для именованного элемента.|  
  
## <a name="return-value"></a>Возвращаемое значение  
 Возвращает одно из следующих значений:  
  
|Возвращаемое значение|Смысл|  
|------------------|-------------|  
|`S_OK`|Выполнено.|  
|`E_INVALIDARG`|Недопустимый аргумент.|  
|`E_POINTER`|Указан недопустимый указатель.|  
|`E_UNEXPECTED`|Вызов не ожидался (например, обработчик скриптов еще не загружен или не инициализирован).|
